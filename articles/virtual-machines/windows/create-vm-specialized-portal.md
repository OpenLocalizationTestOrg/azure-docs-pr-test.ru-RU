# <a name="create-a-vm-from-a-vhd-using-the-azure-portal"></a>Создание виртуальной машины из виртуального жесткого диска с помощью портала Azure


В Azure существует несколько способов создания виртуальных машин. Если у вас уже есть виртуальный жесткий диск (VHD) или его нужно скопировать с существующей виртуальной машины для дальнейшего использования, можно создать новую виртуальную машину, присоединив VHD в качестве диска ОС. В результате этого процесса виртуальный жесткий диск *присоединяется* к новой виртуальной машине как диск операционной системы.

Также можно создать новую виртуальную машину из виртуального жесткого диска виртуальной машины, которая была удалена. Например, при наличии виртуальной машины Azure, которая работает неправильно, можно удалить эту машину и с помощью виртуального жесткого диска создать новую. Можно либо повторно использовать один и тот же виртуальный жесткий диск, либо создать его копию в виде моментального снимка с последующим созданием нового управляемого диска из этого снимка. Для этого требуется еще несколько действий, но так вы сможете гарантировать сохранность исходного виртуального жесткого диска и получите моментальный снимок для отката в случае необходимости.

У вас есть локальная виртуальная машина, на основе которой требуется создать виртуальную машину в Azure. Можно отправить VHD и присоединить его к новой виртуальной машине. Чтобы отправить виртуальный жесткий диск, потребуется PowerShell или другое средство для отправки в учетную запись хранения и последующего создания управляемого диска на основе VHD. Дополнительные сведения см. в разделе [Передача специализированного VHD](create-vm-specialized.md#option-2-upload-a-specialized-vhd).

Если вы хотите создать на основе виртуальной машины или виртуального жесткого диска несколько виртуальных машин, этот метод не следует использовать. Для крупных развертываний нужно [создать образ](capture-image-resource.md), а затем [использовать этот образ для создания нескольких виртуальных машин](create-vm-generalized-managed.md).


## <a name="copy-a-disk"></a>Копирование диска

Создайте моментальный снимок, а затем создайте диск на его основе. Это позволяет сохранять исходный виртуальный жесткий диск на случай возможного отката.

1. В меню слева выберите **Все ресурсы**.
2. В раскрывающемся списке **Все типы** снимите флажок **Выбрать все** и прокрутите вниз, чтобы выбрать **Диски** для поиска доступных дисков.
3. Щелкните диск, который следует использовать. Откроется страница **Обзор** для диска.
4. На странице "Обзор" в меню вверху щелкните **+ Создание моментального снимка**. 
5. Введите имя для моментального снимка.
6. Выберите **группу ресурсов** для моментального снимка. Вы можете выбрать существующую группу ресурсов или создать новую.
7. Выберите, какое хранилище следует использовать: стандартное (жесткие диски) или хранилище уровня "Премиум" (твердотельные накопители).
8. После завершения нажмите кнопку **Создать** для создания моментального снимка.
9. После создания моментального снимка щелкните **+ Создать ресурс** в меню слева.
10. В строке поиска введите **управляемый диск** и выберите **Управляемые диски** из списка.
11. На странице **Управляемые диски** щелкните **Создать**.
12. Введите имя для диска.
13. Выберите **группу ресурсов** для диска. Вы можете выбрать существующую группу ресурсов или создать новую. Эта группа ресурсов также будет использоваться при создании виртуальной машины из диска.
14. Выберите, какое хранилище следует использовать: стандартное (жесткие диски) или хранилище уровня "Премиум" (твердотельные накопители).
15. Убедитесь, что в поле **Тип источника** выбрано значение **Моментальный снимок**.
16. В раскрывающемся списке **Моментальный снимок источника** выберите снимок, который нужно использовать.
17. При необходимости внесите другие изменения и нажмите кнопку **Создать** для создания диска.

## <a name="create-a-vm-from-a-disk"></a>Создание виртуальной машины на основе диска

После получения виртуального диска, который требуется использовать, можно создать виртуальную машину на портале.

1. В меню слева выберите **Все ресурсы**.
2. В раскрывающемся списке **Все типы** снимите флажок **Выбрать все** и прокрутите вниз, чтобы выбрать **Диски** для поиска доступных дисков.
3. Щелкните диск, который следует использовать. Откроется страница **Обзор** для диска.
Убедитесь, что на странице обзора в поле **Состояние диска** указано значение **Отключен**. Если это не так, возможно, потребуется отключить диск от виртуальной машины или удалить виртуальную машину, чтобы освободить диск.
4. В меню в верхней части панели нажмите **+ Создать виртуальную машину**.
5. На странице **основных сведений** для новой виртуальной машины введите имя и либо выберите существующую группу ресурсов, либо создайте новую.
6. На странице **Размер** выберите размер страницы виртуальной машины и щелкните **Выбрать**.
7. На странице **Параметры** можно либо автоматически создать все новые ресурсы с помощью портала, либо выбрать существующие ресурсы **Виртуальная сеть** и **Группа безопасности сети**. Портал всегда создает новый сетевой адаптер и общедоступный IP-адрес для новой виртуальной машины. 
8. Внесите изменения в параметры мониторинга и при необходимости добавьте расширения.
9. Когда все будет готово, нажмите кнопку **ОК**. 
10. Если конфигурация виртуальной машины пройдет проверку, нажмите кнопку **ОК**, чтобы начать развертывание.
