# <a name="data-sources-in-log-analytics"></a>Источники данных в Log Analytics
Служба аналитики журналов собирает данные из подключенных источников и сохраняет его в рабочую область службы анализа журналов.  Собираемые данные зависят от заданного вами источника данных.  Служба аналитики журналов данные в набор записей.  Каждый источник данных создает записи определенного типа, а каждый тип обладает собственным набором свойств.

![Сбор данных в службе Log Analytics](./media/log-analytics-data-sources/overview.png)

Источники данных будут отличаться от [решения для управления](log-analytics-add-solutions.md), который также собирать данные из подключенных источников и создания записей в службе анализа журналов.  Помимо сбора данных, решения обычно включают поиск по журналам и представления, которые помогают анализировать работу конкретного приложения или службы.

## <a name="configuring-data-sources"></a>Настройка источников данных
Настройка источников данных из **данные** меню в службе анализа журналов **Дополнительные параметры**.  Какой-либо настройки доставляется на все подключенные источники в рабочей области.  В настоящее время исключать агенты из этой конфигурации нельзя.

![Настройка событий Windows](./media/log-analytics-data-sources/configure-events.png)

1. На портале Azure выберите **анализа журналов** > рабочей области > **Дополнительные параметры**.
2. Выберите **Данные**.
3. Щелкните источник данных, который требуется настроить.
4. Перейдите по ссылке на документацию для каждого источника данных в приведенной выше таблице и найдите сведения об их настройке.


## <a name="data-collection"></a>Сбор данных
Конфигурации источников данных передаются в агенты, подключенные к Log Analytics напрямую, в течение нескольких минут.  Указанные данные собираются из агента и передаются непосредственно в службу Log Analytics с периодичностью, указанной для каждого источника данных.  Особенности каждого источника данных см. в соответствующей документации.

Для агентов System Center Operations Manager в подключенной группе управления конфигурации источника данных преобразуется в пакеты управления и доставляются группе управления каждые 5 минут по умолчанию.  Агент загружает пакет управления обычным способом и собирает указанные данные. В зависимости от источника данных данные будут отправки на сервер управления, который пересылает данные аналитики журнала или агент будет отправлять данные аналитики журнала без перехода на сервере управления. Ссылаться на [сведения о сборе данных](log-analytics-add-solutions.md#data-collection-details) подробные сведения.  Вы можете прочесть об подробные сведения о подключении Operations Manager, а также анализа журналов и изменение частоты этой конфигурации доставки в [Настройка интеграции с System Center Operations Manager](log-analytics-om-agents.md).

Если агенту не удается подключиться к Log Analytics или Operations Manager, он будет продолжать собирать данные, чтобы передать их после успешного подключения.  Данные могут быть потеряны, если их объем достигнет максимального размера кэша для клиента или агент не сможет установить подключение в течение 24 часов.

## <a name="log-analytics-records"></a>Записи Log Analytics
Все данные, собранные службой аналитики журналов хранятся в рабочей области как записи.  Записи, собираемые различными источниками данных, получают собственный набор свойств и идентифицируются по свойству **Тип**.  Подробные сведения о каждом типе записей см. в документации по каждому типу источников и решению.
