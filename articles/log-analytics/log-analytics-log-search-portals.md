# <a name="portals-for-creating-and-editing-log-queries-in-azure-log-analytics"></a>Порталы для создания и изменения запросов к журналу в службе Azure Log Analytics

Поиск по журналам можно использовать в нескольких местах службы Log Analytics для извлечения данных из рабочей области.  Помимо интерактивной обработки полученных данных, у вас есть два способа создать и редактировать запросы. Оба они описаны ниже.  

## <a name="log-search"></a>Поиск по журналам 
К странице поиска по журналам можно получить доступ на портале Azure.  Он удобен для создания простых запросов, состоящих из одной строки.  Поиск по журналам можно использовать и без запуска внешних порталов. С его помощью вы сможете выполнить множество операций с поиском по журналам, например создать правила оповещений, создать группы компьютеров или экспортировать результаты запроса.  

Поиск по журналам предоставляет много возможностей для редактирования запросов, даже если вы не полностью разбираетесь в языке запросов.  Обзор этих функций вы найдете в статье в [Создание поисков по журналам в Azure Log Analytics с помощью портала поиска по журналам](log-analytics-log-search-log-search-portal.md).


![Страницы поиска по журналам](media/log-analytics-log-search-portals/log-search-portal.png)

## <a name="advanced-analytics-portal"></a>Портал расширенной аналитики
Портал расширенной аналитики представляет собой специализированный портал с расширенными функциональными возможностями, которые недоступны для поиска по журналам на портале Azure.  В числе таких возможностей можно назвать редактирование многострочных запросов, выборочное исполнение кода, Intellisense с зависимостью от контекста и смарт-аналитика.  Портал расширенной аналитики лучше всего подходит для разработки сложных запросов, которые затем можно сохранить в виде поиска по журналам или скопировать в другие элементы Log Analytics.  Чтобы открыть портал расширенной аналитики, используйте ссылку на странице поиска по журналам.

![Портал расширенной аналитики](media/log-analytics-log-search-portals/advanced-analytics-portal.png)


Благодаря этим широким возможностям портал расширенной аналитики, скорее всего, станет для вас основным инструментом создания и редактирования запросов.  Когда вы убедитесь, что запрос работает правильно, его можно скопировать и вставить в другой инструмент, например на страницу поиска по журналам или в конструктор представлений.  Но не забывайте, что только портал расширенной аналитики может работать с многострочными запросами. В этой связи при копировании запросов следует учесть следующее.

- Удалите из запроса все комментарии, прежде чем копировать его и вставлять в другие инструменты.  Комментарием считается любая строка, перед которой стоят две косые черты (//).  Когда вы вставляете нескольких строк запроса в одну строку, разрывы строк удаляются.  Если в этих строках останутся комментарии, все символы после первого комментария станут считаться комментарием.
