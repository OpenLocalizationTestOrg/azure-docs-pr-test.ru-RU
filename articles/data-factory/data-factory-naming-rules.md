# <a name="azure-data-factory---naming-rules"></a>Фабрика данных Azure — правила именования
Привет, в следующей таблице приведены правила именования для артефактов фабрики данных.

| Имя | Уникальность имени | Проверки |
|:--- |:--- |:--- |
| Фабрика данных |Уникально в рамках Microsoft Azure. Именах не учитывается регистр, то есть `MyDF` и `mydf` ссылаться toohello же фабрику данных. |<ul><li>Каждая фабрика данных — связанные tooexactly одной подписки Azure.</li><li>Имена объектов должно начинаться с буквы или цифры и может содержать только буквы, цифры и знак дефиса (-) hello.</li><li>Перед каждым знаком тире (-) и после него должна стоять буква или цифра. Последовательные тире в именах контейнеров использовать нельзя.</li><li>Имя может содержать от 3 до 63 символов.</li></ul> |
| Связанные службы/таблицы/конвейеры |Уникально в рамках фабрики данных. Регистр в именах не учитывается. |<ul><li>Максимальное количество символов в имени таблицы: 260.</li><li>Имена объектов должны начинаться с буквы, цифры или символа подчеркивания (_).</li><li>Следующие символы не допускаются: ".", "+", "?", "/", "<", ">", "*", "%", "&", ":", "\\".</li></ul> |
| Группа ресурсов |Уникально в рамках Microsoft Azure. Регистр в именах не учитывается. |<ul><li>Максимальное количество символов: 1000.</li><li>Имя может содержать буквы, цифры и следующие символы hello: «-», «_ «,», «и».»</li></ul> |
