<span data-ttu-id="46e45-164">Центр Интернета вещей Azure реализует шаблон [взаимодействия с поддержкой службы][lnk-service-assisted-pattern] и выступает в роли посредника во взаимодействии между устройствами и серверной частью решения.</span><span class="sxs-lookup"><span data-stu-id="46e45-164">Azure IoT Hub implements the [service-assisted communication][lnk-service-assisted-pattern] pattern to mediate the interactions between your devices and your solution back end.</span></span> <span data-ttu-id="46e45-165">Задача взаимодействия с поддержкой службы заключается в установлении путей доверенного двунаправленного взаимодействия между системой управления, такой как центр IoT, и специализированными устройствами, которые разворачиваются в недоверенном физическом пространстве.</span><span class="sxs-lookup"><span data-stu-id="46e45-165">The goal of service-assisted communication is to establish trustworthy, bidirectional communication paths between a control system, such as IoT Hub, and special-purpose devices that are deployed in untrusted physical space.</span></span> <span data-ttu-id="46e45-166">Данный шаблон устанавливает следующие принципы.</span><span class="sxs-lookup"><span data-stu-id="46e45-166">The pattern establishes the following principles:</span></span>

* <span data-ttu-id="46e45-167">Безопасность имеет приоритет над всеми другими возможностями.</span><span class="sxs-lookup"><span data-stu-id="46e45-167">Security takes precedence over all other capabilities.</span></span>

* <span data-ttu-id="46e45-168">Устройства не принимают сетевой информации, которую они не запрашивали.</span><span class="sxs-lookup"><span data-stu-id="46e45-168">Devices do not accept unsolicited network information.</span></span> <span data-ttu-id="46e45-169">Устройство устанавливает все подключения и маршруты только в исходящем режиме.</span><span class="sxs-lookup"><span data-stu-id="46e45-169">A device establishes all connections and routes in an outbound-only fashion.</span></span> <span data-ttu-id="46e45-170">Чтобы устройство получило команду от серверной части решения, оно должно регулярно инициировать подключение для проверки любых команд, ожидающих обработки.</span><span class="sxs-lookup"><span data-stu-id="46e45-170">For a device to receive a command from the solution back end, the device must regularly initiate a connection to check for any pending commands to process.</span></span>

* <span data-ttu-id="46e45-171">Устройства должны подключаться и устанавливать маршруты только с общеизвестными службами, которые являются их партнерами, такими как центр IoT.</span><span class="sxs-lookup"><span data-stu-id="46e45-171">Devices should only connect to or establish routes to well-known services they are peered with, such as IoT Hub.</span></span>

* <span data-ttu-id="46e45-172">Пути взаимодействия между устройством и службой и устройством и шлюзом защищены на уровне протокола приложения.</span><span class="sxs-lookup"><span data-stu-id="46e45-172">The communication path between device and service or between device and gateway is secured at the application protocol layer.</span></span>

* <span data-ttu-id="46e45-173">Авторизация и проверка подлинности системного уровня основаны на удостоверениях устройств.</span><span class="sxs-lookup"><span data-stu-id="46e45-173">System-level authorization and authentication are based on per-device identities.</span></span> <span data-ttu-id="46e45-174">Они позволяют практически мгновенно отзывать учетные данные и разрешения для доступа.</span><span class="sxs-lookup"><span data-stu-id="46e45-174">They make access credentials and permissions nearly instantly revocable.</span></span>

* <span data-ttu-id="46e45-175">Двунаправленное взаимодействие для устройств, которые подключаются нерегулярно из-за проблем с питанием или подключением, упрощается путем хранения команд и уведомлений для устройства до того момента, пока оно не подключится для их получения.</span><span class="sxs-lookup"><span data-stu-id="46e45-175">Bidirectional communication for devices that connect sporadically due to power or connectivity concerns is facilitated by holding commands and device notifications until a device connects to receive them.</span></span> <span data-ttu-id="46e45-176">Центр IoT хранит отправляемые команды в соответствующих очередях для каждого устройства.</span><span class="sxs-lookup"><span data-stu-id="46e45-176">IoT Hub maintains device-specific queues for the commands it sends.</span></span>

* <span data-ttu-id="46e45-177">Полезные данные приложения можно защитить отдельно для безопасной передачи через шлюзы к конкретной службе.</span><span class="sxs-lookup"><span data-stu-id="46e45-177">Application payload data is secured separately for protected transit through gateways to a particular service.</span></span>

<span data-ttu-id="46e45-178">Шаблон взаимодействия с поддержкой службы широко используется в отрасли мобильных устройств для реализации служб push-уведомлений, таких как [службы push-уведомлений Windows][lnk-wns], [Google Cloud Messaging][lnk-google-messaging] и [служба push-уведомлений Apple][lnk-apple-push].</span><span class="sxs-lookup"><span data-stu-id="46e45-178">The mobile industry has used the service-assisted communication pattern at enormous scale to implement push notification services such as [Windows Push Notification Services][lnk-wns], [Google Cloud Messaging][lnk-google-messaging], and [Apple Push Notification Service][lnk-apple-push].</span></span>

<span data-ttu-id="46e45-179">Центр Интернета вещей поддерживается по пути общедоступного пиринга ExpressRoute.</span><span class="sxs-lookup"><span data-stu-id="46e45-179">IoT Hub is supported over ExpressRoute's public peering path.</span></span>
