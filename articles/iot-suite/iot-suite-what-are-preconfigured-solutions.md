# <a name="what-is-azure-iot-suite"></a>Что такое Azure IoT Suite?

Azure IoT Suite представляет собой набор *предварительно настроенных решений*, с помощью которых вы можете:

* выполнить развертывание за считаные минуты;
* быстро приступить к работе;
* настроить Azure IoT Suite в соответствии со своими потребностями.

Предварительно настроенные решения *Azure IoT Suite* разработаны с учетом определенных принципов и целей.

В следующем видео представлен обзор предварительно настроенного решения для удаленного мониторинга.

## <a name="preconfigured-solutions-overview"></a>Обзор предварительно настроенных решений

Предварительно настроенное решение представляет собой реализацию с открытым исходным кодом стандартных шаблонов решений Интернета вещей, которые можно развернуть в Azure с помощью подписки. Каждое предварительно настроенное решение объединяет настраиваемый код и службы Azure для реализации определенного сценария или сценариев Интернета вещей. Вы можете настроить любой из сценариев в соответствии со своими требованиями. Ниже приведены соответствующие сценарии.

* Визуализация данных на расширенных панелях мониторинга для получения подробных сведений и состояния решения.
* Настройка правил и оповещений через динамическую телеметрию устройства Интернета вещей.
* Планирование заданий управления для устройства, например обновления программного обеспечения и конфигурации.
* Подготовка собственных настраиваемых физических или виртуальных устройств.
* Устранение проблем и неполадок в группах устройств Интернета вещей.

Каждое предварительно настроенное решение — это полная комплексная реализация, в которой для создания данных телеметрии используются виртуальные или физические устройства. Вы можете использовать эти решения как ускорители решений для следующих целей:

* чтобы предоставить отправную точку для собственных решений Интернета вещей;
* чтобы ознакомиться с распространенными шаблонами, которые используются при проектировании и разработке решений Интернета вещей.

На сегодня доступны три предварительно настроенных решения.

* [Удаленный мониторинг](iot-suite-remote-monitoring-explore.md)
* [Диагностическое обслуживание](iot-suite-predictive-overview.md)
* [Подключенная фабрика](iot-suite-connected-factory-overview.md)

В следующей таблице показано соответствие этих решений конкретным функциям IoT.

| Решение | Прием данных | Удостоверение устройства | Управление устройствами | Пограничные вычисления | Команды и управление | Правила и действия | Прогнозная аналитика |
| ------------------------------------------------------------ | -- | -- | -- | -- | -- | -- | -- |
| [Удаленный мониторинг](iot-suite-remote-monitoring-explore.md)  |Yes |Yes |Yes |-   |Yes |Yes |-   |
| [Диагностическое обслуживание](iot-suite-predictive-overview.md)   |Yes |Yes |-   |-   |Yes |Yes |Yes |
| [Подключенная фабрика](iot-suite-connected-factory-overview.md) |Yes |- |- |Yes |Yes |Yes |-   |

* *Прием данных*: прием данных в облако в нужном масштабе.
* *Удостоверения устройств*: управление уникальными удостоверениями устройств и контроль доступа устройств к решению.
* *Управление устройствами*: управление метаданными устройств и выполнение операций, таких как перезагрузка устройства и обновление встроенного ПО.
* *Команды и управление*: отправка сообщений на устройство из облака для выполнения на устройстве определенных действий.
* *Правила и действия*: правила используются в серверной части решения для операций с определенными данными, отправляемыми с устройства в облако.
* *Прогнозная аналитика*: данные, отправляемые с устройства в облако, анализируются в серверной части решения, что позволяет предсказать время выполнения определенных действий. Пример — анализ данных телеметрии, поступающих от двигателя самолета, который позволяет определить время обслуживания двигателя.

> [!NOTE]
> Чтобы развернуть предварительно настроенные решения и ознакомиться с дополнительными сведениями по их настройке, посетите [Microsoft Azure IoT Suite](https://www.azureiotsuite.com/).

## <a name="azure-services"></a>Службы Azure

При развертывании предварительно настроенного решения в ходе процесса подготовки настраивается ряд служб Azure. В таблице ниже показаны службы, используемые в предварительно настроенных решениях:

|                      | удаленный мониторинг;  | диагностическое обслуживание; | подключенная фабрика. |
| -------------------- | ------------------ | ---------------------- | ----------------- |
| Центр Интернета вещей              | Yes                |                        | Yes               |
| Концентраторы событий           |                    | Yes                    |                   |
| Time Series Insights |                    |                        | Yes               |
| Службы контейнеров   | Yes                |                        |                   |
| Stream Analytics     |                    | Yes                    |                   |
| Веб-приложения             | Yes                | Yes                    | Yes               |
| База данных Cosmos            | Yes                | Yes                    |                    |
| Хранилище Azure         |                    | Yes                    | Yes               |

> [!NOTE]
> Дополнительные сведения о ресурсах, развернутых в предварительно настроенном решении для удаленного мониторинга см. в этой [статье](https://github.com/Azure/azure-iot-pcs-remote-monitoring-dotnet/blob/master/README.md#basic-vs-standard-deployments) на сайте GitHub.

* [Центр Интернета вещей Azure](../iot-hub/index.md). Эта служба предоставляет возможности обмена сообщениями между устройствами и облаком, а также выступает в качестве шлюза для облака и других ключевых служб IoT Suite. Служба позволяет получать сообщения от устройств в реальном времени и отправлять команды на устройства. Служба также позволяет [управлять устройствами](../iot-hub/iot-hub-device-management-overview.md). Например, вы можете настроить, перезагрузить или восстановить заводские настройки на одном или нескольких устройствах, подключенных к центру.
* [Концентраторы событий Azure](../event-hubs/index.md). Эта служба позволяет получать события в крупных масштабах в облаке. Ознакомьтесь со статьей [Сравнение Центра Интернета вещей Azure и концентраторов событий Azure](../iot-hub/iot-hub-compare-event-hubs.md).
* [Аналитика временных рядов Azure](../time-series-insights/index.yml). Предварительно настроенные решения используют эту службу для анализа и отображения данных телеметрии с устройств.
* [Служба контейнеров Azure](../container-service/index.yml). С помощью этой службы в предварительно настроенных решениях выполняется размещение микрослужб и управление ими.
* [Azure Cosmos DB](../cosmos-db/index.yml) и [служба хранилища Azure](../storage/index.yml) для хранения данных.
* [Azure Stream Analytics](../stream-analytics/index.md). Предварительно настроенное решение для прогнозного обслуживания использует эту службу для обработки входящих данных телеметрии, агрегирования и обнаружения событий. Это решение также использует Stream Analytics для обработки информационных сообщений, содержащих, например, метаданные или ответы на команды от устройств.
* [Веб-приложения Azure](../app-service/index.yml) для размещения настраиваемого кода приложения в предварительно настроенных решениях.

Обзор архитектуры стандартного решения Интернета вещей см. в статье [Azure и "Интернет вещей"](iot-suite-what-is-azure-iot.md).

## <a name="whats-new-in-preconfigured-solutions"></a>Обновления в предварительно настроенных решениях

Корпорация Майкрософт обновляет предварительно настроенные решения до новой архитектуры на основе микрослужб. В таблице ниже показано текущее состояние в предварительно настроенных решениях:

| Предварительно настроенное решение | Архитектура  | Языки     |
| ---------------------- | ------------- | ------------- |
| удаленный мониторинг;      | Микрослужбы | Java и .NET |
| диагностическое обслуживание; | MVC           | .NET          |
| подключенная фабрика.      | MVC           | .NET          |

В разделах ниже описаны новые возможности предварительно настроенных решений на основе микрослужб.

### <a name="microservices"></a>Микрослужбы

В новой версии предварительно настроенного решения удаленного мониторинга используется архитектура микрослужб. Это решение состоит из нескольких микрослужб, таких как *диспетчер Центра Интернета вещей* и *диспетчер хранилища*. Вы можете загрузить версии Java и .NET каждой микрослужбы, а также соответствующую документацию для разработчиков. Дополнительные сведения о микрослужбах см. в статье [Пошаговое руководство по работе с настроенным решением для удаленного мониторинга](iot-suite-remote-monitoring-sample-walkthrough.md).

Эта архитектура микрослужб представляет собой надежный шаблон облачных решений, который:

* является масштабируемым;
* предоставляет возможности расширения;
* несложный в эксплуатации;
* позволяет использовать альтернативные службы.

> [!TIP]
> Дополнительные сведения об архитектурах микрослужб см. на странице [архитектуры приложения для .NET](https://www.microsoft.com/net/learn/architecture), а также в записи блога, посвященной [возможностям микрослужб — мощные возможности приложения, обеспечиваемые облачной средой](https://azure.microsoft.com/blog/microservices-an-application-revolution-powered-by-the-cloud/).

При развертывании новой версии удаленного мониторинга необходимо выбрать один из следующих вариантов развертывания:

* **Базовый.** Версия с низкой стоимостью для демонстрации или тестирования развертывания. Все микрослужбы развертываются на одной виртуальной машине Azure.
* **Стандартный.** Развертывание расширенной инфраструктуры для разработки развертывания в рабочей среде. С помощью Службы контейнеров Azure можно развернуть микрослужбы на нескольких виртуальных машинах Azure. Kubernetes управляет контейнерами Docker, на которых размещаются отдельные микрослужбы.

### <a name="language-choices-java-and-net"></a>Выбор языков: Java и .NET

Вы можете развернуть все микрослужбы на Java и .NET. Как и код .NET, код Java представляет собой открытый исходный код, доступный для настройки в соответствии с вашими требованиями.

* [Репозиторий удаленного мониторинга для .NET на GitHub](https://github.com/Azure/azure-iot-pcs-remote-monitoring-dotnet)
* [Репозиторий удаленного мониторинга для Java на GitHub](https://github.com/Azure/azure-iot-pcs-remote-monitoring-java)

Запрос на реализацию на других языках можно оставить на форуме [для Azure IoT](https://feedback.azure.com/forums/321918-azure-iot).

### <a name="react-user-interface-framework"></a>Платформа пользовательского интерфейса на основе библиотеки React

Пользовательский интерфейс создается с помощью библиотеки для JavaScript [React](https://facebook.github.io/react/). Исходный код представляет собой открытый исходный код, который можно загрузить и настроить.
