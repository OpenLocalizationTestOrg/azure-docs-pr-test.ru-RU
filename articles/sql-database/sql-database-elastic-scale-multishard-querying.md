## <a name="example"></a><span data-ttu-id="469f5-115">Пример</span><span class="sxs-lookup"><span data-stu-id="469f5-115">Example</span></span>
<span data-ttu-id="469f5-116">В следующем примере кода показано применение многосегментного формирования запросов с использованием заданного сопоставления **ShardMap** с именем *myShardMap*.</span><span class="sxs-lookup"><span data-stu-id="469f5-116">The following code illustrates the usage of multi-shard querying using a given **ShardMap** named *myShardMap*.</span></span> 

    using (MultiShardConnection conn = new MultiShardConnection( 
                                        myShardMap.GetShards(), 
                                        myShardConnectionString) 
          ) 
    { 
    using (MultiShardCommand cmd = conn.CreateCommand())
           { 
            cmd.CommandText = "SELECT c1, c2, c3 FROM ShardedTable"; 
            cmd.CommandType = CommandType.Text; 
            cmd.ExecutionOptions = MultiShardExecutionOptions.IncludeShardNameColumn; 
            cmd.ExecutionPolicy = MultiShardExecutionPolicy.PartialResults; 

            using (MultiShardDataReader sdr = cmd.ExecuteReader()) 
                { 
                    while (sdr.Read())
                        { 
                            var c1Field = sdr.GetString(0); 
                            var c2Field = sdr.GetFieldValue<int>(1); 
                            var c3Field = sdr.GetFieldValue<Int64>(2);
                        } 
                 } 
           } 
    } 


<span data-ttu-id="469f5-117">Основное различие состоит в построении многосегментных подключений.</span><span class="sxs-lookup"><span data-stu-id="469f5-117">A key difference is the construction of multi-shard connections.</span></span> <span data-ttu-id="469f5-118">Если **SqlConnection** действует для отдельной базы данных, то **MultiShardConnection** принимает в качестве входных данных ***коллекцию сегментов***.</span><span class="sxs-lookup"><span data-stu-id="469f5-118">Where **SqlConnection** operates on a single database, the **MultiShardConnection** takes a ***collection of shards*** as its input.</span></span> <span data-ttu-id="469f5-119">Коллекция сегментов заполняется из карты сегментов.</span><span class="sxs-lookup"><span data-stu-id="469f5-119">Populate the collection of shards from a shard map.</span></span> <span data-ttu-id="469f5-120">Затем выполняется запрос для коллекции сегментов с использованием семантики **UNION ALL** для формирования общего результата.</span><span class="sxs-lookup"><span data-stu-id="469f5-120">The query is then executed on the collection of shards using **UNION ALL** semantics to assemble a single overall result.</span></span> <span data-ttu-id="469f5-121">При необходимости к выходным данным можно добавить имя сегмента, из которого поступает строка, используя свойство **ExecutionOptions** команды.</span><span class="sxs-lookup"><span data-stu-id="469f5-121">Optionally, the name of the shard where the row originates from can be added to the output using the **ExecutionOptions** property on command.</span></span>
