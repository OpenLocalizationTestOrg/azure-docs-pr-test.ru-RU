## <a name="cluster-support"></a>Поддержка кластера
Сервер резервного копирования Azure может защитить данные в следующие кластерные приложения hello:

-   Файловые серверы

-   SQL Server

-   Hyper-V — Если вы защищаете кластер Hyper-V с помощью масштабируемой защиты DPM, нельзя добавить дополнительную защиту для рабочих нагрузок Hyper-V, защищенные hello.

    При запуске Hyper-V в Windows Server 2008 R2, убедитесь, что tooinstall hello обновление описано в КБ [975354](https://support.microsoft.com/en-us/kb/975354).
    Если Hyper-V работает в Windows Server 2008 R2 в кластерной конфигурации, необходимо установить пакет обновления 2 (SP2) и обновление, описанное в статье базы знаний [971394](https://support.microsoft.com/en-us/kb/971394).

-   Exchange Server — Azure Backup Server может защищать кластеры дисков, не являющиеся общими, для поддерживаемых версий Exchange Server (непрерывная репликация в кластере), а также может защищать сервер Exchange Server, для которого настроена локальная непрерывная репликация.

-   SQL Server — Azure Backup Server не поддерживает резервное копирование баз данных SQL Server, размещенных в общих томах кластера (CSV).

Сервер резервного копирования Azure можно защитить рабочих нагрузок кластера, расположенных в hello того же домена, сервер DPM hello, а в дочернем или доверенном домене. Если вы хотите tooprotect источники данных в недоверенных доменах или рабочих группах, используйте NTLM или сертификат для одного сервера или сертификата проверки подлинности только для кластера.
