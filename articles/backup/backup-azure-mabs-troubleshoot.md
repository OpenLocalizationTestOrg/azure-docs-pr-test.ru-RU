|Установка | При установке не удалось обновить метаданные реестра. Этот сбой при установке может привести к избыточному использованию хранилища. | Операция | Сведения об ошибке | Возможное решение |

Чтобы получить сведения о настройке, перейдите по этой ссылке: http://technet.microsoft.com/en-us/library/bb232021.aspx. <br/> Если вы не можете использовать внутреннюю ретрансляцию SMTP и вам нужно выполнить настройку с помощью сервера Office 365, можно настроить IIS в качестве ретранслятора. <br/> Для ретрансляции с SMTP на Office 365 с помощью IIS необходимо настроить сервер DPM. Дополнительные сведения см. по адресу https://technet.microsoft.com/en-us/library/aa995718(v=exchg.65).aspx. <br/> Важное примечание. На шаге 3ж (2) обязательно используйте формат user@domain.com вместо domain\user. <br/> Укажите DPM, что следует использовать имя локального сервера в качестве сервера SMTP, порт 587 и адрес электронный почты пользователя, с которого должны приходить электронные сообщения. <br/> Имя пользователя и пароль на странице настройки DPM SMTP должны быть учетной записью домена в домене, где размещен DPM. <br/> ПРИМЕЧАНИЕ. При изменении адреса SMTP-сервера измените новые параметры, закройте окно параметров, а затем снова откройте его, чтобы убедиться, что новое значение отражается.  Просто изменение и проверка не всегда задействуют новые параметры. Так что лучше всего проверять именно так. <br/> В любое время во время этого процесса можно удалить эти параметры, закрыв консоль DPM и изменив следующие разделы реестра:<br/> HKLM\SOFTWARE\Microsoft\Microsoft Data Protection Manager\Notification\ <br/> Удалите разделы SMTPPassword и SMTPUserName. <br/> Их можно добавить в пользовательском интерфейсе при его запуске.
