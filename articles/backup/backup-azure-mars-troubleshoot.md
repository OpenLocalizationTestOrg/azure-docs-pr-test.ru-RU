# <a name="troubleshoot-azure-backup-agent-configuration-and-registration-issues"></a>Устранение неполадок регистрации и настройки агента резервного копирования Azure
## <a name="recommended-steps"></a>Рекомендуемые действия
См. в следующих таблицах для устранения ошибок, которые могут возникнуть во время настройки или регистрация агента Azure Backup рекомендуемые действия.

## <a name="invalid-vault-credentials-provided-the-file-is-either-corrupted-or-does-not-have-the-latest-credentials-associated-with-recovery-service"></a>Указаны недопустимые учетные данные хранилища. Файл либо поврежден, либо не содержит последних учетных данных, связанных со службой восстановления.
| Сведения об ошибке | Возможные причины | Рекомендованные действия |
| ---     | ---     | ---    |
| **Ошибка** </br> *Указаны недопустимые учетные данные хранилища. Файл либо поврежден, либо не содержит последних учетных данных, связанных со службой восстановления. (ID: 34513)* | <ul><li> Недопустимые учетные данные хранилища (то есть, они были загружены более 48 часов, прежде чем регистрации).<li>Для загрузки во временный файл в папку Windows Temp Azure Backup Agent невозможна. <li>Учетные данные хранилища, в сетевой папке. <li>Отключен протокол TLS 1.0.<li> Настроенный прокси-сервер блокирует подключения. <br> |  <ul><li>Загрузите новые учетные данные хранилища.<li>Последовательно выберите пункты **обозревателя** > **безопасности** > **Internet**. Затем выберите **другой**и прокрутите до раздела процесс загрузки файла. Выберите **включить**.<li>Может потребоваться добавить узел в [надежные сайты](https://docs.microsoft.com/azure/backup/backup-try-azure-backup-in-10-mins#network-and-connectivity-requirements).<li>Измените параметры прокси-сервер. Введите сведения о сервере прокси-сервер. <li> Соответствует компьютере даты и времени.<li>Перейдите в C:/Windows/Temp и проверьте, есть ли более 60 000 или более 65 000 файлов с расширение TMP. Если существует, удалите эти файлы.<li>Для проверки запустите на сервере пакет SDP. Если появляется сообщение о том, что загрузки файлов не может, очень вероятно, что существует большое число файлов в каталог C:/Windows/Temp.<li>Убедитесь, что установлена платформа .NET framework 4.6.2. <li>Если TLS 1.0 была отключена из-за соответствие стандартам PCI, ссылаться на это [страницу устранения неполадок](https://support.microsoft.com/help/4022913). <li>Если у вас есть антивирусного программного обеспечения, установленного на сервере, исключите следующие файлы из антивирусного сканирования: <ul><li>CBengine.exe.<li>CSC.exe, связанных с .NET Framework. Нет CSC.exe для каждой версии .NET, которая устанавливается на сервере. Исключите файлы CSC.exe, привязанные ко всем версиям платформы .NET framework на затронутом сервере. <li>Временная папка или расположение кэша. <br>*Расположение по умолчанию для вспомогательной папки или путь к расположению кэша — C:\Program Files\Microsoft Azure Recovery Services Agent\Scratch*.

| Сведения об ошибке | Возможные причины | Рекомендованные действия |

(ID: 100050) Проверьте параметры сети и наличие подключения к Интернету*.<li>*(407) Требуется аутентификация прокси-сервера* |Прокси-сервер блокирует подключение. |  <ul><li>Перейдите в **обозревателя** > **безопасности** > **Internet**. Выберите **другой** и прокрутите до раздела процесс загрузки файла. Нажмите кнопку **Включить**.<li>Может потребоваться добавить узел в [надежные сайты](https://docs.microsoft.com/azure/backup/backup-try-azure-backup-in-10-mins#network-and-connectivity-requirements).<li>Измените параметры прокси-сервер. Введите сведения о сервере прокси-сервер. <li>Если у вас есть антивирусного программного обеспечения, установленного на сервере, исключите следующие файлы из антивирусного сканирования. <ul><li>CBEngine.exe (вместо dpmra.exe).<li>CSC.exe (связано с .NET Framework). Нет CSC.exe для каждой версии .NET, которая устанавливается на сервере. Исключите файлы CSC.exe, привязанные ко всем версиям платформы .NET framework на затронутом сервере. <li>Временная папка или расположение кэша. <br>*Расположение по умолчанию для вспомогательной папки или путь к расположению кэша — C:\Program Files\Microsoft Azure Recovery Services Agent\Scratch*.
