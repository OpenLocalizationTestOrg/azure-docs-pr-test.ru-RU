## <a name="devices-and-users"></a><span data-ttu-id="fd340-108">Устройства и пользователи</span><span class="sxs-lookup"><span data-stu-id="fd340-108">Devices and users</span></span>
<span data-ttu-id="fd340-109">Службы мобильного взаимодействия идентифицируют пользователей с помощью создания уникального идентификатора для каждого устройства.</span><span class="sxs-lookup"><span data-stu-id="fd340-109">Mobile Engagement identifies users by generating a unique identifier for each device.</span></span> <span data-ttu-id="fd340-110">Этот идентификатор называется идентификатором устройства (или `deviceid`).</span><span class="sxs-lookup"><span data-stu-id="fd340-110">This identifier is called the device identifier (or `deviceid`).</span></span> <span data-ttu-id="fd340-111">Он создается таким образом, чтобы все приложения, запущенные на одном устройстве, использовали один идентификатор устройства.</span><span class="sxs-lookup"><span data-stu-id="fd340-111">It is generated in such a way that all applications running of the same device share the same device identifier.</span></span>

<span data-ttu-id="fd340-112">Косвенно это означает, что Службы мобильного взаимодействия считают, что одно устройство принадлежит одному пользователю. Таким образом, пользователи и устройства являются эквивалентными понятиями.</span><span class="sxs-lookup"><span data-stu-id="fd340-112">Implicitly, it means that Mobile Engagement considers one device to belong to exactly one user, and thus, users and devices are equivalent concepts.</span></span>

## <a name="sessions-and-activities"></a><span data-ttu-id="fd340-113">Сеансы и действия</span><span class="sxs-lookup"><span data-stu-id="fd340-113">Sessions and activities</span></span>
<span data-ttu-id="fd340-114">Сеанс — это однократное использование приложения пользователем с момента начала использования до момента прекращения использования.</span><span class="sxs-lookup"><span data-stu-id="fd340-114">A session is one use of the application performed by a user, from the time the user starts using it, until the user stops.</span></span>

<span data-ttu-id="fd340-115">Действие — это однократное использование определенной подчасти приложения одним пользователем (обычно это экран, но может быть и что-то другое, что подходит для приложения).</span><span class="sxs-lookup"><span data-stu-id="fd340-115">An activity is one use of a given sub-part of the application performed by one user (it is usually a screen, but it can be anything suitable to the application).</span></span>

<span data-ttu-id="fd340-116">За раз пользователь может выполнить только одно действие.</span><span class="sxs-lookup"><span data-stu-id="fd340-116">A user can only perform one activity at a time.</span></span>

<span data-ttu-id="fd340-117">Действие идентифицируется по имени (ограничено 64 символами) и может при необходимости внедрить дополнительные данные (с ограничением в 1024 байт).</span><span class="sxs-lookup"><span data-stu-id="fd340-117">An activity is identified by a name (limited to 64 characters) and can optionally embed some extra data (in the limit of 1024 bytes).</span></span>

<span data-ttu-id="fd340-118">Сеансы вычисляются автоматически из последовательности действий, выполняемых пользователями.</span><span class="sxs-lookup"><span data-stu-id="fd340-118">Sessions are automatically computed from the sequence of activities performed by users.</span></span> <span data-ttu-id="fd340-119">Сеанс начинается, когда пользователь выполняет свое первое действие, и завершается, когда пользователь заканчивает свое последнее действие.</span><span class="sxs-lookup"><span data-stu-id="fd340-119">A session starts when the user starts his first activity and stops when he finishes his last activity.</span></span> <span data-ttu-id="fd340-120">Это означает, что сеансы не требуется запускать или завершать явным образом.</span><span class="sxs-lookup"><span data-stu-id="fd340-120">This means that a session does not need to be explicitly started or stopped.</span></span> <span data-ttu-id="fd340-121">Вместо этого явным образом запускаются или завершаются действия.</span><span class="sxs-lookup"><span data-stu-id="fd340-121">Instead, activities are explicitly started or stopped.</span></span> <span data-ttu-id="fd340-122">При отсутствии действий сеансы также отсутствуют.</span><span class="sxs-lookup"><span data-stu-id="fd340-122">If no activity is reported, no session is reported.</span></span>

## <a name="events"></a><span data-ttu-id="fd340-123">События</span><span class="sxs-lookup"><span data-stu-id="fd340-123">Events</span></span>
<span data-ttu-id="fd340-124">События используются для отправки отчетов о мгновенных действиях (например, о нажатии кнопки или прочитанных статьях).</span><span class="sxs-lookup"><span data-stu-id="fd340-124">Events are used to report instant actions (like button pressed or articles read by users).</span></span>

<span data-ttu-id="fd340-125">Событие может быть связано с текущим сеансом, с выполняющимся заданием или может быть изолированным.</span><span class="sxs-lookup"><span data-stu-id="fd340-125">An event can be related to the current session, to a running job, or it can be a standalone event.</span></span>

<span data-ttu-id="fd340-126">Событие идентифицируется по имени (ограничено 64 символами) и может при необходимости внедрять дополнительные данные (с ограничением в 1024 байт).</span><span class="sxs-lookup"><span data-stu-id="fd340-126">An event is identified by a name (limited to 64 characters) and can optionally embed some extra data (in the limit of 1024 bytes).</span></span>

## <a name="error"></a><span data-ttu-id="fd340-127">Ошибка</span><span class="sxs-lookup"><span data-stu-id="fd340-127">Error</span></span>
<span data-ttu-id="fd340-128">Ошибки используются для отправки отчетов о проблемах, правильно обнаруженных приложением (например, неправильное действие пользователя или сбои вызовов API).</span><span class="sxs-lookup"><span data-stu-id="fd340-128">Errors are used to report issues correctly detected by the application (like incorrect user actions, or API call failures).</span></span>

<span data-ttu-id="fd340-129">Ошибка может быть связана с текущим сеансом, с выполняющимся заданием или может быть изолированной.</span><span class="sxs-lookup"><span data-stu-id="fd340-129">An error can be related to the current session, to a running job, or it can be a standalone error.</span></span>

<span data-ttu-id="fd340-130">Ошибка идентифицируется по имени (ограничено 64 символами) и может при необходимости внедрять дополнительные данные (с ограничением в 1024 байт).</span><span class="sxs-lookup"><span data-stu-id="fd340-130">An error is identified by a name (limited to 64 characters) and can optionally embed some extra data (in the limit of 1024 bytes).</span></span>

## <a name="job"></a><span data-ttu-id="fd340-131">Задание</span><span class="sxs-lookup"><span data-stu-id="fd340-131">Job</span></span>
<span data-ttu-id="fd340-132">Задания используются для отправки отчетов о действиях, имеющих длительность (например, длительность вызовов API, время отображения рекламы, длительность фоновых задач или длительность действий пользователя).</span><span class="sxs-lookup"><span data-stu-id="fd340-132">Jobs are used to report actions having a duration (like duration of API calls, display time of ads, duration of background tasks or duration of user actions).</span></span>

<span data-ttu-id="fd340-133">Задание не связано с сеансом, так как задача может выполняться в фоновом режиме без вмешательства пользователя.</span><span class="sxs-lookup"><span data-stu-id="fd340-133">A job is not related to a session, because a task can be performed in the background, without any user interaction.</span></span>

<span data-ttu-id="fd340-134">Задание идентифицируется по имени (ограничено 64 символами) и может при необходимости внедрять дополнительные данные (с ограничением в 1024 байт).</span><span class="sxs-lookup"><span data-stu-id="fd340-134">A job is identified by a name (limited to 64 characters) and can optionally embed some extra data (in the limit of 1024 bytes).</span></span>

## <a name="crash"></a><span data-ttu-id="fd340-135">Сбой</span><span class="sxs-lookup"><span data-stu-id="fd340-135">Crash</span></span>
<span data-ttu-id="fd340-136">Пакет SDK для Служб мобильного взаимодействия автоматически отправляет отчеты о сбоях приложений, когда проблемы, не обнаруженные приложением, вызывают его сбой.</span><span class="sxs-lookup"><span data-stu-id="fd340-136">Crashes are issued automatically by the Mobile Engagement SDK to report application failures where issues not detected by the application make it crash.</span></span>

## <a name="application-information"></a><span data-ttu-id="fd340-137">Сведения о приложении</span><span class="sxs-lookup"><span data-stu-id="fd340-137">Application information</span></span>
<span data-ttu-id="fd340-138">Сведения о приложении (или информация о приложении) используются, чтобы отмечать пользователей, т. е. связывать данные с пользователями приложения (аналогично файлам cookie за исключением того, что сведения о приложении хранятся на стороне сервера на платформе Служб мобильного взаимодействия).</span><span class="sxs-lookup"><span data-stu-id="fd340-138">Application information (or app info) is used to tag users, that is, to associate some data to the users of an application (this is similar to web cookies, except that app info is stored on the server side on the Azure Mobile Engagement platform).</span></span>

<span data-ttu-id="fd340-139">Сведения о приложении можно зарегистрировать с помощью API пакета SDK для Служб мобильного взаимодействия или API устройств платформы Служб мобильного взаимодействия.</span><span class="sxs-lookup"><span data-stu-id="fd340-139">App info can be registered by using the Mobile Engagement SDK API or by using the Mobile Engagement platform Device API.</span></span>

<span data-ttu-id="fd340-140">Сведения о приложении представляют собой пару "ключ — значение", связанную с устройством.</span><span class="sxs-lookup"><span data-stu-id="fd340-140">App info is a key/value pair associated to a device.</span></span> <span data-ttu-id="fd340-141">Ключ — это имя сведений о приложении (ограничено 64 буквами ASCII [a-z, A-Z], цифрами [0-9] и символами подчеркивания [_]).</span><span class="sxs-lookup"><span data-stu-id="fd340-141">The key is the name of the app info (limited to 64 ASCII letters [a-zA-Z], numbers [0-9] and underscores [_]).</span></span> <span data-ttu-id="fd340-142">Значение (ограничено 1024 символами) может быть любой строкой, целым числом, датой (гггг-ММ-дд) или логическим значением (true или false).</span><span class="sxs-lookup"><span data-stu-id="fd340-142">The value (limited to 1024 characters) can be any string, integer, date (yyyy-MM-dd) or Boolean (true or false).</span></span>

<span data-ttu-id="fd340-143">Любое количество сведений о приложении может быть связано с устройством в рамках, определенных заданными ценами на Службы мобильного взаимодействия.</span><span class="sxs-lookup"><span data-stu-id="fd340-143">Any number of app info can be associated to a device, within the limits defined by the Mobile Engagement pricing terms.</span></span> <span data-ttu-id="fd340-144">Для данного ключа Службы мобильного взаимодействия отслеживают только набор последних значений (без журнала).</span><span class="sxs-lookup"><span data-stu-id="fd340-144">For one given key, Mobile Engagement only keeps track of the latest value set (no history).</span></span> <span data-ttu-id="fd340-145">Если значение сведений о приложении будет задано или изменено, Службы мобильного взаимодействия выполнят принудительное повторное вычисление набора критериев аудитории, указанных в этих сведениях о приложении (если они есть). Это означает, что сведения о приложении можно использовать для активации push-уведомлений в режиме реального времени.</span><span class="sxs-lookup"><span data-stu-id="fd340-145">Setting or changing the value of an app info forces Mobile Engagement to re-evaluate audience criteria set on this app info (if any) meaning that app info can be used to trigger realtime pushes.</span></span>

## <a name="extra-data"></a><span data-ttu-id="fd340-146">Дополнительные данные</span><span class="sxs-lookup"><span data-stu-id="fd340-146">Extra data</span></span>
<span data-ttu-id="fd340-147">Дополнительные данные — это некоторые произвольные данные, которые можно присоединить к событиям, ошибкам, действиям и заданиям.</span><span class="sxs-lookup"><span data-stu-id="fd340-147">Extra data (or extras) is some arbitrary data that can be attached to events, errors, activities and jobs.</span></span>

<span data-ttu-id="fd340-148">Дополнительные данные структурируются так же, как объекты JSON: они созданы из дерева пар "ключ-значение".</span><span class="sxs-lookup"><span data-stu-id="fd340-148">Extras are structured similarly to JSON objects: they are made of a tree of key/value pairs.</span></span> <span data-ttu-id="fd340-149">Ключи ограничены 64 буквами ASCII [a-z, A-Z], цифрами [0-9] и символами подчеркивания [_]), а общий размер дополнительных данных ограничен 1024 знаками (после кодировки в JSON пакетом SDK для Служб мобильного взаимодействия).</span><span class="sxs-lookup"><span data-stu-id="fd340-149">Keys are limited to 64 ASCII letters [a-zA-Z], numbers [0-9] and underscores [_]) and the total size of extras is limited to 1024 characters (once encoded in JSON by the Mobile Engagement SDK).</span></span>

<span data-ttu-id="fd340-150">Дерево пар "ключ-значение" полностью сохранено как объект JSON.</span><span class="sxs-lookup"><span data-stu-id="fd340-150">The whole tree of key/value pairs is stored as a JSON object.</span></span> <span data-ttu-id="fd340-151">Тем не менее только первый уровень пар "ключ — значение" можно разделить для прямого доступа некоторыми сложными функциями. К таким функциям, например, относятся "Сегменты" (т. е. можно легко определить сегмент "Любители научной фантастики", в который входят все пользователи, который за последний месяц не менее десяти раз отправили событие с именем content_viewed с дополнительным ключом content_type, имеющим значение scifi).</span><span class="sxs-lookup"><span data-stu-id="fd340-151">Nevertheless, only the first level of keys/values is decomposed to be directly accessible to some advanced functions like Segments (for example, you can easily define a segment called “SciFi fans” that is made of all users having sent at least 10 times the event named “content_viewed” with the extra key “content_type” set to the value “scifi” in the last month).</span></span> <span data-ttu-id="fd340-152">Таким образом, рекомендуется отправлять только дополнительные данные, созданные из простых списков пар "ключ — значение", использующих скалярные значения (например, строки, даты, целые числа или логические значения).</span><span class="sxs-lookup"><span data-stu-id="fd340-152">It is thus highly recommended to send only extras made of simple lists of key/value pairs using scalar values (for example, strings, dates, integers or Boolean).</span></span>
