# <a name="manage-endpoint-protection-issues-with-azure-security-center"></a>Управление проблемами защиты конечных точек с помощью центра безопасности Azure
Центр безопасности Azure отслеживает состояние защиты от вредоносных программ и предоставляет отчеты в колонке с проблемами защиты конечных точек. Центр безопасности выделяет проблемы, например обнаруженные угрозы и недостаточную защиту, которые могут подвергнуть ваши виртуальные машины и компьютеры угрозам от вредоносных программ. Используя информацию раздела **о проблемах защиты конечных точек**, можно определить план устранения любых обнаруженных проблем.

Центр безопасности сообщает о следующих проблемах защиты конечных точек:

- Endpoint protection not installed on Azure VMs (Защита конечных точек не установлена на виртуальных машинах Azure). Поддерживаемые антивредоносные решения не установлены на этих виртуальных машинах.
- Endpoint protection not installed on non-Azure computers (Защита конечных точек не установлена на компьютерах, не относящихся к Azure). Поддерживаемые антивредоносные программы не установлены на компьютерах, не относящихся к Azure.
- Работоспособность защиты конечных точек:

   - "Сигнатура просрочена". Антивредоносное решение установлено на этих виртуальных машинах и компьютерах, но оно не имеет последних сигнатур антивредоносных программ.
   - "Нет защиты в режиме реального времени". Антивредоносное решение установлено на этих виртуальных машинах и компьютерах, но оно не настроено для защиты в реальном времени.   Служба может быть отключена или центр безопасности не может получить сведения о состоянии, так как решение не поддерживается. Список поддерживаемых решений см. в статье [Интеграция партнеров в центре безопасности Azure](security-center-partner-integration.md).
   - "Нет отчетов". Антивредоносное решение установлено, но оно не передает данные.
   - "Неизвестно". Антивредоносное решение установлено, но его состояние неизвестно или оно уведомляет о неизвестной ошибке.

   > [!NOTE]
   > Список решений безопасности для защиты конечных точек, интегрированных в центр безопасности, см. в разделе [Integrated Azure security solutions](security-center-partner-integration.md#integrated-azure-security-solutions) (Интегрированные решения безопасности Azure).
   >
   >

## <a name="implement-the-recommendation"></a>Выполнение рекомендаций
Устранение проблем защиты конечных точек представлено в виде рекомендаций центра безопасности.  Если среда уязвима к угрозам от вредоносных программ, эти рекомендации будут отображаться в разделе **Рекомендации** и **Вычисления**. Чтобы просмотреть **панель мониторинга проблем защиты конечных точек**, необходимо выполнить рабочий процесс для вычислений.

В этом примере используется колонка **вычислений**.  Мы рассмотрим, как установить антивредоносную программу на виртуальных машинах Azure и на компьютерах, которые не относятся к Azure.

## <a name="install-antimalware-on-azure-vms"></a>Установка антивредоносных программ на виртуальных машинах Azure

1. В главном меню центра безопасности выберите **Вычисления** или **Обзор**.

   ![Выбор вычислений][1]

2. В разделе **Вычисления** выберите **Endpoint protection issues** (Проблемы защиты конечных точек). Откроется панель мониторинга **проблем защиты конечных точек**.

   ![Выбор панели мониторинга проблем защиты конечных точек][2]

   В верхней части панели мониторинга содержатся следующие сведения:

   - Installed endpoint protection providers (Поставщики защиты установленных конечных точек). Здесь представлен список различных поставщиков, определенных центром безопасности.
   - Installed endpoint protection health state (Состояние работоспособности установленных конечных точек). Здесь содержатся сведения о состоянии работоспособности виртуальных машин и компьютеров, на которых установлено решение для защиты конечных точек. На диаграмме показано количество виртуальных машин и компьютеров, которые находятся в работоспособном состоянии и с недостаточной защитой.
   - "Обнаруженные вредоносные программы". Здесь представлено количество виртуальных машин и компьютеров, на которых центр безопасности обнаружил вредоносные программы.
   - Attacked computers (Атакуемые компьютеры). Здесь содержатся данные о количестве виртуальных машин и компьютеров, на которых центр безопасности обнаружил атаки вредоносных программ.

   В нижней части панели мониторинга имеется список проблем защиты конечных точек, которые содержат следующие сведения:  

   - **Всего.** Количество виртуальных машин и компьютеров, на которые повлияла эта проблема.
   - Отрезки с агрегированным количеством виртуальных машин и компьютеров, на которые повлияла эта проблема. Цвет отрезка определяет приоритет:

      - красный цвет — высокий приоритет, проблему следует решить немедленно;
      - оранжевый цвет — средний приоритет, проблему следует решить как можно скорее.

3. Выберите **Endpoint protection not installed on Azure VMs** (Защита конечных точек не установлена на виртуальных машинах Azure).

   ![Выбор раздела Endpoint protection not installed on Azure VMs (Защита конечных точек не установлена на виртуальных машинах Azure)][3]

4. В разделе **Endpoint protection not installed on Azure VMs** (Защита конечных точек не установлена на виртуальных машинах Azure) приведен список виртуальных машин Azure, на которых не установлены антивредоносные программы.  Вы можете установить антивредоносные программы на всех виртуальных машинах в списке или на отдельной виртуальной машине, щелкнув ее.
5. В разделе **Выбор Endpoint Protection** выберите необходимое решение для защиты конечных точек. Для этого примера выберите **Антивредоносное ПО Майкрософт**.
6. Будут выведены дополнительные сведения о решении для защиты конечных точек. Нажмите кнопку **Создать**.

## <a name="install-antimalware-on-non-azure-computers"></a>Установка антивредоносных программ на компьютерах, не относящихся к Azure

1. Вернитесь к панели мониторинга **проблем защиты конечных точек** и выберите **Endpoint protection not installed on non-Azure computers** (Защита конечных точек не установлена на компьютерах, не относящихся к Azure).

   ![Выбор раздела Endpoint protection not installed on non-Azure computers (Защита конечных точек не установлена на компьютерах, не относящихся к Azure)][4]

2. В разделе **Endpoint protection not installed on non-Azure computers** (Защита конечных точек не установлена на компьютерах, не относящихся к Azure) выберите рабочую область. Откроется поисковый запрос Log Analytics, отфильтрованный по рабочей области, со списком компьютеров, на которых не установлены антивредоносные программы. Выберите компьютер из списка для получения дополнительной информации.

   ![Поиск в Log Analytics][5]

Откроется еще один результат поиска с данными только для этого компьютера.

  ![Поиск в Log Analytics][6]

> [!NOTE]
> Мы рекомендуем подготовить решения для защиты конечных точек для всех виртуальных машин и компьютеров, чтобы обнаруживать и удалять вирусы, программы-шпионы и другие вредоносные программы.
