# <a name="use-azure-security-center-recommendations-to-enhance-security"></a>Использование рекомендаций центра безопасности Azure для повышения уровня безопасности
У вас есть возможность снизить вероятность серьезных нарушений безопасности, настроив политику безопасности и применяя рекомендации, предоставляемые центром безопасности Azure. Эта статья покажет, как следует использовать политики безопасности и рекомендации центра безопасности для уменьшения рисков нарушения безопасности.

> [!NOTE]
> В этой статье используются роли и концепции, описанные в [руководстве по планированию использования центра безопасности Azure и работе в нем](security-center-planning-and-operations-guide.md). Рекомендуем еще раз ознакомиться с этим руководством, а затем вернуться к статье.
>
>

## <a name="managing-security-recommendations"></a>Управление рекомендациями по безопасности
Политика безопасности определяет набор элементов управления, которые рекомендуются для ресурсов в указанной подписке или группе ресурсов. В центре безопасности вы можете настраивать политики в соответствии с требованиями безопасности вашей компании. См. дополнительные сведения в статье [Настройка политик безопасности в центре безопасности Azure](security-center-policies.md).

Политики безопасности для групп ресурсов наследуются от уровня подписки.

![Наследование политик безопасности][1]

Если для определенных групп ресурсов нужно установить другие политики, наследование для этой группы ресурсов можно отключить. Для этого установите режим наследования "Уникальный" в колонке "Политика безопасности", а затем измените все настройки, для которых центр безопасности отображает рекомендации.

Например, если для некоторых рабочих нагрузок не нужна политика прозрачного шифрования данных базы данных SQL, отключите эту политику на уровне подписки и включайте ее индивидуально только для групп ресурсов, для которых она нужна.

> [!NOTE]
> В случае конфликта между политикой уровня подписки и политикой уровня группы ресурсов приоритет получает политика уровня группы ресурсов.
>
>

Центр безопасности анализирует состояние безопасности ваших ресурсов Azure. Когда центр безопасности выявляет потенциальные уязвимости в системе безопасности, он создает рекомендации на основании параметров, заданных в политике безопасности. Эти рекомендации помогают настраивать необходимые элементы управления безопасностью.

На данный момент рекомендации центра безопасности учитывают такие аспекты, как обновления системы, конфигурация ОС, сетевые группы безопасности подсетей и виртуальных машин, аудит базы данных SQL, прозрачное шифрование данных для базы данных SQL и брандмауэры веб-приложений. Самые свежие сведения о рекомендациях центра безопасности можно найти в статье [Управление рекомендациями по безопасности в Центре безопасности Azure](security-center-recommendations.md).

## <a name="scenario"></a>Сценарий
Этот сценарий демонстрирует, как использование центра безопасности позволяет снизить вероятность существенных нарушений безопасности путем мониторинга и выполнения рекомендаций центра безопасности. В сценарии используется вымышленная компания Contoso и роли, описанные в [руководстве по планированию использования центра безопасности Azure и работе в нем](security-center-planning-and-operations-guide.md#security-roles-and-access-controls). Роли представляют отдельных пользователей и команды, которые могут использовать центр безопасности для выполнения различных задач, связанных с безопасностью. Вот эти роли:

![Роли в сценарии][2]

Компания Contoso недавно перенесла в Azure часть своих локальных ресурсов. Она намерена внедрить и поддерживать меры защиты, позволяющие снизить уязвимость к атакам ресурсов, расположенных в облаке.

## <a name="recommended-solution"></a>Рекомендуемое решение
Для предотвращения и обнаружения уязвимостей системы безопасности следует использовать центр безопасности. Contoso имеет доступ к центру безопасности через подписку Azure. [Уровень "Бесплатный"](security-center-pricing.md) для центра безопасности автоматически включается для всех подписок Azure, а на всех виртуальных машинах в подписке активируется сбор данных.

Дэвид, специалист компании Contoso по ИТ-безопасности, настраивает **политики безопасности** с помощью центра безопасности. Центр безопасности анализирует состояние безопасности корпоративных ресурсов Azure. Когда центр безопасности выявляет потенциальные уязвимости в системе безопасности, он создает **рекомендации** на основании параметров, заданных в политике безопасности.

Джефф, владелец облачной рабочей нагрузки, отвечает за внедрение и поддержку средств защиты в соответствии с политикой безопасности компании Contoso. Джефф может отслеживать рекомендации, созданные центром безопасности, и применять защитные меры. Эти рекомендации помогают ему настраивать необходимые элементы управления безопасностью.

Чтобы Джефф мог внедрять и поддерживать средства защиты, а также устранять уязвимости, ему нужно:

- отслеживать рекомендации по безопасности в центре безопасности Azure;
- оценивать рекомендации по безопасности и выбирать, какие из них применить, а какие отклонить;
- применять рекомендации по безопасности.

Рассмотрим процесс работы Джеффа и узнаем, как он использует рекомендации центра безопасности в процессе настройки элементов управления для устранения уязвимостей системы безопасности.

## <a name="how-to-implement-this-solution"></a>Реализация решения
Джефф входит на [портал Azure](https://azure.microsoft.com/features/azure-portal/) и открывает консоль центра безопасности. В рамках ежедневного мониторинга он проверяет, есть ли новые рекомендации по безопасности, как описано далее.

1. Джефф выбирает элемент **Рекомендации**, чтобы открыть колонку **Рекомендации**.
   ![Выбор плитки рекомендаций][3]
2. Джефф просматривает список рекомендаций. Он видит, что центр безопасности упорядочивает список рекомендаций по приоритету, от наиболее важных к наименее важным. Он решает выполнить рекомендацию с высоким уровнем приоритета. В колонке **Рекомендации** он выбирает **Установить Endpoint Protection**.
3. Откроется колонка **Установить Endpoint Protection** со списком виртуальных машин, на которых не включена защита от вредоносных программ. Джефф просматривает список виртуальных машин, выбирает все виртуальные машины и нажимает кнопку **Установить на 3 виртуальных машинах**.
   ![Установка Endpoint Protection][4]
4. Откроется колонка **Выбор Endpoint Protection**, в которой Джефф видит два решения для защиты от вредоносных программ. Он выбирает **Антивредоносное ПО Майкрософт**.
5. На экран будут выведены дополнительные сведения о решении защиты от вредоносных программ. Джефф нажимает кнопку **Создать**.
   ![Антивредоносное ПО Майкрософт][5]
6. Джефф вводит необходимые параметры конфигурации в колонке **Установить** и нажимает кнопку **ОК**.

Теперь на выбранных виртуальных машинах включено [антивредоносное ПО Майкрософт](../security/azure-security-antimalware.md).

Джефф продолжает по порядку изучать рекомендации с высоким и средним приоритетом, принимая решения по их применению. Джефф открывает статью [Управление рекомендациями по безопасности в Центре безопасности Azure](security-center-recommendations.md), чтобы разобраться в рекомендациях и последствиях применения каждой из них.

Джефф узнает, что [Центр Microsoft Security Response Center](../security/azure-security-response-center.md) проводит выборочный мониторинг безопасности для сети и инфраструктуры Azure и собирает информацию об угрозах и жалобы о нарушениях от третьих лиц. Если Джефф укажет в корпоративной подписке Azure контактное лицо по вопросам безопасности, корпорация Майкрософт будет сообщать компании Contoso о любых свидетельствах незаконного или несанкционированного доступа к клиентским данным компании. Теперь рассмотрим, как Джефф выполняет рекомендацию **Предоставление сведений о контактном лице по вопросам безопасности**. В приведенном выше списке эта рекомендация имеет средний уровень приоритета.

1. Джефф выбирает элемент **Укажите сведения о контактных лицах по вопросам безопасности** в колонке **Рекомендации**, и открывается колонка **Указать сведения о контактных лицах по вопросам безопасности**.
2. Джефф выбирает подписку Azure, для которой будет предоставлять контактные данные. Откроется вторая колонка **Укажите сведения о контактных лицах по вопросам безопасности** .
   ![Контактное лицо по вопросам безопасности][6]
3. В колонке **Указать сведения о контактных лицах по вопросам безопасности** Джефф вводит:

  - адреса электронной почты для связи по вопросам безопасности, разделенные запятыми (здесь можно ввести неограниченное количество адресов электронной почты);
  - один номер телефона для связи по вопросам безопасности.

4. Чтобы получать по электронной почте оповещения с высоким уровнем серьезности, Джефф включает параметр **Send me emails about alerts**(Отправлять мне электронные сообщения об оповещениях).
5. Он нажимает кнопку **ОК**, чтобы сохранить в подписке сведения о контактных лицах по вопросам безопасности.

И наконец, Джефф переходит к рекомендации **Исправление уязвимостей ОС**, которая имеет низкий приоритет. Он решает, что это в данный момент не актуально. Эту рекомендацию он намерен отклонить. Для этого Джефф нажимает справа кнопку с тремя точками и выбирает действие **Отклонить**.
   ![Отклонение рекомендации][7]

## <a name="conclusion"></a>Заключение
Мониторинг рекомендаций в центре безопасности помогает устранить уязвимости системы безопасности раньше, чем случится атака. Вы можете предотвратить нарушения безопасности, внедряя и поддерживая средства защиты с помощью политик безопасности в центре безопасности.
