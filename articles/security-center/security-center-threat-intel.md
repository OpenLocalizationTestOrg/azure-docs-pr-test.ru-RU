# <a name="threat-intelligence-in-azure-security-center"></a>Анализ угроз в центре безопасности Azure
Эта статья поможет вам использовать анализ угроз в центре безопасности Azure для решения проблем с безопасностью.

## <a name="what-is-threat-intelligence"></a>Что такое анализ угроз?
С помощью функции "Анализ угроз", доступной в центре безопасности, ИТ-администраторы могут выявлять угрозы безопасности в среде. Например, определять, что конкретный компьютер является частью ботнета. Компьютеры могут стать узлами в ботнете, если злоумышленники незаконно установят вредоносную программу, которая тайно подключает этот компьютер к среде управления и контроля. С помощью анализа угроз можно выявить потенциальные угрозы, поступающие по нелегальным каналам связи, таким как даркнет.

Для анализа угроз центр безопасности использует данные, поступающие из нескольких источников в Майкрософт. Центр безопасности использует эти данные для выявления потенциальных угроз в вашей среде. Область **Аналитика угроз** включает три основных раздела.

- Типы обнаруженных угроз
- Источник угроз.
- Карта анализа угроз


## <a name="when-should-you-use-threat-intelligence"></a>Назначение анализа угроз
В рамках [реагирования на нарушение безопасности](https://docs.microsoft.com/azure/security-center/security-center-planning-and-operations-guide#incident-response) необходимо определить степень серьезности нарушения безопасности систем. На этом этапе следует выполнить следующие задачи:

- определить характер атаки;
- определить источник атаки;
- определить цель атаки (была ли атака направлена на вашу организацию для получения конкретных данных, или она случайна);
- определить системы, безопасность которых была нарушена;
- определить файлы, к которым был получен доступ, и степень конфиденциальности этих файлов.

Данные, полученные в результате анализа угроз в центре безопасности, позволяют выполнять эти задачи. 

## <a name="access-the-threat-intelligence"></a>Получение доступа к функции анализа угроз
Чтобы визуализировать текущие результаты анализа угроз для среды, сначала выберите рабочую область, в которой находятся ваши данные. Если у вас одна рабочая область, пропустите селектор рабочих областей и откройте непосредственно панель мониторинга **Аналитика угроз**. Чтобы получить доступ к панели мониторинга:

1. Откройте панель мониторинга **центра безопасности**.

2. В области слева в разделе **Обнаружение** выберите **Аналитика угроз**. Откроется панель мониторинга **Анализ угроз**.

    ![Панель мониторинга аналитики угроз](./media/security-center-threat-intel/security-center-threat-intel-fig1.png)

    > [!NOTE]
    > Если в столбце справа отображается **UPGRADE PLAN** (План обновления), рабочая область использует бесплатную подписку. Повысьте уровень подписки до категории "Стандартный", чтобы использовать эту функцию. Если в столбце справа отображается **REQUIRES UPDATE** (Требуется обновить), обновите [Azure Log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-overview) для использования этой функции. Дополнительные сведения о тарифном плане см. на странице с ценами на центр безопасности Azure. 
    > 
3. Если вам нужно изучить более одной рабочей области, вы можете расставить приоритеты в соответствии со столбцом **Вредоносные IP-адреса**. В нем отображается текущее число вредоносных IP-адресов в этой рабочей области. Щелкните рабочую область, которую необходимо использовать, и появится панель мониторинга **Аналитика угроз**.

    ![Сведения об анализе угроз](./media/security-center-threat-intel/security-center-threat-intel-fig5.png)

4. Эта панель мониторинга состоит из трех плиток:

    а.  **Типы угроз**. Здесь представлены типы угроз, которые были обнаружены в выбранной рабочей области.

    b.  **Страна источника**. Здесь представлен суммарный объем трафика по его источнику.

    c.  **Расположение угроз**. Помогает определить текущие расположения, в которых происходит обмен данными с вашей средой, по всему миру. На приведенной карте оранжевые (входящие) и красные (исходящие) стрелки определяют направления трафика. При выборе одной из этих стрелок отображается тип угрозы и направление трафика.

    d.  **Сведения об угрозах**. Здесь представлены дополнительные сведения об угрозе, выбранной на карте.

Независимо от выбранной плитки панель мониторинга, которая появляется, основана на запросе поиска по журналам. Единственным различием является тип запроса и результат.

### <a name="threat-types"></a>Типы угроз
Выберите плитку **Типы угроз**, чтобы открыть панель мониторинга **Поиск по журналам**. Параметры фильтра отображаются слева, а результаты запроса — справа.

![Поиск по журналам](./media/security-center-threat-intel/security-center-threat-intel-fig3.png)

В результатах запроса угрозы сгруппированы по имени. В области слева можно выбрать атрибут для фильтрации. Например, чтобы просмотреть только угрозы, которые в данный момент подключены к машинам, выберите для **Состояние сеанса** значение **Подключено** > **Применить**.

![Состояние сеанса](./media/security-center-threat-intel/security-center-threat-intel-fig4.png)

Для виртуальных машин Azure на панели мониторинга **Аналитика угроз** будут отображаться только данные сети, которые проходят через агент. Кроме того, при анализе угроз используются следующие данные:

- Данные CEF (тип = CommonSecurityLog)
- WireData (тип = WireData)
- Журналы IIS (тип = W3CIISLog)
- Брандмауэр Windows (тип = WindowsFirewall)
- События DNS (тип = DnsEvents)
