Для этого она либо никогда ничего не записывает на диск, либо шифрует находящиеся на диске программные компоненты.</span><span class="sxs-lookup"><span data-stu-id="64463-161">**Hidden malware and exploitation attempts**: Sophisticated malware is able to evade traditional antimalware products by either never writing to disk or encrypting software components stored on disk.</span></span>  <span data-ttu-id="64463-162">Тем не менее такую программу может обнаружить анализ памяти, так как, функционируя, она не может не оставлять следов в памяти.</span><span class="sxs-lookup"><span data-stu-id="64463-162">However, such malware can be detected using memory analysis, as the malware must leave traces in memory in order to function.</span></span> <span data-ttu-id="64463-163">Когда программа дает сбой, аварийный дамп записывает часть памяти в момент сбоя.</span><span class="sxs-lookup"><span data-stu-id="64463-163">When software crashes, a crash dump captures a portion of memory at the time of the crash.</span></span>  <span data-ttu-id="64463-164">Анализируя память в аварийном дампе, центр безопасности Azure может выявить методы, с помощью которых злоумышленник эксплуатирует уязвимости ПО, получает доступ к конфиденциальным данным и тайно пользуется скомпрометированным компьютером, не снижая его производительность.</span><span class="sxs-lookup"><span data-stu-id="64463-164">By analyzing the memory in the crash dump, Azure Security Center can detect techniques used to exploit vulnerabilities in software, access confidential data, and surreptitiously persist with-in a compromised machine without impacting the performance of your machine.</span></span>
* <span data-ttu-id="64463-165">**Путешествие по компьютерам и внутренняя разведка.** Чтобы находиться в скомпрометированной сети длительное время, занимаясь поиском и сбором ценных данных, злоумышленники часто пытаются переходить со скомпрометированного компьютера на другие компьютеры в той же сети.</span><span class="sxs-lookup"><span data-stu-id="64463-165">**Lateral movement and internal reconnaissance**: To persist in a compromised network and locate/harvest valuable data, attackers often attempt to move laterally from the compromised machine to others within the same network.</span></span> <span data-ttu-id="64463-166">Чтобы обнаруживать попытки злоумышленника укрепиться в сети, центр безопасности отслеживает процессы обработки и входа, например зондирование сети через удаленное выполнение команд и перечисление учетных записей.</span><span class="sxs-lookup"><span data-stu-id="64463-166">Security Center monitors process and login activities in order to discover attempts to expand an attacker’s foothold within the network, such as remote command execution network probing, and account enumeration.</span></span>
* <span data-ttu-id="64463-167">**Вредоносные сценарии PowerShell.** Для выполнения вредоносного кода на целевых виртуальных машинах злоумышленники используют PowerShell для различных целей.</span><span class="sxs-lookup"><span data-stu-id="64463-167">**Malicious PowerShell Scripts**: PowerShell is being used by attackers to execute malicious code on target virtual machines for a variety of purposes.</span></span> <span data-ttu-id="64463-168">Центр безопасности проверяет процессы PowerShell на предмет подозрительной активности.</span><span class="sxs-lookup"><span data-stu-id="64463-168">Security Center inspects PowerShell activity for evidence of suspicious activity.</span></span> 
* <span data-ttu-id="64463-169">**Исходящие атаки.** Злоумышленники часто атакуют облачные ресурсы, чтобы дополнительные атаки исходили уже от них.</span><span class="sxs-lookup"><span data-stu-id="64463-169">**Outgoing attacks**: Attackers often target cloud resources with the goal of using those resources to mount additional attacks.</span></span> <span data-ttu-id="64463-170">Скомпрометированные виртуальные машины, например, могут быть источником атак методом подбора против других виртуальных машин, могут рассылать спам или сканировать открытые порты и другие устройства в Интернете.</span><span class="sxs-lookup"><span data-stu-id="64463-170">Compromised virtual machines, for example, might be used to launch brute force attacks against other virtual machines, send SPAM, or scan open ports and other devices on the internet.</span></span> <span data-ttu-id="64463-171">Изучая сетевой трафик с помощью машинного обучения, центр безопасности может обнаруживать аномалии в исходящих сетевых подключениях.</span><span class="sxs-lookup"><span data-stu-id="64463-171">By applying machine learning to network traffic, Security Center can detect when outbound network communications exceed the norm.</span></span> <span data-ttu-id="64463-172">Что касается спама, центр безопасности сверяет трафик необычной электронной почты с аналитической информацией из Office 365, чтобы определить, является ли почта нежелательной или рассылается в рамках обычной рекламной кампании.</span><span class="sxs-lookup"><span data-stu-id="64463-172">In the case of SPAM, Security Center also correlates unusual email traffic with intelligence from Office 365 to determine whether the mail is likely nefarious or the result of a legitimate email campaign.</span></span>  

### <a name="anomaly-detection"></a><span data-ttu-id="64463-173">Обнаружение аномалий</span><span class="sxs-lookup"><span data-stu-id="64463-173">Anomaly detection</span></span>

* <span data-ttu-id="64463-178">**Входящие атаки методом подбора по протоколу RDP/SSH.** Ваши развертывания могут включать виртуальные машины с большой нагрузкой, куда пользователи входят много раз в день, и другие машины, куда входят очень мало или не входят вообще.</span><span class="sxs-lookup"><span data-stu-id="64463-178">**Inbound RDP/SSH brute force attacks**: Your deployments may have busy virtual machines with a lot of logins each day and other virtual machines that have very few or any logins.</span></span> <span data-ttu-id="64463-179">Центр безопасности Azure может определять стандартные показатели входов на этих виртуальных машинах и с помощью машинного обучения выявлять отклонения от этих показателей.</span><span class="sxs-lookup"><span data-stu-id="64463-179">Azure Security Center can determine baseline login activity for these virtual machines and use machine learning to define what is outside of normal login activity.</span></span> <span data-ttu-id="64463-180">Если количество входов, или время дня, в которое они выполняются, или расположение, из которого они запрашиваются, или какие-либо другие характеристики, связанные с входом, существенно отклоняются от стандартных показателей, система может об этом предупредить.</span><span class="sxs-lookup"><span data-stu-id="64463-180">If the number of logins, or the time of day of the logins, or the location from which the logins are requested, or other login-related characteristics are significantly different from the baseline, then an alert may be generated.</span></span> <span data-ttu-id="64463-181">Степень существенности отклонения также определяется с помощью машинного обучения.</span><span class="sxs-lookup"><span data-stu-id="64463-181">Again, machine learning determines what is significant.</span></span>
