# <a name="azure-security-center-quick-start-guide"></a>Краткое руководство по центру безопасности Azure
Эта статья поможет вам быстро начать работу с центром безопасности Azure. Она содержит инструкции по использованию функций центра безопасности для мониторинга политик безопасности и управления ими.

> [!NOTE]
> С начала июня 2017 г. в центре безопасности будет использоваться Microsoft Monitoring Agent для сбора и хранения данных. См. дополнительные сведения о [миграции платформы центра безопасности Azure](security-center-platform-migration.md). В этой статье описывается, как будет работать центр безопасности после перехода к использованию Microsoft Monitoring Agent.
>
>

## <a name="prerequisites"></a>Предварительные требования
Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной учетной записи](https://azure.microsoft.com/pricing/free-trial/).

Уровень центра безопасности "Бесплатный", который автоматически включен в подписке, позволяет отслеживать состояние безопасности для ресурсов Azure. Он предоставляет базовые возможности для управления политиками безопасности, рекомендации по безопасности и возможность интеграции с продуктами и службами безопасности, созданными партнерами Azure.

Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения о портале Azure см. в связанной [документации](https://azure.microsoft.com/documentation/services/azure-portal/).

## <a name="permissions"></a>Разрешения
В центре безопасности отображается только информация, связанная с ресурсами Azure, относительно которых вам назначена роль владельца, участника или читателя в рамках подписки или группы ресурсов, к которым принадлежит ресурс. Подробные сведения о ролях и разрешенных действиях в центре безопасности см. в статье [Permissions in Azure Security Center](security-center-permissions.md) (Разрешения в центре безопасности Azure).

## <a name="data-collection"></a>Сбор данных
Центр безопасности собирает с виртуальных машин данные, на основе которых оценивает состояние их безопасности, предоставляет рекомендации по безопасности и оповещает об угрозах. Когда вы впервые обращаетесь в центр безопасности, сбор данных включается на всех виртуальных машинах, входящих в вашу подписку. Центр безопасности подготавливает агент Microsoft Monitoring Agent к работе на всех имеющихся и создаваемых поддерживаемых виртуальных машинах Azure. Дополнительные сведения о том, как происходит сбор данных, см. в статье [Включение сбора данных](security-center-enable-data-collection.md).

Рекомендуется включить сбор данных. Если используется категория "Бесплатный" центра безопасности, сбор данных с виртуальных машин можно отключить в политике безопасности. Сбор данных является обязательным для подписок стандартного уровня в центре безопасности. Дополнительные сведения о ценовых категориях "Бесплатный" и "Стандартный" см. в статье [Цены на центр безопасности](security-center-pricing.md).

Следующие шаги описывают, как получить доступ к компонентам центра безопасности и как их использовать. Также мы рассмотрим, как отключить сбор данных, если вы не хотите использовать эту функцию.

> [!NOTE]
> В статье рассматриваются функции службы на примере развертывания. Эта статья не является пошаговым руководством.
>
>

## <a name="access-security-center"></a>Доступ к Центру безопасности
Для получения доступа к Центру безопасности выполните на портале следующие действия.

1. В меню **Microsoft Azure** выберите пункт **Центр безопасности**.

   ![Меню Azure][1]
2. При первом входе в центр безопасности открывается колонка **приветствия**. Выберите **Launch Security Center** (Запустить центр безопасности), чтобы открыть колонку **Центр безопасности** и включить сбор данных.
   ![Экран приветствия][10]
3. Когда вы запустите центр безопасности из колонки приветствия или через меню Microsoft Azure, откроется колонка **Центр безопасности**. Чтобы в следующий раз вы могли быстро открыть колонку **Центр безопасности**, выберите параметр **Закрепить колонку на панели мониторинга** (вверху справа).
   ![Параметр "Закрепить колонку на панели мониторинга"][2]

## <a name="use-security-center"></a>Использование Центра безопасности
В центре безопасности можно настроить политики безопасности для подписок Azure и групп ресурсов. Давайте настроим политику безопасности для вашей подписки.

1. Щелкните плитку **Политика** в колонке **Центр безопасности**.
2. В колонке **Политика безопасности — определить политику для подписки** выберите нужную подписку.
3. В колонке **Политика безопасности** включите параметр **Сбор данных**, чтобы разрешить автоматический сбор журналов. Это позволит подготовить расширение мониторинга для всех действующих и новых виртуальных машин в подписке. (Если используется категория "Бесплатный" центра безопасности, от сбора данных можно отказаться, задав для параметра **Сбор данных** значения **Отключено**. После установки параметра **Сбор данных** в значение **Отключено** вы не сможете получать оповещения и рекомендации из центра безопасности.)
4. В колонке **Политика безопасности** выберите **Политика предотвращения**. Откроется колонка **Политика предотвращения**.
5. В колонке **Политика предотвращения** включите рекомендации, которые вы хотите включить в политику безопасности. Примеры:

   * Если установить параметр **Обновления системы** в значение **Включено**, для всех поддерживаемых виртуальных машин будет выполняться проверка наличия неустановленных обновлений ОС.
   * Установка параметра **Уязвимости ОС** в значение **Включено** обеспечивает сканирование виртуальных машин для выявления конфигураций ОС, которые могут сделать виртуальную машину более уязвимой к атакам.

### <a name="view-recommendations"></a>Просмотр рекомендаций
1. Вернитесь в колонку **Центр безопасности** и выберите элемент **Рекомендации**. Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. Когда центр безопасности выявляет потенциальные уязвимости системы безопасности, он создает рекомендации, отображая их в колонке **Рекомендации**.
   ![Рекомендации в центре безопасности Azure][5]
2. Для просмотра дополнительной информации и (или) выполнения действий по устранению проблем выберите нужную рекомендацию в колонке **Рекомендации**.

### <a name="view-the-security-state-of-your-resources"></a>Просмотр состояния безопасности ресурсов
1. Вернитесь в колонку **Центр безопасности** . Раздел **Предотвращение** на панели мониторинга содержит индикаторы состояния безопасности для виртуальных машин, сети, данных и приложений.
2. Для получения дополнительных сведений выберите **Вычисления**. Откроется колонка **Вычисления** с тремя вкладками:

  - **Обзор** — содержит рекомендации по мониторингу и виртуальным машинам.
  - **Виртуальные машины** — содержит список всех виртуальных машин и текущее состояние безопасности.
  - **Облачные службы** — содержит список веб-ролей и рабочих ролей, отслеживаемых с помощью центра безопасности.

    ![Плитка "Работоспособность ресурсов" в Центре безопасности Azure][6]

3. На вкладке **Обзор** выберите одну из рекомендаций в разделе **VIRTUAL MACHINES RECOMMENDATIONS** (Рекомендации для виртуальной машины), чтобы просмотреть дополнительные сведения или настроить необходимые элементы управления.
4. Выберите виртуальную машину на вкладке **Виртуальные машины**, чтобы просмотреть дополнительные сведения.

### <a name="view-security-alerts"></a>Просмотр оповещений безопасности
1. Вернитесь в колонку **Центр безопасности** и щелкните плитку **Оповещения безопасности**. Откроется колонка **Оповещения безопасности**, где отображается список оповещений. Центр безопасности создает эти оповещения по итогам анализа журналов безопасности и сетевой активности. Кроме того, здесь отображаются оповещения от интегрированных партнерских решений.
   ![Оповещения безопасности в Центре безопасности Azure][7]

   > [!NOTE]
   > Предупреждения безопасности доступны, только если для центра безопасности активирован уровень "Стандартный". Доступна 60-дневная бесплатная пробная версия уровня "Стандартный". В разделе [Дальнейшие действия](#next-steps) вы найдете сведения о том, как активировать уровень "Стандартный".
   >
   >
2. Щелкните оповещение, чтобы увидеть дополнительные сведения. В нашем примере мы выберем **обнаружение измененных системных двоичных файлов**. Откроется колонка с дополнительными сведениями об этом оповещении.
   ![Сведения об оповещении безопасности в центре безопасности Azure][8]

### <a name="view-the-health-of-your-partner-solutions"></a>Просмотрите состояние партнерского решения. Для этого выполните следующее:
1. Вернитесь в колонку **Центр безопасности** . С помощью плитки **Решения партнеров** можно быстро оценить состояние работоспособности партнерских решений, интегрированных в подписку Azure.
2. Выберите элемент **Решения партнеров** . Откроется колонка со списком решений партнеров, подключенных к центру безопасности.
   ![Решения партнеров][9]
3. Выберите партнерское решение. В этом примере мы выберем решение **QualysVa1**.  Откроется колонка, в которой отображается состояние партнерского решения и связанных ресурсов. Нажмите кнопку **Консоль решений** , чтобы открыть партнерский интерфейс управления для этого решения.

## <a name="next-steps"></a>Дальнейшие действия
Из этой статьи вы узнали о мониторинге безопасности и управлении политиками в центре безопасности. Теперь, когда вы ознакомились с функциями центра безопасности, переходите к следующим действиям.

* Настройте политику безопасности для подписки Azure. См. дополнительные сведения в статье [Настройка политик безопасности в центре безопасности Azure](security-center-policies.md).
* Используйте рекомендации в центре безопасности для защиты ресурсов в Azure. Дополнительные сведения об этой функции см. в статье [Управление рекомендациями по безопасности в центре безопасности Azure](security-center-recommendations.md).
* Изучайте текущие оповещения центра безопасности и управляйте ими. Дополнительные сведения см. в статье [Управление оповещениями безопасности в центре безопасности Azure и реагирование на них](security-center-managing-and-responding-alerts.md).
- [Защита данных в центре безопасности Azure](security-center-data-security.md) — сведения об управлении данными и их защите в центре безопасности.
* Ознакомьтесь с [возможностями расширенного обнаружения угроз](security-center-detection-capabilities.md), которые включены в [уровень "Стандартный"](security-center-pricing.md) центра безопасности. Уровень "Стандартный" предоставляется бесплатно первые 60 дней.
* Если у вас есть вопросы, см. [часто задаваемые вопросы о центре безопасности Azure](security-center-faq.md).
