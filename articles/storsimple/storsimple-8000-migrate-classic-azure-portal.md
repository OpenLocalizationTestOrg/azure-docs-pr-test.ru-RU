# <a name="migrate-subscriptions-and-storage-accounts-associated-with-storsimple-device-manager-service"></a>Миграция подписок и учетных записей хранения, связанных со службой диспетчера устройств StorSimple

Вам может потребоваться переместить службу StorSimple в новую регистрацию или подписку. Эти сценарии миграции предполагают изменение учетных записей и изменения в центре обработки данных. Таблица ниже поможет понять, какие из этих сценариев поддерживаются. Также здесь приведены подробные инструкции по перемещению.

## <a name="account-changes"></a>Изменение учетных записей

| Ресурсы для миграции| Поддерживаются| Простой| Процесс поддержки Azure| Подход|
|-----|-----|-----|-----|-----|
| Вся подписка (включая учетные записи службы StorSimple и учетные записи хранилища) перемещается в другую регистрацию. | Да       | Нет       | **Перенос регистрации**<br>Используется:<li>при приобретении нового предложения Azure в рамках нового соглашения;</li><li>при миграции всех учетных записей и подписок из старой регистрации в новую. Сюда входят все службы Azure в рамках старой подписки.</li> | **Шаг 1. Откройте запрос в службу поддержки для операций Azure Enterprise:**<li>перейдите по адресу [http://aka.ms/AzureEnt](http://aka.ms/AzureEnt);</li><li> выберите **Enrollment Administration** (Администрирование регистрации), а затем — **Transfer from one enrollment to a new enrollment** (Перенос из текущей регистрации в новую).<br>**Шаг 2: Введите требуемую информацию.**<br>Укажите:<li>исходный регистрационный номер;</li><li> целевой регистрационный номер;</li><li>дату начала переноса.|
| Служба StorSimple из текущей учетной записи в новую регистрацию.    | Да       | Нет       | **Перенос учетной записи**<br>Используйте следующую команду:<li>если не требуется полный перенос регистрации;</li><li>если в новую регистрацию нужно переместить только определенные учетные записи.</li>| **Шаг 1. Откройте запрос в службу поддержки для операций Azure Enterprise:**<li>перейдите по адресу [http://aka.ms/AzureEnt](http://aka.ms/AzureEnt);</li><li>выберите **Enrollment Administration** (Администрирование регистрации), а затем — **Transfer an EA Account to a new enrollment** (Перенос учетной записи EA в новую регистрацию).<br>**Шаг 2: Введите требуемую информацию.**<br>Укажите:<li>исходный регистрационный номер;</li><li> целевой регистрационный номер;</li><li>дату начала переноса.|
| Служба StorSimple из одной подписки в другую.      | Нет        |    Да         | Поддержка не предусмотрена, этот процесс выполняется вручную|<li>Перенесите данные с устройства StorSimple.</li><li>Выполните для устройства сброс до параметров по умолчанию. При этом все локальные данные на устройстве будут удалены.</li><li>Зарегистрируйте устройство с новой подпиской в службе диспетчера устройств StorSimple.</li><li>Верните данные на устройство.|
| Устройство StorSimple из одной службы диспетчера устройств StorSimple в другую службу в другом регионе.      | Нет        | Да            | Поддержка не предусмотрена, этот процесс выполняется вручную |То же, что и выше.|
| Учетная запись хранения в новую подписку или группу ресурсов     | Да        | Нет             |Переместите учетную запись хранения в новую подписку или группу ресурсов |После перемещения, если ключи доступа к учетной записи хранения обновлены, пользователю потребуется настроить ключи доступа вручную для перенесенной учетной записи хранения с помощью службы диспетчера устройств StorSimple.|
| Классическая учетная запись хранения в учетную запись хранения Azure Resource Manager      | Да        | Нет             |Перейдите с классической модели развертывания на модель Azure Resource Manager |<li>Подробные инструкции по переносу учетной записи хранения с классической модели развертывания на модель Azure Resource Manager см. в статье [Перенос классической учетной записи хранения](../virtual-machines/windows/migration-classic-resource-manager-ps.md#step-62-migrate-a-storage-account).</li><li> Если ключи доступа учетной записи хранения обновлены после переноса, пользователю нужно будет синхронизировать ключи доступа для перенесенной учетной записи с помощью службы диспетчера устройств StorSimple. Это обеспечит нормальную работу устройств StorSimple и способность распределять основные и резервные данные по категориям в Azure. Подробные инструкции по синхронизации ключей доступа, см. в статье [Rotation workflow](storsimple-8000-manage-storage-accounts.md#key-rotation-of-storage-accounts) (Рабочий процесс смены ключей).</li><li> В случае облачного устройства StorSimple, если классическая учетная запись хранения перенесена, но базовая виртуальная машина по-прежнему остается в классической модели, устройство должно функционировать надлежащим образом. Если базовая виртуальная машина для облачного устройства переносится, функции отключения и удаления не будут работать.</li><li> Необходимо создать новые облачные устройства StorSimple на портале Azure и затем выполнить отработку отказа со старых облачных устройств. Создать облачное устройство StorSimple на новом портале Azure с помощью классической учетной записи хранения нельзя. Для этого потребуется учетная запись хранилища Azure Resource Manager. Дополнительные сведения см. в статье [Развертывание и администрирование облачного устройства StorSimple в Azure (обновление 3 и более поздние версии)](storsimple-8000-cloud-appliance-u2.md).</li>|То же, что и выше.|
