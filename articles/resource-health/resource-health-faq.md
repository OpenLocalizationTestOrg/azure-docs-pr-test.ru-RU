## <a name="what-is-azure-resource-health"></a>Что такое служба работоспособности ресурсов Azure?
Служба работоспособности ресурсов позволяет выполнять диагностику и получать поддержку, если проблема Azure влияет на ресурсы. Она представляет сведения о текущем состоянии работоспособности ресурсов и о состоянии работоспособности ресурсов за прошедший период, а также помогает устранить проблемы. Служба работоспособности ресурсов обеспечивает поддержку, если вам необходима помощь в решении проблемы со службой Azure.  

## <a name="what-is-the-resource-health-intended-for"></a>Для чего предназначена служба работоспособности ресурсов?
При обнаружении проблемы с ресурсом служба работоспособности ресурсов позволяет определить основную причину. Она предоставляет справку по устранению проблем и обеспечивает поддержку, если вам необходима дополнительная помощь в решении проблемы со службой Azure.

## <a name="what-health-checks-are-performed-by-resource-health"></a>Какие проверки выполняет служба работоспособности ресурсов?
Служба работоспособности ресурсов выполняет различные проверки на основе [типа ресурса](resource-health-checks-resource-types.md). Эти проверки предназначены для решения трех типов проблем: 
1. Незапланированные события, например непредвиденная перезагрузка узла.
2. Запланированные события, например запланированные обновления ОС узла.
3. События, вызванные действиями пользователя, например перезагрузка виртуальной машины.

## <a name="what-does-each-of-the-health-status-mean"></a>Что означает состояние работоспособности?
Существует три разных состояния работоспособности:
1. Доступно. На платформе Azure отсутствуют известные проблемы, которые могут повлиять на этот ресурс.
2. Недоступно. Служба работоспособности ресурсов обнаружила проблемы, влияющие на ресурс.
3. Неизвестно. Служба работоспособности ресурсов не может определить работоспособность ресурса, так как она перестала получать сведения о нем. 

## <a name="what-does-the-unknown-status-mean-is-something-wrong-with-my-resource"></a>Что означает неизвестное состояние? С ресурсом возникла проблема?
Когда служба работоспособности ресурсов прекращает получать сведения о конкретном ресурсе, работоспособность будет переведена в состояние "Неизвестно". Хотя это состояние не является основным обозначением состояния ресурса, в случае возникновения проблем оно может указать на проблемы, возникающие в Azure.

## <a name="how-can-i-get-help-for-a-resource-that-is-unavailable"></a>Как получить справку по недоступному ресурсу?
Вы можете отправить запрос на поддержку в колонке работоспособности ресурсов. Вам не нужно соглашение о поддержке Майкрософт, чтобы открыть запрос, когда ресурс недоступен из-за событий платформы.

## <a name="does-resource-health-differentiate-between-unavailability-cased-by-platform-problems-versus-something-i-did"></a>Различает ли служба работоспособности ресурсов недоступность из-за проблем с платформой и из-за действий пользователя?
Да, если ресурс недоступен, служба работоспособности ресурсов указывает основную причину в одной из этих категорий: 
1.  Инициируемое пользователем действие.
2.  Запланированное событие. 
3.  Незапланированное событие.

На портале инициированные пользователем действия выделяются с помощью синего значка уведомления, а запланированные и незапланированные события — с помощью красного значка предупреждения. Дополнительные сведения см. в статье [Обзор службы работоспособности ресурсов Azure](Resource-health-overview.md).  

## <a name="can-i-integrate-resource-health-with-my-monitoring-tools"></a>Можно ли интегрировать службу работоспособности ресурсов в средства мониторинга?
Работоспособность ресурсов — это служба, предназначенная для диагностики и устранения проблем, влияющих на ресурсы, в службе Azure. Хотя вы можете использовать API работоспособности ресурсов для программного получения сведений о состоянии работоспособности, мы советуем использовать метрики для мониторинга ресурсов. После обнаружения проблем служба работоспособности ресурсов поможет определить основную причину, а также выполнить действия по их устранению. Перейдите на страницу [Azure Monitor](https://docs.microsoft.com/azure/monitoring-and-diagnostics/), чтобы узнать больше об использовании метрик для проверки ресурсов.

## <a name="where-do-i-find-resource-health"></a>Где найти службу работоспособности ресурсов?
После входа на портал Azure доступ к службе работоспособности ресурсов можно получить несколькими способами:
1. Перейдите к нужному ресурсу. На панели навигации слева щелкните **Работоспособность ресурсов**.
2. Перейдите к колонке Azure Monitor.  На панели навигации слева щелкните **Работоспособность ресурсов**.
3. Чтобы открыть колонку **Справка и поддержка**, щелкните вопросительный знак в правом верхнем углу портала и выберите **Справка и поддержка**. Открыв колонку, щелкните **Работоспособность ресурсов**.

API работоспособности ресурсов также можно использовать для получения сведений о состоянии работоспособности ресурсов.

## <a name="is-resource-health-available-for-all-resource-types"></a>Доступна ли служба работоспособности для всех типов ресурсов?
Список проверок работоспособности и типов ресурсов, поддерживаемых в службе работоспособности ресурсов, можно найти [здесь](resource-health-checks-resource-types.md).

## <a name="what-should-i-do-if-my-resource-is-showing-available-but-i-believe-it-is-not"></a>Что делать, если ресурс отображается как доступный, но на самом деле это не так?
При проверке работоспособности ресурса справа в разделе состояния работоспособности можно щелкнуть **Report incorrect health status** (Сообщить о неправильном состоянии работоспособности). Перед отправкой отчета вы можете указать дополнительные сведения о том, почему текущее состояние работоспособности является неправильным.

## <a name="is-resource-health-available-for-all-azure-regions"></a>Доступна ли служба работоспособности ресурсов для всех регионов Azure? 
Служба работоспособности ресурсов доступна во всех регионах Azure, за исключением следующих:
* Правительство штата Вирджиния
* Правительство штата Айова
* Восточная часть US DoD
* Центральная часть US DoD
* Центральная Германия
* Северо-восточная Германия
* Восток Китая
* Север Китая

## <a name="how-is-resource-health-different-from-the-service-health-dashboard-or-the-azure-portal-service-notifications"></a>Чем служба работоспособности ресурсов отличается от панели мониторинга работоспособности службы или уведомлений службы на портале Azure?
Служба работоспособности ресурсов предоставляет более конкретные сведения, чем панель мониторинга работоспособности службы Azure.

[Состояние Azure](https://status.azure.com) и уведомления службы портала информируют о проблемах в работе службы, влияющих на множество клиентов (например, регион Azure), а служба работоспособности ресурсов предоставляет более подробные события, относящиеся только к конкретному ресурсу. Например, если узел неожиданно перезагружается, служба работоспособности ресурсов оповещает только тех клиентов, чьи виртуальные машины выполнялись на этом узле.

Важно отметить, что для обеспечения полного представления о событиях, влияющих на ресурсы, служба работоспособности ресурсов также сообщает о событиях, опубликованных в уведомлениях службы и на панели мониторинга работоспособности службы.

## <a name="do-i-need-to-activate-resource-health-for-each-resource"></a>Нужно ли активировать службу работоспособности ресурсов для каждого ресурса?
Нет, сведения о работоспособности доступны для всех типов ресурсов в службе работоспособности ресурсов. 

## <a name="do-we-need-to-enable-resource-health-for-my-organization"></a>Нужно ли включать службу работоспособности ресурсов для организации?
Нет.  Служба работоспособности ресурсов Azure доступна на портале Azure без каких-либо требований к настройке.

## <a name="is-resource-health-available-free-of-charge"></a>Предоставляется ли служба работоспособности ресурсов бесплатно?
Да.  Эта служба предоставляется бесплатно.

## <a name="what-are-the-recommendations-that-resource-health-provides"></a>Какие рекомендации предоставляет служба работоспособности ресурсов?
В зависимости от состояния работоспособности служба работоспособности ресурсов предоставляет рекомендации, позволяющие максимально ускорить устранение неполадок. Рекомендации по доступным ресурсам направлены, прежде всего, на устранение наиболее распространенных проблем клиентов. Если ресурс недоступен из-за незапланированного события Azure, внимание будет уделяться в первую очередь оказанию помощи во время и после завершения процесса восстановления.
