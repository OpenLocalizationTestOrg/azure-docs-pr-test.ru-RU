# <a name="verify-throughput-and-latency-metrics-for-a-storage-account"></a>Проверка пропускной способности и задержки показателей для учетной записи хранения

Это руководство является четвертой и последней частью цикла. В предыдущих занятий вы узнали, как отправка и загрузка larges объемов случайные данные для учетной записи хранилища Azure. Этот учебник показывает, как метрики можно использовать для просмотра пропускной способности и задержке на портале Azure.

В четвертой части цикла вы узнаете, как выполнять такие задачи:

> [!div class="checklist"]
> * Настройка диаграммы на портале Azure
> * Проверка метрик пропускной способности и задержки

[Метрики хранилища Azure](../common/storage-metrics-in-azure-monitor.md?toc=%2fazure%2fstorage%2fblobs%2ftoc.json) использует монитор Azure для предоставления унифицированного представления о производительности и доступности учетной записи.

## <a name="configure-metrics"></a>Настройка метрик

Перейдите к **метрики (Предварительная версия)** под **параметры** вашей учетной записи хранилища.

Выберите BLOB-объектов из **службы SUB** раскрывающегося списка.

В разделе **МЕТРИКА**, выберите один из метрик, найти в следующей таблице:

Следующие показатели помогут оценить задержку и пропускную способность приложения. Метрики, которые настраиваются на портале находятся в 1 минуту средние значения. Если транзакция завершена середине минуты минуты данных является halfed для среднего. В приложении были истекло время ожидания операции отправки и загрузки и предоставляемых вывод фактическое количество времени, потребовавшегося отправка и загрузка файлов. Эти сведения можно использовать в сочетании с портала метрики для полного понимания пропускной способности.

|Метрика|Определение|
|---|---|
|**Задержка E2E успех**|Средняя совокупная задержка успешных запросов к службе хранения или указанной операции API. Это значение включает необходимое время обработки в пределах службы хранилища Azure для считывания запроса, отправки ответа и получения подтверждения ответа.|
|**Задержка успех сервера**|Среднее время, используемое для обработки успешного запроса службы хранилища Azure. Это значение не включает в себя сетевую задержку, указанную в метрике SuccessE2ELatency. |
|**Транзакции**|Количество запросов к службе хранения или указанной операции API. Это число включает успешные и неудачные запросы, а также запросы, вызвавшие ошибку. В этом примере размер блока установлено в 100 МБ. В этом случае каждый блок 100 МБ, считается транзакцией.|
|**Входящих сообщений**|Объем входящих данных. Это количество включает входящий трафик из внешнего клиента в службу хранилища Azure, а также входящий трафик в пределах Azure. |
|**Исходящие данные**|Объем исходящих данных. Это количество включает исходящий трафик из внешнего клиента в службу хранилища Azure, а также исходящий трафик в пределах Azure. В результате это количество не отражает исходящий трафик для тарификации. |

Выберите **последние 24 часа (автоматически)** рядом с **время**. Выберите **последний час** и **минуту** для **степень детализации времени**, нажмите кнопку **применить**.

![Метрики учетной записи хранения](./media/storage-blob-scalable-app-verify-metrics/figure1.png)

Диаграмма может содержать более одной метрики, назначенные им, но назначение более одной метрики отключает возможность группирования по измерениям.

## <a name="dimensions"></a>Измерения

Различные метрики имеют разные размерности. Одно измерение, которое доступно **имя API** измерения. Это измерение прерывается диаграмму в каждый отдельный вызов API.
