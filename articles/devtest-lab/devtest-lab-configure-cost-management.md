# <a name="view-the-monthly-estimated-lab-cost-trend-in-azure-devtest-labs"></a>Просмотр помесячной тенденции изменения оценочной стоимости в Azure DevTest Labs
Функция управления затратами в лаборатории для разработки и тестирования помогает отслеживать расходы на лабораторию. В этой статье демонстрируется, как использовать диаграмму **Помесячная тенденция изменения оценочной стоимости** для просмотра оценочной стоимости на данный момент для текущего календарного месяца, а также прогнозируемой стоимости на конец текущего календарного месяца. В этой статье также показано, как для лучшего управления расходов лаборатории, задав оплаты целевые и пороговые значения, при достижении триггер DevTest Labs результатов для вас.

## <a name="viewing-the-monthly-estimated-cost-trend-chart"></a>Просмотр диаграммы "Помесячная тенденция изменения расчетной стоимости"
Для просмотра диаграммы "Помесячная тенденция изменения оценочной стоимости" выполните указанные ниже действия. 

1. Войдите на [портале Azure](http://go.microsoft.com/fwlink/p/?LinkID=525040).
1. При необходимости щелкните **Все службы**, а затем выберите в списке **DevTest Labs**. Возможно, лаборатория уже отображается на панели мониторинга в разделе **Все ресурсы**.
1. Из списка лабораторий выберите нужную лабораторию.  
1. В области лаборатории **Обзор** выберите **Configuration and policies** (Конфигурация и политики).   
1. В левой части экрана, в разделе **ОТСЛЕЖИВАНИЯ затрат**выберите **тенденции затрат**.

   На снимке экрана ниже показан пример диаграммы стоимости. 
   
    ![Диаграмма стоимости](./media/devtest-lab-configure-cost-management/graph.png)

Значение **Оценочная стоимость** — это стоимость на данный момент в текущем календарном месяце. А значение **Плановые затраты** — это оценочная стоимость за весь текущий календарный месяц, рассчитанная исходя из затрат лаборатории за предыдущие пять дней.

Обратите внимание, что суммы затрат округляются до ближайшего целого числа. Например:  

* 5.01 округляется до 6 
* 5.50 округляется до 6
* 5.99 округляется до 6

Как говорится в над диаграммой, затраты, см в диаграмме по умолчанию — *предполагаемое* расходы с помощью [Оплата по мере использования](https://azure.microsoft.com/offers/ms-azr-0003p/) предлагаются скидки. Можно также задать собственные тратит целевых объектов, отображаемых на графиках, [управление издержек для лаборатории.](#managing-cost-targets-for-your-lab)

Кроме того, при расчете стоимости *не* учитываются следующие данные:

* подписки CSP и Dreamspark в настоящее время не поддерживаются, так как для расчета затрат лаборатории Azure DevTest Labs использует [интерфейсы API Azure для выставления счетов](../billing/billing-usage-rate-card-overview.md) , а они не поддерживают указанные подписки;
* индивидуальные тарифы — В настоящее время нельзя использовать ставок предложения (показаны в рамках подписки), согласованные с Майкрософт или партнерами. Используются только ставок с оплатой по мере использования.
* налоги;
* скидки;
* валюта выставления счетов. В настоящее время затраты на лабораторию приводятся только в долларах США.

## <a name="managing-cost-targets-for-your-lab"></a>Управление издержек для лаборатории
DevTest Labs позволяет лучше управлять затраты в лаборатории, задав оплаты целевой объект, который затем можно просмотреть на диаграмме ежемесячные тренда Оценка затрат. DevTest Labs можно также отправить вам уведомления при достижении порогового значения или тратит указанный целевой объект. 

1. В состав лаборатории **Обзор** выберите **конфигурации и политики**.
1. В левой части экрана, в разделе **ОТСЛЕЖИВАНИЯ затрат**выберите **тенденции затрат**.

    ![Управление целевой кнопки](./media/devtest-lab-configure-cost-management/cost-trend.png)

1. В **тенденции затрат** выберите **управление целевой**.

    ![Управление целевой кнопки](./media/devtest-lab-configure-cost-management/cost-trend-manage-target.png)

1. В целевой области Управление укажите желаемый расходов и пороговых значений. Можно также задать ли каждый выбранный порог сообщается, на диаграмме тенденции затрат или с помощью веб-перехватчика уведомление.

    ![Управление целевой области](./media/devtest-lab-configure-cost-management/cost-trend-manage-target-pane.png)

   - Выберите период времени, когда должна отслеживаться издержек.
      - **Ежемесячные**: издержек отслеживаются для каждого месяца.
      - **Фиксированный**: издержек отслеживаются для диапазона дат, укажите дату начала и окончания. Как правило это соответствуют продолжительность проекта запланировано.
   - Укажите **целевой стоимость**. Например это может быть, насколько вы планируете тратят на выполнение этой лаборатории в период времени, которые определены.
   - Выберите, чтобы включить или отключить любое пороговое значение требуется сообщил — с шагом в 25% — до 125% на указанный **целевой стоимость**.
      - **Уведомить**: по достижении этого порогового значения, при этом появится указать URL-адрес веб-перехватчика.
      - **Размещения на диаграмме**: по достижении этого порогового значения, результаты отображаются на графе тенденции затрат, которые можно просмотреть, как описано в [Просмотр диаграммы ежемесячные тенденций Предполагаемая стоимость](#viewing-the-monthly-estimated-cost-trend-chart).
   - Если вы решили **уведомления** по достижении порогового значения, необходимо указать URL-адрес веб-перехватчика. Выберите в области интеграции стоимость **щелкните здесь, чтобы добавить интеграцию**.

      Введите URL-адрес веб-перехватчик в области уведомлений Настройка, а затем выберите **ОК**.

       ![Настроить панель уведомлений](./media/devtest-lab-configure-cost-management/configure-notification.png)

      - При указании **уведомление**, необходимо определить URL-адрес веб-перехватчика.
      - Аналогично, если определить URL-адрес веб-перехватчика, необходимо задать **уведомления** для **на** в области пороговое значение стоимости.
      - Необходимо создать веб-перехватчика, прежде чем входить его здесь.  

      Дополнительные сведения об объектах webhook см. в статье [Создание функции Azure объекта webhook или API](../azure-functions/functions-create-a-web-hook-or-api-function.md).
