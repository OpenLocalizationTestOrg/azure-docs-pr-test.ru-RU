# <a name="service-limits-in-azure-search"></a>Ограничения поиска Azure
Максимальные ограничения хранилища, рабочих нагрузок, количества индексов, документов и других объектов зависят от того, на каком уровне [вы подготовили службу поиска Azure для работы](search-create-service-portal.md): **Бесплатный**, **Базовый** или **Стандартный**.

* **Бесплатный** представляет собой мультитенантную службу, которая предоставляется вместе с подпиской Azure. Это параметр дополнительных бесплатно для существующих подписчиков, который позволяет tooexperiment со службой hello перед регистрацией для выделенных ресурсов.
* Уровень **Базовый** предоставляет выделенные вычислительные ресурсы для небольших рабочих нагрузок в рабочей среде.
* Для уровня **Стандартный** используются выделенные виртуальные машины. Это обеспечивает увеличение емкости хранилища и вычислительной мощности на всех уровнях. В категории "Стандартный" доступно четыре уровня: S1, S2, S3 и S3, высокая плотность (S3 HD).

> [!NOTE]
> Служба подготавливается на определенном уровне. Если вам требуется больше емкости tooget уровни toojump, необходимо подготовить новую службу (имеется не обновление на месте). Дополнительные сведения см. в статье [Выбор SKU или ценовой категории для службы поиска Azure](search-sku-tier.md). Дополнительные сведения о настройке емкость в службе toolearn вы уже подготовленных см. в разделе [масштабировать уровни ресурсов для запросов и индексирования рабочих нагрузок](search-capacity-planning.md).
