<span data-ttu-id="962db-105">Cloudyn, партнер по разработке корпорации Майкрософт и ведущий поставщик облачных возможностей управления, был выбран для закрытого предварительного ознакомления с новыми API использования ресурсов и RateCard в Microsoft Azure.</span><span class="sxs-lookup"><span data-stu-id="962db-105">Cloudyn, a Microsoft development partner and a leading provider of cloud management capabilities, was chosen for a private preview of the new Microsoft Azure Resource Usage and RateCard APIs.</span></span>  <span data-ttu-id="962db-106">API использования предоставляет доступ к оценочным данным о потреблении в Azure.</span><span class="sxs-lookup"><span data-stu-id="962db-106">The Usage API provides access to estimated Azure consumption data for a subscription.</span></span> <span data-ttu-id="962db-107">API RateCard предоставляет полные сведения о ценах на все службы Azure для клиентов без Соглашения Enterprise.</span><span class="sxs-lookup"><span data-stu-id="962db-107">The RateCard API provides complete pricing information of all Azure services, for non-Enterprise Agreement (EA) customers.</span></span> <span data-ttu-id="962db-108">Вместе эти API обеспечивают полноценную платформу для ввода данных в ИТ-средства финансового управления (IT Financial Management — ITFM), такие как Cloudyn.</span><span class="sxs-lookup"><span data-stu-id="962db-108">Integrated together, these APIs provide a complete information basis for input into IT Financial Management (ITFM) tools such as those provided by Cloudyn.</span></span>

## <a name="introduction"></a><span data-ttu-id="962db-109">Введение</span><span class="sxs-lookup"><span data-stu-id="962db-109">Introduction</span></span>
<span data-ttu-id="962db-110">Так называемое умножение данных из API использования на данные из API RateCard (использование [единицы] цена [цена за единицу] = подробные сведения об использовании и стоимости) позволяет получить наиболее детальную, точную и надежную информацию о выставлении счетов, доступную для Azure в настоящее время.</span><span class="sxs-lookup"><span data-stu-id="962db-110">The so-called “multiplication” of data from the Usage API with data from the RateCard API (usage [units] price[$unit] = Detailed Usage and Cost) creates the most granular, accurate and reliable billing information available for Azure today.</span></span>

![Обзор ITFM][1]

<span data-ttu-id="962db-112">Применение этих API предоставляет ключевую информацию по уровню использования и расходов клиентов, позволяя Cloudyn анализировать учетные записи клиентов простым программным образом и выполнять различные ITFM-задачи для своих клиентов.</span><span class="sxs-lookup"><span data-stu-id="962db-112">Consuming these APIs provides key information on customers’ usage and costs, allowing Cloudyn to analyze customer accounts in a simple, programmatic manner, and to perform various ITFM tasks for its customers.</span></span>

## <a name="integrating-cloudyn-with-the-ratecard-and-usage-apis"></a><span data-ttu-id="962db-113">Интеграция Cloudyn с API RateCard и использования</span><span class="sxs-lookup"><span data-stu-id="962db-113">Integrating Cloudyn with the RateCard and Usage APIs</span></span>
<span data-ttu-id="962db-114">API RateCard требует несколько входных параметров, таких как сведения о регионе, валюта и языковой стандарт, однако самым важным является OfferDurableID, который указывает тип предложения Azure, используемого клиентом (оплата по мере использования, планов обязательств по прежним версиям на 6 и 12 месяцев, предложения MSDN, предложения MPN, рекламные предложения и другие).</span><span class="sxs-lookup"><span data-stu-id="962db-114">The RateCard API requires several input parameters -- like region info, currency and locale -- but the most important one is OfferDurableID, which specifies the type of Azure offering the customer is using (Pay-as-you-Go, legacy 6 and 12-month commitment plans, MSDN offers, MPN offers, promotional offers and others).</span></span> <span data-ttu-id="962db-115">OfferDurableID можно найти в области идентификатора предложения [портала использования и выставления счетов Azure](https://account.windowsazure.com/Subscriptions)для заданной подписки.</span><span class="sxs-lookup"><span data-stu-id="962db-115">The OfferDurableID can be found in the [Azure Usage and Billing portal](https://account.windowsazure.com/Subscriptions), under the "Offer ID" for the given subscription.</span></span>

<span data-ttu-id="962db-116">После регистрации в службах [Cloudyn для Azure](https://www.cloudyn.com/microsoft-azure/) клиенты могут добавить свой код OfferDurableID, который позволяет Cloudyn получить сведения о соответствующих ценах через API RateCard.</span><span class="sxs-lookup"><span data-stu-id="962db-116">Upon registration for [Cloudyn for Azure](https://www.cloudyn.com/microsoft-azure/) services, customers can add their OfferDurableID code, which allows Cloudyn to pull their relevant pricing information through the RateCard API.</span></span>  <span data-ttu-id="962db-117">Сведения о различных типах предложений можно найти на странице [Microsoft Azure Offer Details](https://azure.microsoft.com/support/legal/offer-details/) (Сведения о предложении Microsoft Azure).</span><span class="sxs-lookup"><span data-stu-id="962db-117">Information on the different types of offers can be found one the [Microsoft Azure Offer Details](https://azure.microsoft.com/support/legal/offer-details/) page.</span></span>

![Обзор подсистемы ITFM Cloudyn][2]

<span data-ttu-id="962db-119">Cloudyn применяет как API использования, так и API RateCard в дополнение к API производительности Azure, чтобы создать дополнительные слои визуализации, аналитики, выдачи оповещений, ведения отчетов, управления стоимостью и практических рекомендаций, составляющих надежное облачное ITFM-средство для клиентов Azure.</span><span class="sxs-lookup"><span data-stu-id="962db-119">Cloudyn uses both the Usage and RateCard APIs, in addition to the Azure Performance API, to create additional layers of visualization, analytics, alerting, reporting, cost management and actionable recommendations, providing Azure customers a reliable enterprise cloud ITFM tool.</span></span>

## <a name="cloudyn-itfm-use-cases-enabled-by-usage-and-ratecard-api-integration"></a><span data-ttu-id="962db-120">Варианты использования ITFM Cloudyn, возможные благодаря интеграции API использования и RateCard</span><span class="sxs-lookup"><span data-stu-id="962db-120">Cloudyn ITFM use cases enabled by Usage and RateCard API integration</span></span>
<span data-ttu-id="962db-121">К наиболее распространенным вариантам использования ITFM Cloudyn, возможным благодаря интеграции API использования и RateCard, относятся следующие:</span><span class="sxs-lookup"><span data-stu-id="962db-121">Common Cloudyn ITFM use cases enabled by usage and RateCard APIs include:</span></span>

* <span data-ttu-id="962db-122">**Анализ затрат** позволяет разделить затраты на облако по любому собственному определяющему измерению (поставщик, служба, учетная запись, регион и т. д.).</span><span class="sxs-lookup"><span data-stu-id="962db-122">**Cost Analysis** - Allows cloud costs to be broken down to any native identifying dimension (provider, service, account, region etc.).</span></span> <span data-ttu-id="962db-123">API использования и RateCard Azure позволяют легко справиться с этой задачей, предоставляя наиболее детальную декомпозицию данных об использовании и затратах для каждой учетной записи, которые затем группируются и фильтруются системой Cloudyn и выводятся пользователю в графической или табличной форме.</span><span class="sxs-lookup"><span data-stu-id="962db-123">The Azure Usage and RateCard APIs make this an easy task, by providing the most granular breakdown of usage and cost data per account, which is then grouped and filtered by Cloudyn and presented to the user, in a graphic or tabular form.</span></span>

![Круговая диаграмма анализа затрат][3]

* <span data-ttu-id="962db-125">**Комплексное распределение затрат** позволяет финансовым и ИТ-руководителям получить представление о фактической разбивке расходов, стимулирующих факторах и тенденциях в облачном развертывании.</span><span class="sxs-lookup"><span data-stu-id="962db-125">**Cost Allocation 360** - Enables finance and IT managers to uncover the actual cost breakdown, drivers and trends of their cloud deployment.</span></span> <span data-ttu-id="962db-126">Он также позволяет руководителям легче сопоставлять расходы на развертывание с филиалами, подразделениями, регионами и т. п., предоставляя беспрецедентные сведения о расходах на облако и стимулируя использование возвратных платежей и виртуальных счетов на предприятии.</span><span class="sxs-lookup"><span data-stu-id="962db-126">It further allows managers to easily associate deployment expenses with business units, departments, regions, and more, providing unprecedented insights into cloud costs, and facilitating enterprise chargebacks and showbacks.</span></span> <span data-ttu-id="962db-127">API использования и RateCard Azure служат в качестве входных данных для подсистемы распределения затрат в Cloudyn, которая дополняет API, определяя методы и бизнес-логику для распределения непомеченных или непомечаемых ресурсов.</span><span class="sxs-lookup"><span data-stu-id="962db-127">The Azure Usage and RateCard APIs serve as input to Cloudyn’s cost allocation engine, which complements the APIs by defining methods and business logic for allocating untagged or untaggable resources.</span></span>

![Диаграмма комплексного распределения затрат][4]

* <span data-ttu-id="962db-129">**Cost-Effective Sizing** (Выбор размера с учетом рентабельности) — предоставляет рекомендации относительно выбора размера недозагруженных виртуальных машин, тем самым уменьшая расходы клиента на машины слишком большого размера или с избыточным уровнем подготовки.</span><span class="sxs-lookup"><span data-stu-id="962db-129">**Cost-Effective Sizing** - Provides right-sizing recommendations for underutilized virtual machines, thus reducing the customer’s expenses on oversized or over-provisioned machines.</span></span> <span data-ttu-id="962db-130">Это осуществляется путем оценки метрик ЦП и ОЗУ виртуальной машины (посредством API производительности), времени работы (посредством API использования) и стоимости (посредством API RateCard).</span><span class="sxs-lookup"><span data-stu-id="962db-130">It does so by examining virtual machine CPU and RAM metrics (via Performance API), hours of run-time (via Usage API) and cost (via RateCard API).</span></span> <span data-ttu-id="962db-131">После этого Cloudyn выдает рекомендации по выбору размера на основе недозагруженных ресурсов ЦП или ОЗУ (производительность) и вычисляет ожидаемую экономию, умножая разность цен (RateCard) между виртуальными машинами на фактическое время использование (использование) недозагруженной машины.</span><span class="sxs-lookup"><span data-stu-id="962db-131">Cloudyn then provides right-sizing recommendations based on underutilized CPU or RAM resources (Performance), and calculates estimated savings by multiplying the price delta (RateCard) between the VMs by the actual time-utilization (Usage) of the underutilized machine.</span></span>

![Выбор размера с учетом рентабельности][5]

* <span data-ttu-id="962db-133">**Cloud Porting Recommendations** (Рекомендации по переносу в облако) — советы финансового характера по переносу в облако.</span><span class="sxs-lookup"><span data-stu-id="962db-133">**Cloud Porting Recommendations** - Provides financial advice on cloud porting.</span></span> <span data-ttu-id="962db-134">Анализирует текущие затраты пользователя на облачные ресурсы, которые развернуты у основных облачных поставщиков, и сравнивает их со стоимостью эквивалентного развертывания в Azure.</span><span class="sxs-lookup"><span data-stu-id="962db-134">It examines a user's current costs of cloud resources which are deployed on major cloud vendors, and compares it to the cost of an equivalent deployment on Azure.</span></span> <span data-ttu-id="962db-135">Затем он создает детальные финансовые рекомендации по переносу отдельных ресурсов в Azure.</span><span class="sxs-lookup"><span data-stu-id="962db-135">It then provides granular, per-resource, financially-based porting recommendations to Azure.</span></span> <span data-ttu-id="962db-136">После определения эквивалентного развертывания в Azure (на основе метрик производительности и предпочтений пользователя) Cloudyn использует API RateCard для оценки стоимости такого эквивалентного развертывания в Azure.</span><span class="sxs-lookup"><span data-stu-id="962db-136">After assessing the equivalent deployment required on Azure (based on performance metrics and user preferences), Cloudyn uses the RateCard API to evaluate the cost of the equivalent deployment on Azure.</span></span>
* <span data-ttu-id="962db-137">**Performance Reports** (Отчеты о производительности) — эти отчеты, основанные на API производительности Azure, предоставляют целый спектр возможностей — от уровня использования ЦП и ОЗУ до рекомендаций по оптимизации.</span><span class="sxs-lookup"><span data-stu-id="962db-137">**Performance Reports** - Enabled by Azure’s performance API, these reports provide an array of features from CPU and RAM utilization to optimization recommendations.</span></span> <span data-ttu-id="962db-138">Ниже приведен пример отчета об использовании экземпляра, где представлена декомпозиция экземпляра по средней загрузке ЦП.</span><span class="sxs-lookup"><span data-stu-id="962db-138">Below is an instance utilization report example, presenting instance breakdown by average CPU utilization.</span></span>

* <span data-ttu-id="962db-140">**Диспетчер категорий** — эффективная функция в Cloudyn, которая позволяет упорядочить неорганизованные облачные ресурсы.</span><span class="sxs-lookup"><span data-stu-id="962db-140">**Category manager** - A powerful feature in Cloudyn that brings order to unorganized cloud resources.</span></span> <span data-ttu-id="962db-141">Она позволяет пользователям свободно создавать свои собственные уникальные категории (теги) для эффективного измерения показателей и ведения отчетов в соответствии с привычной для них практикой ведения бизнеса.</span><span class="sxs-lookup"><span data-stu-id="962db-141">It provides users the freedom to create their own unique categories (tags) for effective measuring and reporting that is in line with business practices.</span></span> <span data-ttu-id="962db-142">Кроме того, пользователи могут легко контролировать и классифицировать несогласованные теги (т. е. опечатки и другие отклонения) и автоматически обнаруживать непомеченные ресурсы для определения точной стоимости.</span><span class="sxs-lookup"><span data-stu-id="962db-142">Further, users can easily regulate and categorize inconsistent tagging (i.e. typos and other discrepancies) and automatically detect untagged resources for accurate cost attribution.</span></span>
