# <a name="add-or-change-azure-subscription-administrators"></a>Добавление или изменение администраторов подписки Azure

Администраторы подписки Azure классические и Azure [управление доступом на основе ролей (RBAC)](../active-directory/role-based-access-control-what-is.md) являются две системы для управления доступом к ресурсам Azure:

* Роли администратора классическую подписку обеспечивают управления базовый доступ и включить учетную запись администратора, администратор службы и администраторов.
    * При регистрации для новой подписки Azure, ваша учетная запись имеет учетную запись администратора и администратора службы по умолчанию.
    * После входа можно добавить администраторов.
* RBAC — это новая система, которая предлагает подробного управления доступом с многие встроенные роли, области и пользовательскими ролями.
    * Тем не менее пользователи только RBAC роли и роли администратора не классическую подписку невозможно управлять классический развертываний Azure.

Для обеспечения возможности контроля и упрощения управления доступом, мы рекомендуем использовать RBAC для всех требований к управлению доступом. Если это возможно рекомендуется изменить настройку с помощью RBAC существующие политики доступа. 

<a name="add-an-admin-for-a-subscription"></a>

## <a name="add-an-rbac-owner-admin-for-a-subscription-in-azure-portal"></a>Добавление администратора владельца RBAC для подписки на портале Azure 

Чтобы добавить пользователя-администратора за администрирование служб подписки Azure, сообщите ему роль владельца RBAC в подписку. Роль владельца может управлять ресурсами назначенной подписки, но не имеет доступа к другим подпискам.

1. Посетите [ **подписки** на портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade).
2. Выберите подписку, которую требуется предоставить доступ.
3. В меню выберите **Управление доступом (IAM)**.
4. В **роли** выберите **владельца**. 
5. В **назначать доступ к** выберите **пользователя Azure AD, группы или приложение**. 
6. В **выберите** введите адрес электронной почты пользователя, которого нужно добавить в качестве владельца. Выберите пользователя, а затем щелкните **Сохранить**.

    ![Снимок экрана, на котором показана выбранная роль "Владелец"](./media/billing-add-change-azure-subscription-administrator/add-role.png)

Это дает пользователю полный доступ ко всем ресурсам, включая вправо, чтобы делегировать доступ другим пользователям. Для предоставления доступа в другой области, например группы ресурсов, посетите меню IAM для данной области. 

## <a name="add-or-change-co-administrator"></a>Добавление или изменение соадминистратора

В качестве соадминистратора можно добавить только владельца. Пользователей с такими ролями, как "участник" и "читатель", невозможно добавить в качестве соадминистратора.

> [!TIP]
> Необходимо добавить учетную запись «Владелец» как соадминистратора, если пользователю требуется управлять развертыванием Azure классический. Мы рекомендуем использовать RBAC для других целей.

1. Если владелец еще не назначен, сделайте это, следуя инструкциям выше.
2. **Щелкните правой кнопкой мыши** только что добавленного владельца, а затем выберите **Добавить соадминистратора**. Если вы не видите **добавить как соадминистратора** вариант, обновите страницу или повторите другой Интернет-браузер. 

    ![Снимок экрана, на котором показаны элементы для добавления соадминистратора](./media/billing-add-change-azure-subscription-administrator/add-coadmin.png)

    Чтобы удалить разрешения соадминистратора, **щелкните правой кнопкой мыши** соадминистратора, а затем выберите **Удалить соадминистратора**.

    ![Снимок экрана, на котором показаны элементы для удаления соадминистратора](./media/billing-add-change-azure-subscription-administrator/remove-coadmin.png)

## <a name="change-the-service-administrator-for-an-azure-subscription"></a>Изменение администратора служб для подписки Azure

Только администратор учетной записи может изменить администратора службы для подписки. По умолчанию для новой подписки администратор учетных записей является также администратором служб. Если администратор службы была изменена на другого пользователя, администратор учетной записи теряет доступ к порталу Azure. Тем не менее администратор учетной записи всегда могут использовать центр учетных записей для изменения администратора службы обратно на самих себя.

1. Убедитесь, что сценарий поддерживается, просмотрев [ограничения на изменение администраторов служб](#limits).
1. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/subscriptions) в качестве администратора учетной записи.
1. Выберите подписку.
1. В правой части выберите **Изменить сведения о подписке**.

    ![Снимок экрана, отображающий кнопку "Изменить подписку" в Центре управления учетной записью](./media/billing-add-change-azure-subscription-administrator/editsub.png)
1. В поле **АДМИНИСТРАТОР СЛУЖБЫ** введите адрес электронной почты нового администратора службы.

    ![Снимок экрана, отображающий поле для изменения электронного адреса администратора служб](./media/billing-add-change-azure-subscription-administrator/changeSA.png)

<a name="limits"></a>

### <a name="limitations-for-changing-service-administrators"></a>Ограничения для изменения администраторов служб

* Каждая подписка связана с каталогом Azure AD. Чтобы найти каталог, с которой связана подписка, перейдите к [ **подписки**](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade), затем выберите подписку для просмотра каталога.
* Если вы выполняете вход с использованием рабочей или учебной учетной записи, вы можете добавлять другие учетные записи в своей организации в качестве администратора служб. Например, abby@contoso.com может добавить bob@contoso.com в качестве администратора служб, но он не может добавить john@notcontoso.com, если john@notcontoso.com не находится в каталоге contoso.com. Пользователи, выполнившие вход с использованием рабочей или учебной учетной записи, могут и далее добавлять пользователей учетной записи Майкрософт в качестве администратора служб.

  | Метод входа | Возможность добавления пользователя учетной записи Майкрософт в качестве администратора служб | Возможность добавления рабочей или учебной учетной записи в той же организации, к которой относится администратор служб | Возможность добавления рабочей или учебной учетной записи не в той же организации, к которой относится администратор служб |
  | --- | --- | --- | --- |
  |  Учетная запись Майкрософт |Yes |Нет  |Нет  |
  |  Рабочая или учебная учетная запись |Yes |Yes |Нет  |

## <a name="change-the-account-administrator-for-an-azure-subscription"></a>Изменение администратора учетной записи для подписки Azure

Администратор учетной записи является пользователь, который первоначально зарегистрировались для подписки Azure и отвечает выставления счетов владельцем подписки. Сведения о том, как изменить администратора учетной записи для подписки, см. в статье [Передача прав владения подпиской Azure другой учетной записи](billing-subscription-transfer.md).

<a name="check-the-account-administrator-of-the-subscription"></a>

**Не уверены, кто является администратором учетной записи?** Выполните следующие действия.

1. Посетите [ **подписки** на портале Azure](https://portal.azure.com/#blade/Microsoft_Azure_Billing/SubscriptionsBlade).
1. Выберите подписку, которую требуется проверить, а затем просмотрите раздел **Параметры**.
1. Выберите **Свойства**. Администратор учетной записи подписки отображается в **администратор учетной записи** поле.  

## <a name="types-of-classic-subscription-admins"></a>Типы администраторов классическую подписку

 Учетная запись администратора, администратор службы и соадминистратор являются три вида ролей администратора классическую подписку в Azure. Учетной записи, используемый для регистрации в Azure автоматически устанавливается как учетная запись администратора и администратора службы. Затем можно добавить дополнительных администраторов. Ниже перечислены точного различия между эти три роли администратора. 

> [!TIP]
> Для улучшения контроля и подробного управления доступом рекомендуется использовать на основе ролей Azure управления доступом (RBAC), что дает пользователям возможность добавить к нескольким ролям. Дополнительные сведения см. в разделе [управления доступом на основе роли Azure Active Directory](../active-directory/role-based-access-control-what-is.md).

| Классический администратор подписки | Ограничение | ОПИСАНИЕ |
| --- | --- | --- |
| Администратор учетной записи |Один на учетную запись Azure |Это пользователь, который зарегистрировались для подписки Azure и прав доступа к [центр учетных записей](https://account.azure.com/Subscriptions) и выполнять различные задачи управления. Эти сведения включают возможность создавать новые подписки, отмены подписки, измените выставление счетов за подписку и изменить администратора службы. По существу администратор учетной записи — выставления счетов владелец подписки. RBAC администратор учетной записи не назначена роль.|
| Администратор службы |Один на подписку Azure |Эта роль авторизована управлять службами на [портале Azure](https://portal.azure.com). По умолчанию для новой подписки администратор учетных записей является также администратором службы. В RBAC владельца роли дается для администратора службы на уровне подписки.|
| Соадминистратором (CA) |200 на подписку |Эта роль имеет те же права доступа, что и администратор службы, но не может менять связь подписок с каталогами Azure. Роль владельца в RBAC дается совместного администратора на уровне подписки.|
