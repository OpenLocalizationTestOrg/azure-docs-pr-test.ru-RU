# <a name="manage-reserved-virtual-machine-instances"></a>Управление зарезервированными экземплярами виртуальных машин

После покупки зарезервированного экземпляра виртуальной машины Azure может потребоваться применить резервирование к другой подписке, отличной от указанной во время покупки. Кроме того, если соответствующие виртуальные машины работают в нескольких подписках, можно изменить область резервирования на общую. Чтобы добиться максимальной скидки на резервирование, убедитесь, что количество экземпляров, которые вы приобрели, соответствует атрибутам и количеству виртуальных машин, которые у вас работают. Чтобы узнать больше о зарезервированных экземплярах виртуальных машин, перейдите на страницу [Зарезервированные экземпляры виртуальных машин Azure](https://go.microsoft.com/fwlink/?linkid=862121).

## <a name="change-the-scope-for-a-reservation"></a>Изменение области для резервирования
 Скидка на резервирование применяется к виртуальным машинам, которые соответствуют вашему резервированию и попадают в его область. Областью резервирования может быть одна или все подписки в контексте выставления счетов. Если указать одну подписку, резервирование сопоставляется с работающими в ней виртуальными машинами. Если указать общую область, то Azure сопоставляет резервирование с виртуальными машинами, работающими во всех подписках в контексте выставления счетов. Контекст выставления счетов зависит с подписки, использованной для покупки резервирования. Чтобы узнать больше, ознакомьтесь с разделом [Предоплата за виртуальные машины с помощью зарезервированных экземпляров виртуальных машин](https://go.microsoft.com/fwlink/?linkid=861721).

Вот как можно обновить область резервирования. 
1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Больше служб** > **Резервирования**.
3. Выберите резервирование.
4. Выберите **Параметры** > **Конфигурация**.
5. Изменение область. Если общую область изменить на отдельную область, то можно будет выбрать только подписку, владельцем которой вы являетесь. Можно выбрать только подписки в том же контексте выставления счетов, что и резервирование. Контекст выставления счетов определяется подпиской, выбранной при покупке резервирования. Область применяется только подпискам с оплатой по мере использования MS-AZR-0003P и подпискам с соглашением о регистрации Enterprise MS-AZR-0017P. Для подписок с соглашением о регистрации Enterprise и подписок для разработки и тестирования скидка на резервирование недоступна.

## <a name="split-a-single-reservation-into-two-reservations"></a>Разделение отдельного резервирования на два резервирования
 После покупки нескольких экземпляров может потребоваться назначить экземпляры в резервирование разным подпискам. По умолчанию все экземпляры (количество, указанное во время покупки) имеют одну область. Это либо одна подписка, либо общая область. Например, вы купили 10 виртуальных машин уровня "Стандартный D2" и указали в качестве области подписку А. Теперь вам нужно изменить область 7 зарезервированных экземпляров виртуальной машины на подписку A и оставшихся 3 — на подписку Б. Разделение резервирования позволяет распределить экземпляры, чтобы более детально управлять областью. Можно упростить выделение для подписок, выбрав общую область. Но с целью управления затратами или бюджетирования можно распределить количество между определенными подписками.

 Можно разделить резервирование на два резервирования с помощью PowerShell, интерфейса командной строки или API.

### <a name="split-a-reservation-by-using-powershell"></a>Разделение резервирования с помощью PowerShell
1. Получите идентификатор резервирования, выполнив следующую команду.

    ```powershell
    # Get the reservation orders you have access to
    Get-AzureRmReservationOrder
    ```
2. Получите сведения о резервировании.

    ```powershell
    Get-AzureRmReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId b8be062a-fb0a-46c1-808a-5a844714965a
    ```
3. Разделите резервирование на две части и распределите экземпляры.

    ```powershell
    # Split the reservation. The sum of the Reserved VM instances, the quantity, must equal the total number of instances in the reservation that you're splitting.
    Split-AzureRmReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId b8be062a-fb0a-46c1-808a-5a844714965a -Quantity 3,2
    ```
1. Чтобы изменить область, выполните следующую команду.

    ```powershell
    Update-AzureRmReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId 5257501b-d3e8-449d-a1ab-4879b1863aca -AppliedScopeType Single -AppliedScope /subscriptions/15bb3be0-76d5-491c-8078-61fe3468d414
    ```

## <a name="add-or-change-users-who-can-manage-a-reservation"></a>Добавление или изменение пользователей, которые могут управлять резервированием
Вы можете делегировать управление резервированием, добавив пользователей в роли для резервирования. По умолчанию пользователь, который купил резервирование, и учетная запись администратора участвуют в роли владельца резервирования. 

Можно управлять доступом к резервированиям независимо от подписок, получающих скидку на резервирование. Когда вы предоставляете какому-либо пользователю разрешение на управление резервированием, оно не дает ему права для управления подпиской. И если вы предоставляете какому-либо пользователю разрешение на управление подпиской, указанной в области резервирования, оно не дает ему права для управления резервированием.
 
Чтобы делегировать управление доступом к резервированию, сделайте следующее. 
1.  Войдите на [портал Azure](https://portal.azure.com).
2.  Выберите **Больше служб** > **Резервирование**, чтобы отобразить список доступных вам резервирований.
3.  Выберите резервирование, доступ к которому требуется делегировать другим пользователям.
4.  В меню выберите **Управление доступом (IAM)**.
5.  Выберите **Добавить** > **Роль** > **Владелец** (или другую роль, если вы хотите предоставить ограниченный доступ). 
6. Укажите адрес электронной почты пользователя, которого требуется добавить в роль "Владелец". 
7. Выберите пользователя, а затем щелкните **Сохранить**.
