# <a name="overview-of-integration-between-application-gateway-and-azure-security-center"></a>Общие сведения об интеграции шлюза приложений с центром безопасности Azure

В этой статье приведены сведения о том, как интеграция шлюза приложений с центром безопасности Azure позволяет обеспечить защиту ресурсов веб-приложений. Брандмауэр веб-приложения (WAF) шлюза приложений можно интегрировать с [центром безопасности](../security-center/security-center-intro.md). Это позволяет обеспечить полный обзор всей среды, благодаря чему вы сможете выявлять и предотвращать угрозы от незащищенных веб-приложений, а также реагировать на них.

## <a name="overview"></a>Обзор

Брандмауэр веб-приложения шлюза приложений — это рекомендуемый компонент центра безопасности, обеспечивающий защиту веб-приложений от распространенных эксплойтов и уязвимостей. Веб-ресурсы, не защищенные WAF, имеют в центре безопасности рекомендации высокого уровня серьезности. Рекомендации в отношении брандмауэров веб-приложений приведены на странице **Обзор** в разделе **Приложения**.

![Интеграция с центром безопасности][1]

Если щелкнуть любую рекомендацию в отношении брандмауэра веб-приложения, откроется новая колонка с подробными сведениями о ней.

## <a name="add-a-web-application-firewall-to-an-existing-resource"></a>Добавление брандмауэра веб-приложения в имеющийся шлюз приложений

Выберите **Другие службы** > **Безопасность+идентификация** > **Центр безопасности**, а затем в колонке **Центр безопасности — Обзор** щелкните **Приложения**. Таблица в колонке **Центр безопасности — Приложения** содержит список приложений, обнаруженных центром безопасности в вашей подписке.

![Веб-приложения][3]

Если щелкнуть веб-приложения с критической ошибкой, откроется колонка **Состояние защиты приложения**. На рисунке ниже показан пример веб-приложение, незащищенного брандмауэром веб-приложения. 

![Незащищенные веб-ресурсы][2]

В разделе **Рекомендации** щелкните **Добавить брандмауэр веб-приложения**, чтобы открыть колонку **Добавить брандмауэр веб-приложения**.

Если у вас нет шлюза приложений или вы хотите создать другой, в колонке **Create a new Web Application Firewall** (Создание брандмауэра веб-приложения) нажмите кнопку **Создать** и выберите **Microsoft — Шлюз приложений**. Выполните шаги по созданию шлюза приложений. На этом этапе вы добавили веб-приложение как защищенный ресурс. Теперь этот ресурс помечен в центре безопасности как защищенный брандмауэром веб-приложения. Но он еще не добавлен как элемент внутреннего пула.

Имеющийся шлюз приложений можно выбрать в разделе **Использование существующего решения**.

![Колонка добавления брандмауэра веб-приложения][4]

Если вы добавляете веб-приложение в шлюз приложений через центр безопасности, этот ресурс автоматически не добавляется как элемент внутреннего пула. Это необходимо сделать напрямую в ресурсе шлюза приложений.

## <a name="add-a-resource-to-an-existing-web-application-firewall"></a>Добавление ресурса в имеющийся брандмауэр веб-приложения

Выберите **Другие службы** > **Безопасность+идентификация** > **Центр безопасности**, а затем в колонке **Центр безопасности — Обзор** щелкните **Решения партнеров**. Имеющиеся шлюзы приложений, поддерживающие центр безопасности, отображаются в колонке **Решения партнеров**.

![Решения партнеров][7]

Щелкните **Связать приложение**, чтобы открыть колонку **Связь приложений**. Здесь можно выбрать имеющиеся приложения. Выберите приложения, которые необходимо защитить, и нажмите кнопку **ОК**. Эти действия не добавляют веб-приложение во внутренний пул. Ресурсы только помечаются как защищенные, за счет чего центр безопасности может отслеживать их. Чтобы добавить ресурс как элемент внутреннего пула, необходимо войти в шлюз приложений. Для этого в текущей колонке щелкните **Консоль решений**, чтобы перейти к ресурсу шлюза приложений, где веб-приложение можно добавить во внутренний пул.

![Сторонние приложения][6]

## <a name="finalize-configuration"></a>Завершение конфигурации

Центр безопасности отслеживает приложения, добавленные в шлюз приложений как защищенный ресурс.  Он отслеживает работоспособность этого ресурса и обеспечивает его защиту в шлюзе приложений. Далее во внутренний пул шлюза приложений необходимо добавить частный IP-адрес, общедоступный IP-адрес или сетевой адаптер виртуальной машины. Во время выполнения этих действий отображаются дополнительные рекомендации по **завершению защиты приложения**.

![Колонка добавления брандмауэра веб-приложения][5]

## <a name="security-alerts"></a>Оповещения системы безопасности

В центре безопасности выберите **Обнаружение** > **Оповещения системы безопасности**.  Здесь отображаются оповещения WAF для шлюзов приложений. Оповещения упорядочены по правилу WAF.

![Оповещения системы безопасности][8]

Если щелкнуть конкретное правило WAF, откроется список оповещений. Каждое оповещение содержит дополнительные сведения, а также ссылку на шлюз приложений.
 
![Сведения в оповещении][9]
