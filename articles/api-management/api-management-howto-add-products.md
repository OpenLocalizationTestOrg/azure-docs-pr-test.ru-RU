# <a name="create-and-publish-a-product"></a>создавать и публиковать продукт;  

В службе управления API Azure продукт содержит один или несколько API вместе с квотой и условиями использования. Как только продукт будет опубликован, разработчики могут подписаться на него и начать использовать его интерфейсы API.  

Из этого руководства вы узнаете, как выполнять такие задачи:

> [!div class="checklist"]
> * создавать и публиковать продукт;
> * добавлять API к продукту.

![Добавленный продукт](media/api-management-howto-add-products/added-product.png)

## <a name="prerequisites"></a>Технические условия

+ Выполните задачи в кратком руководстве по [созданию экземпляра службы управления API Azure](get-started-create-service-instance.md).
+ Также выполните задачи из руководства по [импорту и публикации первого API](import-and-publish.md).

## <a name="create-and-publish-a-product"></a>Создание и публикация продукта

1. Щелкните **Продукты** в меню слева, чтобы отобразить страницу **Продукты**.
2. Нажмите кнопку **+Продукт**.

    ![Добавленный продукт](media/api-management-howto-add-products/add-product.png)

    Добавляя продукт, укажите следующее: 

    |ИМЯ|ОПИСАНИЕ|
    |---|---|
    |Отображаемое имя|Имя, которое будет отображаться на **портале разработчика**.|
    |ИМЯ|Описательное имя продукта.|
    |ОПИСАНИЕ|В поле **Описание** можно ввести подробные сведения о продукте, например его назначение, API-интерфейсы, к которым он предоставляет доступ, и другую полезную информацию.|
    |Состояние|Нажмите кнопку **Опубликовано**, чтобы опубликовать продукт. Прежде чем можно будет вызывать интерфейсы API в продукте, сам продукт должен быть опубликован. По умолчанию новые продукты не опубликованы и видны только группе **Администраторы**.|
    |Запрос утверждения|Установите флажок **Require subscription approval** (Требуется утверждение администратора), если вы хотите, чтобы администратор рассматривал и принимал или отклонял попытки подписки на этот продукт. Если флажок не установлен, попытки подписаться будут утверждаться автоматически. |
    |Ограничение числа подписок|Чтобы ограничить число одновременных подписок, укажите соответствующий лимит. |
    |Условия использования|Можно указать условия использования продукта, которые должны принимать подписчики, если они хотят использовать этот продукт.|
    |Интерфейсы API|Продуктами называют ассоциации из одного или нескольких API. Вы можете включить определенное количество API-интерфейсов и предлагать их разработчикам через портал разработчика. <br/> При разработке продукта можно добавить уже существующий API. API можно добавить к продукту позже с помощью страницы **Настройки** для продуктов или при создании самого API.|<br/>Чтобы получить доступ к API, разработчикам необходимо сначала подписаться на продукт. После этого они получат ключ подписки, который подходит для любого API в этом продукте.<br/> Создавая экземпляр службы управления API, вы автоматически становитесь его администратором. Поэтому вы по умолчанию будете подписаны на все продукты.|

3. Нажмите кнопку **создать** для создания нового продукта.

### <a name="add-more-configurations"></a>Добавление дополнительных конфигураций

Вы можете продолжить настраивать сохраненный продукт, выбрав вкладку **Настройки**. 

Просматривать или добавлять подписчиков в продукт можно с помощью вкладки **Подписки**.

Настраивать видимость продукта для разработчиков или гостей можно на вкладке **Управление доступом**.

## <a name="add-apis"> </a>Добавление интерфейсов API в продукт

Продуктами называют ассоциации из одного или нескольких API. Вы можете включить определенное количество API-интерфейсов и предлагать их разработчикам через портал разработчика. При разработке продукта можно добавить уже существующий API. API можно добавить к продукту позже с помощью страницы **Настройки** для продуктов или при создании самого API.

Чтобы получить доступ к API, разработчикам необходимо сначала подписаться на продукт. После этого они получат ключ подписки, который подходит для любого API в этом продукте. Создавая экземпляр службы управления API, вы автоматически становитесь его администратором. Поэтому вы по умолчанию будете подписаны на все продукты.

### <a name="add-an-api-to-an-existing-product"></a>Добавление API к существующему продукту

1. Выберите продукт.
2. Выберите вкладку API-интерфейсов.
3. Нажмите кнопку **+API**.
4. Выберите API и нажмите кнопку **Создать**.
