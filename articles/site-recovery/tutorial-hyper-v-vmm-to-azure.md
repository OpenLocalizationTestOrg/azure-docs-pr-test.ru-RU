# <a name="set-up-disaster-recovery-of-on-premises-hyper-v-vms-in-vmm-clouds-to-azure"></a>Настройка аварийного восстановления виртуальных машин Hyper-V локально в облаках VMM в Azure

Служба [Azure Site Recovery](site-recovery-overview.md) помогает реализовать стратегию аварийного восстановления за счет управления процессами репликации, отработки отказа и восстановления локальных компьютеров и виртуальных машин Azure.

В этом руководстве показано, как настроить аварийное восстановление локальных виртуальных машин Hyper-V в Azure. Учебник относится для виртуальных машин Hyper-V, управляемых по диспетчера виртуальных машин System Center (VMM). Из этого руководства вы узнаете, как выполнять такие задачи:

> [!div class="checklist"]
> * Выберите репликации исходной и целевой.
> * Настройка среды репликации источника, включая локальные компоненты Site Recovery и целевая среда репликации.
> * Настройте сетевое сопоставление для сопоставления между сетями виртуальных Машин VMM и виртуальными сетями Azure.
> * Создание политики репликации
> * Включение репликации для виртуальных машин.

Это третье руководство в серии. В этом руководстве предполагается, что вы уже выполнили задачи в предыдущих руководствах:

1. [Подготовка Azure](tutorial-prepare-azure.md).
2. [Подготовка локального Hyper-V](tutorial-prepare-on-premises-hyper-v.md)

Прежде чем начать, бывает полезно [Изучите архитектуру](concepts-hyper-v-to-azure-architecture.md) в этом сценарии аварийного восстановления.



## <a name="select-a-replication-goal"></a>Выберите цель репликации

1. В **все службы** > **хранилищ служб восстановления**, щелкните имя хранилища, мы используем в этих учебниках **ContosoVMVault**.
2. В **Приступая к работе**, нажмите кнопку **Site Recovery**. Нажмите кнопку **Подготовка инфраструктуры**
3. В **цель защиты** > **вашей машины, расположенные в которых**выберите **локальной**.
4. В **место реплицировать компьютеры**выберите **для Azure**.
5. В **вашей машины виртуализировать**выберите **Да, с помощью Hyper-V**.
6. В **вы используете System Center VMM**выберите **Да**. Нажмите кнопку **ОК**.

    ![Целью репликации](./media/tutorial-hyper-v-vmm-to-azure/replication-goal.png)



## <a name="set-up-the-source-environment"></a>Настройка исходной среды

При настройке исходной среды следует установить поставщик Azure Site Recovery и агент служб восстановления Azure, а также зарегистрировать локальные серверы в хранилище. 

1. В разделе **Подготовка инфраструктуры** щелкните **Источник**.
2. В поле **Подготовка источника** щелкните **+ VMM**, чтобы добавить сервер VMM. В колонке **Добавление сервера** в поле **Тип сервера** должно быть указано **System Center VMM server** (Сервер System Center VMM).
3. Загрузите установщик для поставщика Microsoft Azure Site Recovery.
4. Скачайте ключ регистрации хранилища. Он понадобится при запуске программы установки поставщика. Ключ действителен в течение пяти дней после создания.
5. Загрузите агент служб восстановления.

    ![Загрузка](./media/tutorial-hyper-v-vmm-to-azure/download-vmm.png)

### <a name="install-the-provider-on-the-vmm-server"></a>Установка поставщика на сервер VMM

1. В мастере Установка поставщика Azure Site Recovery > **центра обновления Майкрософт**, согласие на использование Центра обновления Майкрософт для проверки наличия обновлений поставщика.
2. В **установки**примите расположение установки по умолчанию для поставщика и нажмите кнопку **установить**. 
3. После установки, мастер регистрации Microsoft Azure Site Recovery > **параметры хранилища**, нажмите кнопку **Обзор**и в **файла ключа**, выберите файл ключа хранилища, которые загружено.
4. Укажите подписку Azure Site Recovery и имя хранилища (**ContosoVMVault**). Укажите понятное имя для сервера VMM, чтобы идентифицировать его в хранилище.
5. В **параметры прокси-сервера**выберите **прямое подключение к Azure Site Recovery без учетной записи-посредника**.
6. Примите расположение по умолчанию для сертификата, который используется для шифрования данных. После отработки отказа будет расшифровать зашифрованные данные.
7. В **синхронизация метаданных облака**выберите **синхронизировать метаданные на портале Site Recovery**. На каждом сервере это действие требуется выполнять только один раз. Нажмите кнопку **зарегистрировать**.
8. После регистрации сервера в хранилище нажмите кнопку **Готово**.

Когда регистрация завершится, Azure Site Recovery извлечет метаданные с сервера, а сервер VMM отобразится в колонке **Инфраструктура Site Recovery**.

### <a name="install-the-recovery-services-agent"></a>Установка агента служб восстановления

Установите агент на каждом узле Hyper-V, содержащий виртуальные машины, которые требуется реплицировать.

1. В мастер установки агента служб Microsoft Azure Recovery > **проверка предварительных требований**, нажмите кнопку **Далее**. Все пропущенные необходимые компоненты будут автоматически устанавливаться.
2. В **параметры установки**, примите расположение установки, а также расположение кэша. Диску кэша требуется по крайней мере 5 ГБ места для хранения. Мы рекомендуем диск с 600 ГБ свободного места (минимум). Нажмите **Install**(Установить).
3. На странице **Установка** после завершения установки нажмите кнопку **Закрыть** для завершения работы мастера.

    ![Установка агента](./media/tutorial-hyper-v-vmm-to-azure/mars-install.png)
    

## <a name="set-up-the-target-environment"></a>Настройка целевой среды

1. Выберите **Подготовка инфраструктуры** > **Цель**.
2. Выберите подписку и группу ресурсов (**ContosoRG**), в которой виртуальные машины Azure будет создан после отработки отказа.
3. Выберите **диспетчер ресурсов»** модели развертывания.

Site Recovery проверяет наличие одной или нескольких совместимых учетных записей хранения и сетей Azure.


## <a name="configure-network-mapping"></a>Настройка сетевого сопоставления

1. Выберите **Site Recovery Infrastructure** (Инфраструктура Site Recovery) > **Сетевые сопоставления** > **Сетевое сопоставление** и щелкните значок **+Network Mapping** (+Сетевое сопоставление).
2. В разделе **Add network mapping** (Добавить сетевое сопоставление) выберите исходный сервер VMM. Выберите **Azure** в качестве целевой среды.
3. Проверьте подписку и модель развертывания после отработки отказа.
4. В поле **Исходная сеть** выберите исходную локальную сеть виртуальной машины.
5. На странице **Целевая сеть** выберите сеть Azure, в которой будут размещены реплицированные виртуальные машины Azure. Нажмите кнопку **ОК**.

    ![Сетевое сопоставление](./media/tutorial-hyper-v-vmm-to-azure/network-mapping-vmm.png)

## <a name="set-up-a-replication-policy"></a>Настройка политики репликации

1. Щелкните **Prepare infrastructure** (Подготовка инфраструктуры) > **Параметры репликации** > **+Create and associate** (+Создание и связывание)
2. В **создать и связать политику**, укажите имя политики, **ContosoReplicationPolicy**.
3. Оставьте параметры по умолчанию и нажмите кнопку **ОК**.
    - **Частота копирования** указывает, что разностные данные (после начальной репликации) будет реплицировать каждые пять минут.
    - **Хранения точки восстановления** , периоды хранения для каждой из точек восстановления будет два два часа.
    - **Частота моментальных снимков согласованности приложений** указывает, что точек восстановления, содержащих приложения моментальных снимков будут создаваться каждый час.
    - **Время начала начальной репликации**, указывает, что начальная репликация будет запущен немедленно.
    - **Шифровать данные, хранящиеся в Azure** -по умолчанию **Off** параметр указывает, что статических данных в Azure не зашифрованы.
4. После создания политики щелкните **ОК**. При создании политики она автоматически сопоставляется с облаком VMM.

## <a name="enable-replication"></a>Включение репликации

1. В **репликация приложения**, нажмите кнопку **источника**. 
2. В **источника**, выберите облако VMM. Нажмите кнопку **ОК**.
3. В **целевой**, убедитесь в качестве целевой подписки хранилища Azure и выберите **диспетчера ресурсов** модели.
4. Выберите **contosovmsacct1910171607** учетной записи хранилища и **ContosoASRnet** сети Azure.
5. В **виртуальные машины** > **выберите**, выберите виртуальную Машину, нужно реплицировать. Нажмите кнопку **ОК**.

 Ход выполнения действия **включения защиты** можно отслеживать, выбрав **Задания** > **Задания Azure Site Recovery**. После завершения задания **Завершение подготовки защиты** начальная репликация завершается, а виртуальная машина готова к отработке отказа.
