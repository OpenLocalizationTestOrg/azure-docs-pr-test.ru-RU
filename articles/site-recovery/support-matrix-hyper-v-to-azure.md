# <a name="support-matrix-for-hyper-v-replication-to-azure"></a>Таблица поддержки для репликации Hyper-V в Azure


В этой статье перечислены поддерживаемые компоненты и параметры для аварийного восстановления виртуальных машин Hyper-V локально в Azure, с помощью [Azure Site Recovery](site-recovery-overview.md) службы.


## <a name="supported-scenarios"></a>Поддерживаемые сценарии использования.

**Сценарий** | **Дополнительные сведения**
--- | --- 
**Hyper-V с VMM** | Можно выполнить аварийное восстановление в Azure для виртуальных машин, работающих на узлах Hyper-V, управляемых в структуре, диспетчер виртуальных машин System Center (VMM)<br/><br/> Можно развернуть этот сценарий в портале Azure или с помощью PowerShell.<br/><br/> Если узлы Hyper-V управляются с помощью VMM, можно также выполнить аварийное восстановление для получателей на локальном сайте. Чтение [в этой статье](tutorial-vmm-to-vmm.md) для получения дополнительных сведений об этом сценарии.
**Hyper-V без VMM** | Можно выполнить аварийное восстановление в Azure для виртуальных машин, работающих на узлах Hyper-V, которые не управляются VMM.<br/><br/> Можно развернуть этот сценарий в портале Azure или с помощью Powershell. 


## <a name="on-premises-servers"></a>Локальные серверы

**Сервер** | **Требования** | **Дополнительные сведения**
--- | --- | ---
**Hyper-V (выполнение без VMM)** | Windows Server 2016, Windows Server 2012 R2 с последними обновлениями. | При настройке узла Hyper-V в Site Recovery сочетание узлов под управлением Windows Server 2016 и 2012 R2 не поддерживается.<br/><br/> Для виртуальных машин, размещенных на узле под управлением Windows Server 2016 не поддерживается восстановление в альтернативное расположение.
**Hyper-V (под управлением VMM)** | 2016 VMM, VMM 2012 R2 | При использовании VMM в VMM 2016 должны управляться узлов Windows Server 2016.<br/><br/> В настоящее время не поддерживается облаков VMM, которые смешаны узлов Hyper-V под управлением Windows Server 2016 и 2012 R2.<br/><br/> Среды, в которых включить обновление существующего сервера VMM 2012 R2 до 2016 не поддерживаются.

## <a name="azure-vm-requirements"></a>Требования виртуальной Машины Azure

Локальных виртуальных машин, которые реплицируются в Azure должны отвечать требованиям виртуальной Машине Azure, представленные в этой таблице.

**Компонент** | **Требования** | **Дополнительные сведения**
--- | --- | ---
**Операционная система на виртуальной машине** | Восстановление узла поддерживает все операционные системы, которые являются [поддерживается Azure](https://technet.microsoft.com/library/cc794868%28v=ws.10%29.aspx).  | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Архитектура операционной системы на виртуальной машине** | 64-разрядная | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Размер диска операционной системы** | Виртуальные машины поколения 1 до 2048 ГБ.<br/><br/> Виртуальные машины поколения 2 до 300 ГБ.  | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Число дисков операционной системы** | 1 | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Число дисков с данными** | 16 или меньше  | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Размер виртуального жесткого диска с данными** | До 4095 ГБ. | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Сетевые адаптеры** | Поддерживаются несколько адаптеров |
**Общий виртуальный жесткий диск** | Не поддерживается | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Диск FC** | Не поддерживается | Если не поддерживается, проверка на соответствие обязательным требованиям завершится с ошибкой.
**Формат жесткого диска** | VHD  <br/><br/> VHDX | Site Recovery автоматически преобразует VHDX в VHD, после отработки отказа в Azure. При восстановлении к локальной системе виртуальные машины продолжают использовать формат VHDX.
**BitLocker** | Не поддерживается | Необходимо отключить шифрование BitLocker, прежде чем включить репликацию для виртуальной Машины.
**Имя виртуальной машины** | От 1 до 63 символов, при этом допустимы только буквы, цифры и дефисы Имя виртуальной машины должно начинаться и заканчиваться буквой или цифрой. | Измените значение в свойства Виртуальной машины в Site Recovery.
**Тип виртуальной машины** | Поколение 1<br/><br/> Поколение 2 — Windows | Поддерживаются виртуальные машины второго поколения с диском ОС типа "Базовый" (включая один или два тома данных в формате VHDX) и объемом менее 300 ГБ.<br></br>Виртуальные машины Linux второго поколения не поддерживаются. [Подробнее](https://azure.microsoft.com/blog/2015/04/28/disaster-recovery-to-azure-enhanced-and-were-listening/)|
