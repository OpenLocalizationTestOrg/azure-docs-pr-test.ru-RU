# <a name="create-a-public-ip-address-in-an-availability-zone-with-the-azure-portal"></a>Создание общедоступного IP-адреса в зоне доступности с помощью портала Azure.

Общедоступный IP-адрес можно развернуть в зоне доступности Azure (предварительная версия). Зона доступности представляет собой физически отдельную зону в регионе Azure. Ниже перечислено, что вы можете узнать.

> * Создание общедоступного IP-адреса в зоне доступности
> * Определение связанных ресурсов, созданных в зоне доступности

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

> [!NOTE]
> Зоны доступности используются в режиме предварительной версии в сценариях разработки и тестирования. Поддержка реализована для некоторых ресурсов, регионов и семейств размеров виртуальных машин Azure. Дополнительные сведения о начале работы и о том, какие ресурсы, регионы и семейства размеров виртуальных машин Azure поддерживаются для зон доступности, см. в разделе [Обзор зон доступности](https://docs.microsoft.com/azure/availability-zones/az-overview). Чтобы получить поддержку, обратитесь на форум [StackOverflow](https://stackoverflow.com/questions/tagged/azure-availability-zones) или [создайте запрос в службу поддержки Azure](../azure-supportability/how-to-create-azure-support-request.md?toc=%2fazure%2fvirtual-network%2ftoc.json).

## <a name="log-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу https://portal.azure.com. 

## <a name="create-a-zonal-public-ip-address"></a>Создание зонального общедоступного IP-адреса

1. Щелкните **Создать** в верхнем левом углу портала Azure.
2. Выберите **Сети**, а затем **Общедоступный IP-адрес**.
3. Введите или выберите значения для следующих параметров, выберите подписку, примите значения по умолчанию для остальных параметров, а затем нажмите кнопку **Создать**.

    |Настройка|Значение|
    |---|---|
    |SKU| **Базовый**: назначаются с помощью статического или динамического метода выделения. Могут быть назначены любому ресурсу Azure, которому может быть присвоен общедоступный IP-адрес, например сетевым интерфейсам, VPN-шлюзам, шлюзам приложений и подсистемам балансировки нагрузки, доступным в Интернете. Общедоступный IP-адрес можно назначить конкретной зоне с помощью параметра **Зона доступности**. Без поддержки избыточности в пределах зоны. Дополнительные сведения см. в [обзоре зон доступности](https://docs.microsoft.com/azure/availability-zones/az-overview). **Стандартный**: назначаются только с помощью статического метода выделения. Могут быть назначены сетевым интерфейсам или стандартным балансировщикам нагрузки с доступом в Интернет. Если вы собираетесь назначить общедоступный IP-адрес стандартному балансировщику нагрузки с доступом в Интернет, необходимо выбрать вариант "Стандартный". Дополнительные сведения об SKU балансировщиков нагрузки Azure см. в [соответствующем разделе](https://docs.microsoft.com/azure/load-balancer/load-balancer-standard-overview). Поддержка избыточности в пределах зоны по умолчанию. Можно создавать зональные и гарантированные IP-адреса в конкретной зоне доступности. Стандартные SKU доступны в режиме предварительной версии. Прежде чем создавать общедоступный IP-адрес стандартного SKU, необходимо зарегистрироваться для участия в предварительной версии. Дополнительные сведения см. в разделе о [регистрации для участия в предварительной версии стандартных SKU](https://docs.microsoft.com/azure/load-balancer/load-balancer-standard-overview#preview-sign-up). Стандартные SKU можно создавать только в поддерживаемых расположениях.  См. список [поддерживаемых расположений (регионов)](https://docs.microsoft.com/azure/load-balancer/load-balancer-standard-overview#region-availability). Также проверяйте страницу [обновлений виртуальной сети Azure](https://azure.microsoft.com/updates/?product=virtual-network) на наличие дополнительных регионов.|   
    |Имя|Имя должно быть уникальным в пределах выбранной группы ресурсов.|
    |Группа ресурсов|Нажмите кнопку "Создать новый", а затем введите myResourceGroup.|
    |Расположение|Западная Европа|
    |Зона доступности|Если вы выбрали **стандартный** SKU, можно выбрать *Избыточный в пределах зоны*, если необходимо обеспечить устойчивость IP-адреса в разных зонах. При выборе **основного** SKU IP-адрес не является устойчивым в разных зонах. Независимо от выбранного SKU IP-адрес можно назначить конкретной зоне. |

    Параметры отображаются на портале, как показано на следующем рисунке:

    ![Создание зонального общедоступного IP-адреса](./media/create-public-ip-availability-zone-portal/public-ip-address.png) 

> [!NOTE]
> При назначении общедоступного IP-адреса стандартных SKU сетевому интерфейсу виртуальной машины необходимо явно разрешить предполагаемый трафик с помощью [группы безопасности сети](security-overview.md#network-security-groups). Обмен данными с ресурсом будет невозможен, пока вы не создадите и не свяжете группу безопасности сети и явно не разрешите нужный трафик.
