# <a name="whats-new-in-azure-data-catalog"></a>Новые возможности каталога данных Azure
Обновления для **каталога данных Azure** выпускаются на регулярной основе. Не все версии содержат новые компоненты, доступные пользователям. Некоторые версии ориентированы на возможности внутренней службы. На этой странице описываются новые доступные пользователям возможности, которые добавлены в службу каталога данных Azure.

## <a name="whats-new-for-november-2017"></a>Новые возможности по состоянию на сентябрь 2017 г. 
По состоянию на ноябрь 2017 г. в каталог данных Azure добавлены следующие возможности:

* Поддержка прямых ссылок на конкретные термины в бизнес-глоссарии на портале каталога данных. Пользователи могут копировать ссылки из бизнес-глоссария и внедрять их в документы, сообщения электронной почты, отчеты и другие расположения, тем самым создавая прямые ссылки на определения терминов в глоссарии.
* Поддержка субъектов-служб Azure Active Directory. Администраторы каталога данных могут предоставлять доступ к каталогу клиентским приложениям, использующим субъекты-службы (похоже на предоставление разрешений пользователям и группам безопасности). Дополнительные сведения см. в статье [Объекты приложения и субъекта-службы в Azure Active Directory (Azure AD)](../active-directory/develop/active-directory-application-objects.md).
* Поддержка аутентификации Azure Active Directory при подключении к источникам данных в базе данных SQL Azure и хранилище данных SQL Azure с помощью средства регистрации источника данных каталога данных. Дополнительные сведения см. в статье [Использование аутентификации Azure Active Directory для аутентификации с помощью базы данных SQL или хранилища данных SQL](../sql-database/sql-database-aad-authentication.md).


## <a name="whats-new-for-september-2017"></a>Новые возможности выпуска за сентябрь 2017 г. 
По состоянию на сентябрь 2017 г. в каталог данных Azure добавлены следующие возможности:

* Возможность извлекать метаданные связи соединения из источников данных DB2 при регистрации связанных таблиц с помощью средства для регистрации источников данных.
* Поддержка регистрации источников данных MongoDB версии 3.4 с помощью средства для регистрации источников данных.
* Поддержка удаления за одну операцию всех метаданных объектов, содержащихся в базе данных или другом контейнере, который удаляется из каталога данных.
* Поддержка просмотра до 1000 тегов, терминов бизнес-глоссария или других аспектов поиска при уточнении поиска на портале каталога данных.


## <a name="whats-new-for-august-2017"></a>Новые возможности выпуска в августе 2017 г. 
По состоянию на август 2017 г. в каталог данных Azure добавлены следующие возможности:

*   Для разработчиков доступен новый пример создания метаданных связи и управления ими с помощью REST API каталога данных. Пример *импорта сведений о связи в каталог данных* доступен на [странице примеров кода для каталога данных](https://azure.microsoft.com/resources/samples/?service=data-catalog&sort=0). 
* Возможность извлекать метаданные связи соединения из источников данных Teradata при регистрации связанных таблиц с помощью средства для регистрации источников данных.
* Поддержка объектов SQL Server, использующих функцию с табличным значением (TVF), при регистрации источников данных SQL Server с помощью средства для регистрации источников данных.
* Внесены значительные обновления и усовершенствования, которые повышают производительность и удобство использования портала каталога данных.

## <a name="whats-new-for-july-2017"></a>Новые возможности выпуска в июле 2017 г. 
По состоянию на июль 2017 г. в каталог данных Azure добавлены следующие возможности:
*   Повышена точность управления разрешенными операциями с метаданными, включая следующие:
    - администраторы каталога могут ограничить возможность пользователей добавлять теги и связанные метаданные в каталог, предоставив доступ к каталогу только для чтения;
    - администраторы каталога могут ограничить возможность пользователей регистрировать новые источники данных в каталоге;
    - администраторы каталога могут ограничить возможность пользователей стать владельцами метаданных ресурса данных в каталоге;
    - чтобы упростить управление разрешениями, их можно предоставлять пользователям и группам безопасности Azure Active Directory.
* Поддержка связей между зарегистрированными ресурсами данных и обнаружения связанных ресурсов данных на портале каталога данных, включая:
    - извлечение метаданных связи из источников данных SQL Server (в том числе базы данных SQL Azure), Oracle и MySQL при использовании средства для регистрации источников данных в каталоге данных;
    - обнаружение связанных ресурсов данных при просмотре метаданных ресурса на портале каталога данных;
    - операции определения и обнаружения связей между ресурсами данных, а также управления связями с помощью REST API каталога данных.

Дополнительные сведения об управлении разрешениями в каталоге данных см. в статье [Как обеспечить безопасный доступ к каталогу данных и ресурсам данных](data-catalog-how-to-secure-catalog.md).
Дополнительные сведения о связях в каталоге данных см. в статье [Как просматривать связанные ресурсы данных в каталоге данных Azure](data-catalog-how-to-view-related-data-assets.md).

## <a name="whats-new-for-june-2017"></a>Новые возможности выпуска в июне 2017 г.  
По состоянию на июнь 2017 г. в каталог данных Azure добавлены следующие возможности.
*   Поддержка источников данных Sybase, Apache Cassandra и MongoDB. Теперь пользователи могут зарегистрировать и обнаруживать базы данных и таблицы Cassandra и MongoDB, а также базы данных, таблицы и представления Sybase.

> [!NOTE]
> При регистрации таблиц MongoDB и Cassandra, которые включают столбцы со сложными типами данных, например записи и массивы, эти столбцы не будут включены в метаданные, добавленные в каталог данных.

## <a name="whats-new-for-may-2017"></a>Новые возможности выпуска в мае 2017 г. 
По состоянию на май 2017 г. в каталог данных Azure добавлены следующие возможности.
*   Для массового импорта терминов бизнес-глоссария доступен новый образец для разработчиков. Образец массового импорта глоссария доступен на [странице примеров каталога данных для разработчиков](https://docs.microsoft.com/azure/data-catalog/data-catalog-samples). 
*   Поддержка редактирования сведений о подключении ODBC на портале каталога данных Azure. Владельцы ресурсов данных и администраторы каталога данных теперь могут изменять сведения о подключении для зарегистрированных источников данных ODBC без необходимости повторной регистрации этих источников.
*   Поддержка интерактивных URL-адресов в описаниях и определениях бизнес-глоссария. Если URL-адреса включены в свойства описания и определения терминов бизнес-глоссария, URL-адреса будут отображаться как гиперссылки на портале каталога данных.
*   Поддержка отображения имен экспертов помимо адресов электронной почты экспертов. При просмотре экспертов в свойствах ресурса данных на портале каталога данных полное имя эксперта из Azure Active Directory будет отображаться во всплывающей подсказке.

## <a name="whats-new-for-april-2017"></a>Новые возможности выпуска в апреле 2017 г. 
По состоянию на апрель 2017 г. в каталог данных Azure добавлены следующие возможности.
*   Поддержка источников данных ODBC. Теперь пользователи могут регистрировать и обнаруживать базы данных, таблицы и представления ODBC с помощью средства регистрации источника данных каталога данных.

## <a name="whats-new-for-march-2017"></a>Новые возможности выпуска в марте 2017 г. 
По состоянию на март 2017 г. в каталог данных Azure добавлены следующие возможности.
*   Поддержка использования групп безопасности AAD для администраторов каталога данных.
*   Каталог данных Azure теперь доступен в новом регионе Azure. Клиенты могут выполнять подготовку каталога данных Azure в западно-центральной части США, а также в восточной части США, западной части США, Западной Европе, восточной Австралии, Северной Европе и Юго-Восточной Азии. Дополнительные сведения см. на странице [Регионы Azure](https://azure.microsoft.com/regions/).

## <a name="whats-new-for-february-2017"></a>Новые возможности выпуска в феврале 2017 г. 
По состоянию на февраль 2017 г. в каталог данных Azure добавлены следующие возможности.
*   Поддержка дополнительных параметров в средстве регистрации источника данных каталога данных. При регистрации больших источников данных пользователи могут указать значения времени ожидания команды.
*   Поддержка источников данных FTP и PostgreSQL. Теперь пользователи могут регистрировать и обнаруживать FTP-файлы и каталоги, а также базы данных, таблицы и представления PostgreSQL.

## <a name="whats-new-for-january-2017"></a>Новые возможности выпуска в январе 2017 г. 
По состоянию на январь 2017 г. в каталог данных Azure добавлены следующие возможности.
*   Каталог данных Azure теперь совместим с [CSA STAR](https://www.microsoft.com/trustcenter/compliance/csa-star-certification).
*   Интеграция с набором функций [Скачать и преобразовать в Excel 2016 и Power Query для Excel](https://support.office.com/article/Introduction-to-Microsoft-Power-Query-for-Excel-6E92E2F4-2079-4E1F-BAD5-89F6269CD605). Пользователи Excel могут предоставлять общий доступ к запросам и обнаруживать запросы с помощью каталога данных Azure в Excel. Эта функциональность доступна пользователям с лицензиями Power BI Pro.

## <a name="whats-new-for-december-2016"></a>Новые возможности выпуска в декабре 2016 г.
По состоянию на декабрь 2016 г. в каталог данных Azure добавлены следующие возможности.
*   Каталог данных Azure теперь совместим с [HIPAA](https://www.microsoft.com/trustcenter/Compliance/HIPAA) и [Предложениями по модели ЕС](https://www.microsoft.com/TrustCenter/Compliance/EU-Model-Clauses).
*   Поддержка редактирования информации о подключении к источнику данных. Владельцы ресурсов данных и администраторы каталога данных теперь могут изменять сведения о подключении для зарегистрированных источников данных без необходимости повторной регистрации этих источников.
*   Поддержка источников данных Salesforce.com. Теперь пользователи могут регистрироваться и обнаруживать объекты Salesforce.


## <a name="whats-new-for-november-2016"></a>Новые возможности выпуска в ноябре 2016 г.
По состоянию на ноябрь 2016 г. в каталог данных Azure добавлены следующие возможности.
*   Теперь каталог данных Azure соответствует стандартам [ISO/IEC 27001](https://www.microsoft.com/trustcenter/compliance/iso-iec-27001) и [ISO/IEC 27018](https://www.microsoft.com/TrustCenter/Compliance/iso-iec-27018).
*   Поддержка регистрации источников данных ODBC вручную с помощью портала каталога данных и REST API.

## <a name="whats-new-for-september-2016"></a>Новые возможности выпуска в сентябре 2016 г.
По состоянию на сентябрь 2016 г. в каталог данных Azure добавлены следующие возможности.

* Поддержка источников данных IBM DB2. Теперь пользователи могут регистрироваться и обнаруживать базы данных, таблицы и представления DB2.
* Поддержка источников данных Azure Cosmos DB. Теперь пользователи могут регистрироваться и обнаруживать базы данных и коллекции Cosmos DB.
* Поддержка настройки имени каталога на портале каталога данных. Теперь администраторы каталога могут указать текст, который будет отображаться в заголовке на портале, например название организации.

## <a name="whats-new-for-august-2016"></a>Новые возможности выпуска в августе 2016 г.
По состоянию на август 2016 г. в каталог данных Azure добавлены следующие возможности.

* Улучшенные возможности регистрации источников данных служб SQL Server Master Data Services (MDS). Теперь пользователи могут добавить данные для предварительного просмотра и профили данных при регистрации сущностей MDS с помощью инструмента регистрации источников данных в каталоге данных.
* Поддержка сохраненных поисковых запросов организации, определенных администратором. Теперь администраторы могут сохранить на портале каталога данных поисковые запросы для личного использования или для всех пользователей каталога. Сохраненные поисковые запросы организации могут видеть все пользователи каталога. Они могут служить в качестве стандартизированных отправных точек для обнаружения источников данных.
* Обновления представления свойств на портале каталога данных. Теперь все свойства ресурсов данных отображаются и управляются в единой области регулируемого размера, что обеспечивает более согласованные условия работы и упрощает обнаружение данных.

## <a name="whats-new-for-july-2016"></a>Новые возможности выпуска в июле 2016 г.
По состоянию на июль 2016 г. в каталог данных Azure добавлены следующие возможности.

* Поддержка источников данных служб SQL Server Master Data Services (MDS). Теперь пользователи могут регистрироваться и обнаруживать модели и сущности MDS.
* Поддержка хранимых процедур SQL Server. Теперь пользователи могут регистрироваться и обнаруживать объекты хранимых процедур в источниках данных SQL Server.
* Поддержка дополнительных языков на портале каталога данных Azure и в инструменте регистрации источников данных (до 18 языков). Локализация интерфейса каталога данных Azure осуществляется на основе языковых параметров, заданных в Windows или веб-браузере пользователя.
* Обновления и улучшения домашней страницы портала каталога данных, включая повышение производительности и оптимизацию работы пользователей.

## <a name="whats-new-for-june-2016"></a>Новые возможности выпуска в июне 2016 г.
По состоянию на июнь 2016 г. в каталог данных Azure добавлены следующие возможности.

* Поддержка изменения размеров столбцов в представлении списка при обнаружении ресурсов данных на портале каталога данных. Теперь пользователи могут изменять размер отдельных столбцов для более удобного чтения объемных метаданных ресурса, например тегов и описаний.
* На портале каталога данных в меню "Открыть в" добавлен пункт "Power Query для Excel". Теперь пользователи могут открывать поддерживаемые источники данных в Excel 2016, Excel 2010 или в Excel 2013 с установленной надстройкой [Power Query для Excel](https://support.office.com/article/Introduction-to-Microsoft-Power-Query-for-Excel-6E92E2F4-2079-4E1F-BAD5-89F6269CD605) .
* Поддержка источников данных службы хранилища таблиц Azure. Теперь пользователи могут регистрироваться и обнаруживать объекты таблицы в источниках данных службы хранилища Azure.

## <a name="whats-new-for-may-2016"></a>Новые возможности выпуска в мае 2016 г.
По состоянию на май 2016 г. в каталог данных Azure добавлены следующие возможности.

* Бизнес-глоссарий, который позволяет администраторам каталога определять бизнес-термины и иерархии для создания общего бизнес-словаря. Пользователи могут помечать зарегистрированные ресурсы данных терминами глоссария, чтобы упростить обнаружение и понимание содержимого каталога. Дополнительные сведения см. в статье [Как настроить бизнес-глоссарий для регулируемого добавления тегов](data-catalog-how-to-business-glossary.md).  
* Улучшенный бизнес-глоссарий каталога данных предоставляет пользователям возможность обновлять несколько терминов за одну операцию. Пользователи могут выбрать несколько условий для изменения в следующих полях:
  * Родительский термин: пользователь может выбрать новый родительский термин, в случае чего после обновления все выбранные термины станут для этого термина дочерними. Если все выбранные термины имеют один и тот же родительский термин, он отображается в текстовом поле; в противном случае поле родительского термина будет пустым.   
  * Теги и заинтересованные лица: пользователи могут добавлять и удалять теги и заинтересованных лиц для нескольких терминов в глоссарии, используя такой же интерфейс, как при назначении тегов для несколько ресурсов данных.

> [!NOTE]
> Бизнес-глоссарий доступен только в выпуске "Стандартный" каталога данных Azure. Выпуск "Бесплатный" не предоставляет возможности для использования регулируемого добавления тегов или бизнес-глоссария.

## <a name="whats-new-for-march-2016"></a>Новые возможности выпуска в марте 2016 г.
По состоянию на март 2016 г. в каталог данных Azure добавлены следующие возможности.

* Объединенная конечная точка REST API для программного доступа к возможностям поиска и управления ресурсами каталога для службы каталога данных Azure. Эта конечная точка API поиска и конечная точка API каталога устарели, и их поддержка прекращена 21 марта 2016 года. Изменения в семантике API отсутствуют. Изменился только универсальный код ресурса (URI) конечной точки. Дополнительные сведения см. в статье [Справочник по API REST каталога данных](https://msdn.microsoft.com/library/azure/mt267595.aspx). * Поддержка источников данных Hadoop HDFS. Пользователи теперь могут регистрироваться и обнаруживать файлы и каталоги HDFS.
* Поддержка предоставления информации о запросе на доступ для зарегистрированных источников данных. Пользователи теперь могут предоставлять указания по запросам доступа для любого зарегистрированного ресурса данных, включая ссылки в электронных сообщениях или URL-адреса, что упрощает интеграцию с существующими инструментами и процессами.
* Всплывающие подсказки для тегов и экспертов, которые позволяют быстрее обнаруживать, какие пользователи предоставили метаданные для зарегистрированных ресурсов данных и тип этих метаданных.
* Добавлена новая кнопка "Пользователь" и меню в верхней панели навигации. Это меню позволяет пользователю просмотреть учетную запись, используемую для входа в каталог данных Azure, и при необходимости выйти из каталога. Кроме того, в этом меню отображается имя каталога, которое может пригодиться разработчикам, использующим REST API каталога данных Azure.
* Только для выпуска "Стандартный": теперь в качестве владельцев ресурсов данных каталог данных Azure позволяет добавлять и учетные записи пользователей, и группы безопасности. Чтобы добавить группу безопасности в качестве владельца для выбранных ресурсов данных, можно ввести отображаемое имя группы или электронный адрес участника-пользователя группы, если он имеется.
* Поддержка источников данных хранилища больших двоичных объектов Azure. Пользователи теперь могут регистрироваться и обнаруживать большие двоичные объекты и каталоги службы хранилища Azure.
