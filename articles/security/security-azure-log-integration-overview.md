# <a name="introduction-to-microsoft-azure-log-integration"></a>Введение в службу интеграции журналов Microsoft Azure
Узнайте о службе интеграции журналов Azure, ее основных возможностях и принципах работы.

## <a name="overview"></a>Обзор

Служба интеграции журналов Azure — это бесплатное решение, которое позволяет интегрировать необработанные журналы из ресурсов Azure с локальными системами управления сведениями о безопасности и событиями (SIEM).

Служба интеграции журналов Azure собирает события из журналов средства просмотра событий Windows, [журналов действий Azure](../monitoring-and-diagnostics/monitoring-overview-activity-logs.md), [оповещений центра безопасности Azure](../security-center/security-center-intro.md) и [журналов диагностики Azure](../monitoring-and-diagnostics/monitoring-overview-of-diagnostic-logs.md), которые поступают из ресурсов Azure. Благодаря такой интеграции решение SIEM может обеспечить доступ ко всем ресурсам, локальным или облачным, на унифицированной панели мониторинга, что позволяет выполнять статистическую обработку, сопоставление и анализ, а также оповещать о событиях безопасности.

>[!NOTE]
В настоящее время поддерживаются только облако Azure для коммерческих организаций и облако Azure для государственных организаций. Другие облака не поддерживаются.

## <a name="what-logs-can-i-integrate"></a>Какие журналы можно интегрировать?
Azure создает подробные журналы для каждой службы Azure. Эти журналы делятся на три типа.

* **Журналы управления и контроля**, которые предоставляют информацию о таких операциях [Azure Resource Manager](../azure-resource-manager/resource-group-overview.md), как CREATE, UPDATE и DELETE. К этому типу относятся, например, журналы действий Azure.
* **Журналы плоскости данных**, которые предоставляют информацию о событиях, возникающих при использовании ресурса Azure. Примерами журнала этого типа являются каналы средства просмотра событий Windows **System**, **Security** и **Application** на виртуальной машине Windows. Другой пример — ведение журнала диагностики, настраиваемое с помощью Azure Monitor.
* Журнал **Обработанные событий** предоставляет информацию о проанализированных событиях и оповещениях, обработанных от вашего имени. К событиям такого типа относятся оповещения центра безопасности Azure, генерируемые им после обработки и анализа вашей подписки для определения вашей текущей системы безопасности.

Интеграция журналов Azure поддерживает ArcSight, QRadar и Splunk. В любом случае обратитесь к поставщику SIEM, чтобы выяснить, есть ли у него собственный соединитель. В некоторых случаях не требуется использование журналов Azure, если доступны собственные соединители. Дополнительные сведения о типах поддерживаемых журналов см. в часто задаваемых вопросах.

>[!NOTE]
Хотя служба интеграции журналов Azure — бесплатное решение, за хранение файлов журналов в службе хранилища Azure взимается плата.

Помощь сообщества можно получить на [форуме MSDN по интеграции журналов Azure](https://social.msdn.microsoft.com/Forums/office/home?forum=AzureLogIntegration). На этом форуме участники сообщества AzLog могут помогать друг другу, делясь вопросами, ответами, советами и рекомендациями о том, как максимально использовать возможности интеграции журналов Azure. Кроме того, команда разработчиков службы интеграции журналов Azure отслеживает этот форум и помогает при любой возможности.

Можно также отправить [запрос в службу поддержки](../azure-supportability/how-to-create-azure-support-request.md). Для этого укажите в запросе службу **Интеграции журналов**.
