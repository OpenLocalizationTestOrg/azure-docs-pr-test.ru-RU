# <a name="physical-access-requirements-for-pci-dss-compliant-environments"></a>Требования к физическому доступу для сред, соответствующих стандарту PCI DSS 
## <a name="pci-dss-requirement-9"></a>Требование 9 (стандарт PCI DSS)

**Ограничение физического доступа к данным владельца карты**

> [!NOTE]
> Эти требования определены [Советом по стандартам безопасности индустрии платежных карт (PCI)](https://www.pcisecuritystandards.org/pci_security/) как часть [Стандарта безопасности данных (DSS) PCI версии 3.2](https://www.pcisecuritystandards.org/document_library?category=pcidss&document=pci_dss). Сведения о процедурах тестирования и рекомендации по каждому требованию см. в документации стандарта PCI DSS.

Любой физический доступ к данным или системам, в которых хранятся данные владельцев карт, дает возможность отдельным лицам обращаться к устройствам или данным и удалять системы или печатные копии и должен быть соответствующим образом ограничен. В рамках требования 9 к "внутреннему персоналу" относятся сотрудники на условиях полной и частичной занятости, временные сотрудники, подрядчики и консультанты, физически находящиеся в локальном расположении организации. "Посетитель" относится к поставщику, гостю любого внутреннего персонала, работникам служб или любому, кто должен зайти на объект на короткий срок, обычно не более одного дня. "Носители" относится ко всем бумажным и электронным носителям, содержащим данные владельца карты.

## <a name="pci-dss-requirement-91"></a>Требование 9.1 (стандарт PCI DSS)

**9.1**. Используйте соответствующие средства контроля входа в организацию для ограничения и контроля физического доступа к системам в среде данных владельцев карт.

**Обязанности:&nbsp;&nbsp;`Microsoft Azure Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Microsoft Azure отвечает за внедрение, обеспечение и мониторинг безопасности физического доступа для центров обработки данных. |
| **Клиент<br />(проект PCI‑DSS&nbsp;)** | Не применяется|



### <a name="pci-dss-requirement-911"></a>Требование 9.1.1 (стандарт PCI DSS)

**9.1.1**. Используйте видеокамеры либо механизмы контроля доступа (или и то, и другое), чтобы контролировать индивидуальный физический доступ к закрытым областям. Просмотрите собранные данные и сопоставьте их с другими записями. Храните их в течение как минимум трех месяцев, если законом не установлено иное.

> [!NOTE]
> "Закрытые области" относятся к любому центру обработки данных, серверной комнате или любой области, где расположены системы, которые хранят, обрабатывают или передают данные владельцев карт. Это исключает общественные области, где присутствуют только терминалы розничной продажи, например кассовые зоны в магазине.

**Обязанности:&nbsp;&nbsp;`Microsoft Azure Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Microsoft Azure отвечает за реализацию и работу систем видеонаблюдения и биометрического контроля доступа, а также контроль за ними для центров обработки данных. |
| **Клиент<br />(проект PCI‑DSS&nbsp;)** | Не применяется|



### <a name="pci-dss-requirement-912"></a>Требование 9.1.2 (стандарт PCI DSS)

**9.1.2**. Реализуйте физические или логические средства управления для ограничения доступа к общедоступным сетевым разъемам. 

Например, сетевые разъемы, расположенные в общественных местах и ​​в зонах, доступных для посетителей, могут быть отключены и включаться только при явном разрешении доступа к сети. В качестве альтернативы могут быть реализованы процессы постоянного сопровождения посетителей в зонах с активными сетевыми разъемами.

**Обязанности:&nbsp;&nbsp;`Microsoft Azure Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | В платформе Microsoft Azure отсутствуют общедоступные сетевые разъемы. |
| **Клиент<br />(проект PCI‑DSS&nbsp;)** | Не применяется|



### <a name="pci-dss-requirement-913"></a>Требование 9.1.3 (стандарт PCI DSS)

**9.1.3**. Ограничьте физический доступ к точкам беспроводного доступа, шлюзам, карманным устройствам, сетевым и коммуникационным аппаратным средствам и линиям связи.

**Обязанности:&nbsp;&nbsp;`Microsoft Azure Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Физический доступ к сетевому оборудованию Microsoft Azure надежно контролируется с помощью списков доступа, нескольких форм аутентификации, физических барьеров входа, и доступ к оборудованию должен быть одобрен для определенной бизнес-задачи. |
| **Клиент<br />(проект PCI‑DSS&nbsp;)** | Не применяется|



## <a name="pci-dss-requirement-92"></a>Требование 9.2 (стандарт PCI DSS)

**9.2**. Разработайте процедуры, позволяющие легко различать внутренний персонал и посетителей, включая:
- определение внутреннего персонала и посетителей (например, ношение значков);
- изменения требований к доступу;
- отзыв или прекращение действия удостоверения внутреннего персонала и просроченный пропуск посетителя (например, идентификационный значок).

**Обязанности:&nbsp;&nbsp;`Microsoft Azure Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Microsoft Azure отвечает за внедрение, обеспечение и мониторинг безопасности физического доступа и идентификацию сотрудников или подрядчиков при посещении центров обработки данных. |
| **Клиент<br />(проект PCI‑DSS&nbsp;)** | Не применяется|



## <a name="pci-dss-requirement-93"></a>Требование 9.3 (стандарт PCI DSS)

**9.3**. Контролируйте физический доступ внутреннего персонала к закрытым областям следующим образом:
- Доступ должен быть разрешен и предоставляться на основе отдельных выполняемых задач.
- Доступ отменяется сразу после завершения, а все механизмы физического доступа, такие как ключи, карты доступа и т. д., возвращаются или отключаются.

**Обязанности:&nbsp;&nbsp;`Microsoft Azure Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Разрешения доступа к центрам обработки данных Майкрософт контролируются с использованием списка доступа, одобренного командой центра обработки данных, на основе принципа минимальных привилегий. Список контроля доступа пересматривается, проверяется и обновляется ежеквартально.<br /><br />В центрах обработки данных Microsoft Azure используются устройства физического доступа, такие как ворота периметра, считыватели электронных карточек доступа, биометрические считыватели, шлюзовые кабины, порталы и устройства, исключающие повторное использование пропусков. Устройства карточек доступа постоянно контролируются. |
| **Клиент<br />(проект PCI‑DSS&nbsp;)** | Не применяется|

### <a name="pci-dss-requirement-993"></a>Требование 9.9.3 (стандарт PCI DSS)
