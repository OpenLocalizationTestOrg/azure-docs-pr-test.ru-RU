* <span data-ttu-id="40a35-152">**Регрессия увеличивающегося дерева принятия решений** использует преимущества перекрытия компонентов или взаимодействия между ними.</span><span class="sxs-lookup"><span data-stu-id="40a35-152">**Boosted decision tree regression** takes advantage of feature overlap or interaction among features.</span></span> <span data-ttu-id="40a35-153">Это означает, что в любой заданной точке данных по значению одного компонента можно в некоторой степени спрогнозировать значение другого.</span><span class="sxs-lookup"><span data-stu-id="40a35-153">That means that, in any given data point, the value of one feature is somewhat predictive of the value of another.</span></span> <span data-ttu-id="40a35-154">Например, при анализе ежедневных данных по высокой и низкой температуре, зная низкую температуру за день, можно сделать правдоподобное предположение относительно высокой температуры.</span><span class="sxs-lookup"><span data-stu-id="40a35-154">For example, in daily high/low temperature data, knowing the low temperature for the day allows you to make a reasonable guess at the high.</span></span> <span data-ttu-id="40a35-155">Информация, содержащаяся в двух компонентах, отчасти избыточна.</span><span class="sxs-lookup"><span data-stu-id="40a35-155">The information contained in the two features is somewhat redundant.</span></span>

* <span data-ttu-id="40a35-156">Классифицировать данные на несколько категорий можно с помощью изначально многоклассового классификатора или объединив набор двухклассовых классификаторов в **множество**.</span><span class="sxs-lookup"><span data-stu-id="40a35-156">Classifying data into more than two categories can be done by either using an inherently multi-class classifier, or by combining a set of two-class classifiers into an **ensemble**.</span></span> <span data-ttu-id="40a35-157">При использовании множества для каждого класса есть отдельный двухклассовый классификатор, который разделяет данные на две категории — "этот класс" и "другой класс".</span><span class="sxs-lookup"><span data-stu-id="40a35-157">In the ensemble approach, there is a separate two-class classifier for each class - each one separates the data into two categories:  “this class” and “not this class.”</span></span> <span data-ttu-id="40a35-158">Затем эти классификаторы выбирают правильное назначение для точки данных.</span><span class="sxs-lookup"><span data-stu-id="40a35-158">Then these classifiers vote on the correct assignment of the data point.</span></span> <span data-ttu-id="40a35-159">Это рабочий принцип, лежащий в основе [многоклассовой классификации "один — все"][one-vs-all-multiclass].</span><span class="sxs-lookup"><span data-stu-id="40a35-159">This is the operational principle behind [One-vs-All Multiclass][one-vs-all-multiclass].</span></span>

* <span data-ttu-id="40a35-160">В некоторых методах, включая логистическую регрессию и байесовскую точечную машину, предполагаются **границы линейного класса**.</span><span class="sxs-lookup"><span data-stu-id="40a35-160">Several methods, including logistic regression and the Bayes point machine, assume **linear class boundaries**.</span></span> <span data-ttu-id="40a35-161">То есть то, что границы между классами предположительно являются прямыми линиями (или гиперплоскостями в более общем случае).</span><span class="sxs-lookup"><span data-stu-id="40a35-161">That is, they assume that the boundaries between classes are approximately straight lines (or hyperplanes in the more general case).</span></span> <span data-ttu-id="40a35-162">Зачастую это характеристика данных, неизвестная до того момента, пока вы не попытались выделить ее, но обычно ее можно заранее узнать с помощью визуализации.</span><span class="sxs-lookup"><span data-stu-id="40a35-162">Often this is a characteristic of the data that you don’t know until after you’ve tried to separate it, but it’s something that typically can be learned by visualizing beforehand.</span></span> <span data-ttu-id="40a35-163">Если границы класса выглядят очень нерегулярными, воспользуйтесь деревьями принятия решений, джунглями принятия решений, методом опорных векторов или нейронными сетями.</span><span class="sxs-lookup"><span data-stu-id="40a35-163">If the class boundaries look very irregular, stick with decision trees, decision jungles, support vector machines, or neural networks.</span></span>

* <span data-ttu-id="40a35-164">Нейронные сети можно использовать с категориальными переменными, создав **вспомогательную переменную** для каждой категории и задавая для нее значение 1 в случаях, где категория применяется, и 0, когда она не применяется.</span><span class="sxs-lookup"><span data-stu-id="40a35-164">Neural networks can be used with categorical variables by creating a **dummy variable** for each category, setting it to 1 in cases where the category applies, 0 where it doesn’t.</span></span>
