# <a name="azure-container-registry-skus"></a>Номера SKU реестра контейнеров Azure

Реестр контейнеров Azure (ACR) доступен на нескольких уровнях обслуживания, известных как номера SKU. Эти номера SKU предоставляют прогнозируемые цены и несколько параметров для выравнивания для емкость и использование шаблонов частного реестра Docker в Azure.

| SKU | Управляемые | ОПИСАНИЕ |
| --- | :-------: | ----------- |
| **базовая;** | Yes | Недорогой начальный уровень для разработчиков, изучающих реестр контейнеров Azure. Реестры уровня "Базовый" располагают теми же программными возможностями, что и реестры уровней "Стандартный" и "Премиум" (интеграция проверки подлинности Azure Active Directory, удаление образов и веб-перехватчики), но на этом уровне действуют ограничения, касающиеся размера и использования. |
| **Стандартный** | Yes | Реестры уровня "Стандартный" предоставляют те же возможности, что и реестры уровня "Базовый", но больший размер хранилища и более высокую пропускную способность образов. Реестры уровня "Стандартный" подходят для большинства рабочих сценариев. |
| **Премиальный** | Yes | В реестрах уровня "Премиум" ограничения расширены. Это касается объема хранилища и количества параллельных операций, что делает возможными сценарии с большим объемом данных. Помимо выше изображения пропускной способности "премиум" добавлены функции, такие как [георепликации] [ container-registry-geo-replication] для управления одной реестра по нескольким регионам, обслуживание сети закрыть реестра для каждого развертывание. |
| Классический | Нет  | Номер SKU реестра уровня "Классический" включил первоначальный выпуск службы реестра контейнеров Azure в Azure. Реестры уровня "Классический" поддерживаются учетной записью хранения, которую создает Azure в подписке, из-за чего ACR предоставляет меньше возможностей более высокого уровня, таких как повышенная пропускная способность и георепликация. Из-за ограниченных возможностей номера SKU уровня "Классический" мы планируем в будущем отказаться от него. |

При выборе более высокого уровня SKU предоставляет дополнительные производительность и масштабируемость, однако все управляемые конфигурации обеспечивают те же программные возможности. С несколько уровней обслуживания можно начать работу с Basic, то преобразование в Standard и Premium, как ваш увеличения использования реестра.

> [!NOTE]
> В связи с запланированным отказом от номера SKU реестра уровня "Классический" мы рекомендуем использовать уровни "Базовый", "Стандартный" и "Премиум" для всех новых реестров. Сведения о преобразовании существующих классический реестра см. в разделе [обновление реестра классический][container-registry-upgrade].
>

## <a name="managed-vs-unmanaged"></a>Управляемые и неуправляемые

Basic, Standard и Premium SKU называются *управляемых* реестрами и реестров классический как *неуправляемые*. Основное различие между ними является, как хранятся образы контейнеров.

### <a name="managed-basic-standard-premium"></a>Управляемые (Basic, Standard, Premium)

Управляемый реестров, поддерживаемых управляется Azure учетную запись хранилища Azure. То есть учетной записи хранилища, в котором хранятся образы не появляется в вашей подписке Azure. Есть несколько преимуществ, полученных с помощью одного из управляемых реестра SKU, рассматриваются подробные в [обновление реестра классический][container-registry-upgrade]. Эта статья посвящена SKU управляемого реестра и их возможности.

### <a name="unmanaged-classic"></a>Неуправляемые (классические)

Классический реестры являются «неуправляемого» в том смысле, что учетной записи хранилища, поддерживающего реестра классический находится в пределах *вашей* подписки Azure. Таким образом вы несете ответственность за управление учетной записи хранилища, в котором хранятся образы контейнера. С неуправляемой реестров переключаться между нельзя SKU по мере изменения потребностей (отличного от [обновление] [ container-registry-upgrade] управляемого реестр), и несколько возможностей управляемого реестров недоступны (например, Удаление образа контейнера, [георепликации] [ container-registry-geo-replication] и [веб-перехватчиков][container-registry-webhook]).

Дополнительные сведения об обновлении реестра классический до одного из управляемых SKU см [обновление реестра классический][container-registry-upgrade].

## <a name="changing-skus"></a>Изменение номеров SKU

Конфигурация реестра на портале Azure или Azure CLI можно изменить. Вы может свободно перемещаться между управляемого SKU, пока SKU, чтобы перейти к имеет необходимые максимальную емкость. При переключении в один из управляемых продуктов из классического невозможно переместить обратно на обычную--он является необратимым.

### <a name="azure-cli"></a>Инфраструктура CLI Azure

Для перемещения между SKU в Azure CLI, используйте [обновление записи контроля доступа az] [ az-acr-update] команды. Например, чтобы перейти на Premium:

```azurecli
az acr update --name myregistry --sku Premium
```

### <a name="azure-portal"></a>Портал Azure

В реестре контейнеров **Обзор** на портале Azure выберите **обновление**, затем выберите новый **SKU** из раскрывающегося списка SKU.

![Обновление номера SKU реестра контейнеров на портале Azure][update-registry-sku]

При наличии реестра классический нельзя выбрать управляемый SKU на портале Azure. Вместо этого необходимо сначала [обновление] [ container-registry-upgrade] управляемого реестр (см. [изменение из классического](#changing-from-classic)).

## <a name="changing-from-classic"></a>Переход с уровня "Классический"

Существуют дополнительные рекомендации необходимо учитывать при переносе неуправляемые классический реестра для одного из управляемых Basic, Standard или Premium SKU. Если в реестре классический содержит большое количество изображений и значительно больше, процесс миграции может занять некоторое время. Кроме того `docker push` операции недоступны до завершения миграции.

Дополнительные сведения об обновлении реестра классический до одного из управляемых SKU см [обновление реестра контейнера классический][container-registry-upgrade].
