# <a name="complete-an-access-review-of-members-of-a-group-or-users-access-to-an-application-in-azure-ad"></a>Завершение проверки доступа для участников группы или пользователей с доступом к приложению с помощью Azure AD

Администраторы могут запросить проверку участников группы или пользователей, назначенных для приложения, с помощью [создания проверки доступа](active-directory-azure-ad-controls-create-access-review.md). Azure AD автоматически отправляет проверяющим сообщение, в котором им предлагается проверить доступ. Если пользователь не получил сообщение, ему можно отправить [инструкции по выполнению проверки доступа](active-directory-azure-ad-controls-perform-access-review.md).  По завершении периода проверки доступа или после того, как администратор остановил проверку, нужно следовать инструкциям в этой статье, чтобы просмотреть и применить результаты.

## <a name="viewing-an-access-review-in-the-azure-portal"></a>Просмотр проверки доступа на портале Azure

1. Перейдите к [странице проверки доступа](https://portal.azure.com/#blade/Microsoft_AAD_ERM/DashboardBlade/), а затем на вкладку **Программы** и выберите программу, содержащую элемент управления проверки доступа.
2. Щелкните пункт управления и выберите элемент управления проверки доступа.  Если в программе есть много элементов управления, вы можете отфильтровать для отображения элементов определенного типа, сортировать по их состоянию или искать по имени элемента управления проверки доступа или отображаемого имени владельца, который его создал. 

## <a name="stopping-a-review-that-has-not-yet-completed"></a>Остановка невыполненной проверки

Если проверка еще не достигла запланированной даты окончания, то администратор может остановить ее, нажав кнопку **Остановить**, чтобы завершить раньше.  Если к этому моменту еще не все пользователи были проверены, то они уже не смогут пройти проверку после ее остановки. Нельзя перезапустить проверку после ее остановки.

## <a name="applying-the-changes"></a>Применение изменений 

После того, как проверка доступа завершается из-за достижения даты окончания или из-за остановки администратором, вы можете реализовать результат проверки с помощью кнопки **Применить**, тем самым обновив группу или приложение. Если пользователю было отказано в доступе во время проверки, когда администратор нажмет эту кнопку, Azure AD удалит членство или назначение в приложении. 

Нажатие кнопки "Применить" не будет влиять на динамические или на созданные в локальном каталоге группы.  Если вы хотите изменить группу, которая создана в локальной среде, загрузите результаты и примените изменения к представлению группы в этом каталоге.

## <a name="downloading-the-results-of-the-review"></a>Загрузка результатов проверки

Чтобы получить результаты проверки, перейдите на вкладку "Утверждения" и нажмите кнопку **Скачать**.  Полученный CSV-файл можно просмотреть в Excel или в других программах, в которых открываются CSV-файлы.

## <a name="optional-deleting-a-review"></a>Удаление проверки (необязательно)
Если проверка вам больше не нужна, удалите ее. При нажатии кнопки **Удалить** проверка будет удалена из Azure AD.

> [!IMPORTANT]
> При удалении проверки предупреждение не выдается. Поэтому убедитесь, что выбрана нужная проверка.
