Каждый пользователь настраивает свой сайт самостоятельно.</span><span class="sxs-lookup"><span data-stu-id="325ac-118">**Self-service group management** An example of this scenario is two users who ## <a name="making-a-group-available-for-end-user-self-service"></a><span data-ttu-id="325ac-125">Включение функции самообслуживания для пользователей группы</span><span class="sxs-lookup"><span data-stu-id="325ac-125">Making a group available for end user self-service</span></span>

2. <span data-ttu-id="325ac-127">На вкладке **Настройка** задайте для параметра **Делегированное управление группой** значение "Включено".</span><span class="sxs-lookup"><span data-stu-id="325ac-127">On the **Configure** tab, set **Delegated group management** to Enabled.</span></span>
3. <span data-ttu-id="325ac-128">Затем задайте значение "Включено" для параметра **Пользователи могут создавать группы безопасности** или **Users can create Office groups** (Пользователи могут создавать группы Office).</span><span class="sxs-lookup"><span data-stu-id="325ac-128">Set **Users can create security groups** or **Users can create Office groups** to Enabled.</span></span>

<span data-ttu-id="325ac-129">Если параметр **Пользователи могут создавать группы безопасности** включен, всем пользователям в каталоге разрешено создавать новые группы безопасности и добавлять в них пользователей.</span><span class="sxs-lookup"><span data-stu-id="325ac-129">When **Users can create security groups** is enabled, all users in your directory are allowed to create new security groups and add members to these groups.</span></span> <span data-ttu-id="325ac-130">Эти новые группы также будут отображаться на панели доступа для всех остальных пользователей.</span><span class="sxs-lookup"><span data-stu-id="325ac-130">These new groups would also show up in the Access Panel for all other users.</span></span> <span data-ttu-id="325ac-131">Пользователи смогут создавать запросы на присоединение к таким группам, если это разрешено параметром политики для группы.</span><span class="sxs-lookup"><span data-stu-id="325ac-131">If the policy setting on the group allows it, other users can create requests to join these groups.</span></span> <span data-ttu-id="325ac-132">Если параметр **Пользователи могут создавать группы безопасности** отключен, пользователи не могут создавать новые и изменять имеющиеся группы, владельцами которых они являются.</span><span class="sxs-lookup"><span data-stu-id="325ac-132">If **Users can create security groups** is disabled, users can't create groups and can't change existing groups for which they are an owner.</span></span> <span data-ttu-id="325ac-133">Однако они могут управлять членством в этих группах и утверждать запросы на присоединение к группам от других пользователей.</span><span class="sxs-lookup"><span data-stu-id="325ac-133">However, they can still manage the memberships of those groups and approve requests from other users to join their groups.</span></span>

<span data-ttu-id="325ac-134">С помощью параметра **Пользователи, которые могут использовать самостоятельное управление группами безопасности** можно также более точно управлять доступом благодаря создавать группы** включен, всем пользователям в каталоге разрешено создавать новые группы и добавлять в них пользователей.</span><span class="sxs-lookup"><span data-stu-id="325ac-135">When **Users can create groups** is enabled, all users in your directory are allowed to create new groups and add members to these groups.</span></span> <span data-ttu-id="325ac-136">Задав для параметра **Пользователи, которые могут использовать самостоятельное управление группами безопасности** значение Some (Некоторые), вы ограничите управление группами, предоставив его только определенной группе могут использовать самостоятельное управление группами безопасности** значение "Все", вы разрешите всем пользователям в каталоге создавать новые группы.</span><span class="sxs-lookup"><span <span data-ttu-id="325ac-139">Кроме того, в поле **Группа, которая может использовать самостоятельное управление группами безопасности** можно указать пользовательское имя для группы, пользователи которой имеют право использовать самостоятельное управление.</span><span class="sxs-lookup"><span data-stu-id="325ac-139">You can also use the **Group that can use self-service for security groups** box to specify a custom name for a group whose members can use self-service.</span></span>
