# <a name="azure-active-directory-proof-of-concept-playbook-ingredients"></a>Сборник тренировочных заданий по подтверждению концепции для Azure Active Directory: ингредиенты 

## <a name="theme"></a>Тема
Azure AD предоставляет решения для управления удостоверениями и доступом по нескольким областям предприятия. Мы классифицируем сценарии по следующим областям: 

* [Большое количество приложений, одно удостоверение](active-directory-playbook-implementation.md#theme---lots-of-apps-one-identity) 
* [Повышение уровня безопасности](active-directory-playbook-implementation.md#theme---increase-your-security) 
* [Самостоятельное масштабирование](active-directory-playbook-implementation.md#theme---scale-with-self-service) 

Определение темы для PoC помогает сконцентрировать усилия, что перекликается с бизнес-целями, которые зачастую являются главными триггерами интереса к подтверждению концепции. 

## <a name="environment"></a>Среда

Важно детально определить среду, в которой будет реализовываться подтверждение концепции. В идеале на ее основе можно будет продолжить работу после выполнения PoC. Целевая среда является критически важной, поэтому необходимо найти правильный баланс между ее максимальной реалистичностью и необходимостью учитывать различные ограничения и рекомендации. Ниже описаны типичные среды для PoC.
* **Рабочая среда.** Сценарии будут реализованы в динамической среде с использованием уже развернутых облачных служб Майкрософт (рабочая среда AD, Office 365, клиент Azure AD или решение единого входа). 
* **Пользовательский тест приемки (UAT) или среда разработки.** Вам предоставляется инфраструктура тестирования (параллельная среда AD и потенциальный клиент Azure AD или решение единого входа) с тестовыми данными, напоминающими рабочую среду. Как правило, тестовая среда совместно используется несколькими проектами на предприятии.

Большинство сценариев в этом руководстве аддитивно по своей природе. Поэтому их можно развернуть в рабочем клиенте без влияния на работу пользователей за пределами PoC. В этом документе мы будем отмечать, какие сценарии могут оказывать влияние на работу клиента. Для таких случаев вы можете использовать другую (не рабочую) среду. 


## <a name="target-users"></a>Целевые пользователи

Важно определить целевой набор пользователей, которые будут испытывать эти сценарии, в частности, когда среда является рабочей или тестовой. Ниже описаны категории целевых пользователей для PoC:
* **Пилотные пользователи.** Реальные пользователи в среде, которые будут использовать решение с учетной записью, применяемой для ежедневных рабочих заданий.
* **Тестовые пользователи.** Тестовые учетные записи, созданные в тестовой среде. 

Большинство сценариев в этом руководстве может быть выполнено пилотными пользователями. В этом документе мы будем при необходимости отмечать целевых пользователей, в отношении которых есть особые рекомендации.
