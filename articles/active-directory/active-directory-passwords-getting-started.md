# <a name="quickstart-azure-ad-self-service-password-reset"></a>Быстрое начало работы с самостоятельным сбросом пароля в Azure AD

> [!IMPORTANT]
> **Вы здесь потому, что возникают проблемы при входе?** Если это так, [с помощью этих инструкций можно изменить и сбросить пароль](active-directory-passwords-update-your-own-password.md).

## <a name="rapidly-deploy-self-service-password-reset"></a>Быстрое развертывание самостоятельного сброса пароля

С помощью системы самостоятельного сброса пароля (SSPR) ИТ-администраторы могут предоставить пользователям возможность самостоятельно сбрасывать или разблокировать их пароли и учетные записи. Эта система предоставляет подробные отчеты, которые позволяют отслеживать ее использование, а также создает уведомления в случае несанкционированного использования или нарушения использования.

В этом руководстве предполагается, что у вас уже есть рабочий пробный или лицензированный клиент Azure AD. Сведения о настройке Azure AD см. на странице [бесплатной пробной версии Azure Active Directory](https://azure.microsoft.com/trial/get-started-active-directory/).

1. В имеющемся клиенте Azure AD выберите **Сброс пароля**.

2. В окне **Свойства** для параметра "Разрешен самостоятельный сброс пароля" выберите одно из следующих значений:
    * "Никто". Никто не может использовать функцию SSPR.
    * "Группа". Только члены указанной группы Azure AD могут использовать функцию SSPR.
    * "Все". Все пользователи с учетными записями в вашем клиенте Azure AD могут использовать функцию SSPR.

3. В окне **Способы проверки подлинности** выберите одно из следующих значений:
    * "Число способов, необходимых для сброса". Минимально поддерживаемое число — 1, максимально поддерживаемое — 2.
    * "Способы, доступные пользователям". Вам понадобится по крайней мере один способ, но можно выбрать несколько.
        * **Письмо.** На указанный адрес электронной почты для проверки подлинности отправляется письмо с кодом.
        * **Мобильный телефон.** Пользователь может принять вызов или получить сообщение с кодом на указанный номер мобильного телефона.
        * **Рабочий телефон.** На указанный номер служебного телефона выполняется вызов, и пользователю сообщается код.
        * **Контрольные вопросы**. Пользователь должен указать следующие параметры:
            * "Число вопросов, необходимое для регистрации". Минимальное число вопросов, необходимое для регистрации. Пользователь может ответить на большее количество, чтобы создать пул вопросов. Для этого параметра можно задать значение в диапазоне от 3 до 5. Количество определенных вопросов должно быть не меньше числа вопросов, необходимого для регистрации.
                * Чтобы добавить личные вопросы, при выборе секретных вопросов нажмите кнопку "Настройка".
            * "Число вопросов, необходимое для выполнения сброса". Для этого параметра можно задать значение в диапазоне от 3 до 5. Правильно ответив на указанное число вопросов, пользователь сможет сбросить или разблокировать пароль.

4. Рекомендуется. Параметр **Настройка** позволяет изменить страницу или адрес электронной почты, по которому выполняется переход после нажатия ссылки "Связаться с администратором".

5. Необязательно. В окне **Регистрация** доступны следующие параметры:
    * "Требовать регистрацию пользователей при входе"
    * "Количество дней, по истечении которых пользователям будет предложено повторно подтвердить данные проверки подлинности"

6. Необязательно. В окне **Уведомление** доступны следующие параметры:
    * "Уведомлять пользователей о сбросе пароля";
    * "Уведомлять всех администраторов, если другие администраторы сбрасывают свои пароли".

**На этом этапе вы настроили SSPR для своего клиента Azure AD**. Вы можете остановиться здесь или настроить синхронизацию паролей с локальным доменом AD.

> [!NOTE]
> Тестируйте SSPR с использованием учетной записи пользователя, а не администратора, так как Майкрософт имеет строгие требования к проверке подлинности учетных записей Azure типа "администратор". Дополнительные сведения о политике паролей администратора см. в [статье о политике паролей](active-directory-passwords-policy.md#administrator-password-policy-differences).

## <a name="configure-synchronization-to-existing-identity-source"></a>Настройка синхронизации с имеющимся источником удостоверений

Чтобы включить синхронизацию локального источника удостоверений с Azure AD, установите и настройте [Azure AD Connect](./connect/active-directory-aadconnect.md) на сервере организации. Это приложение выполняет синхронизацию пользователей и групп имеющегося источника удостоверений с клиентом Azure AD.

* [Обновление с DirSync или Azure AD Sync до Azure AD Connect](./connect/active-directory-aadconnect-dirsync-deprecated.md)
* [Приступая к работе с Azure AD Connect с использованием стандартных параметров](./connect/active-directory-aadconnect-get-started-express.md)
* [Настройте обратную запись паролей](active-directory-passwords-writeback.md#configuring-password-writeback) для записи паролей Azure AD обратно в ваш локальный каталог.

## <a name="disabling-self-service-password-reset"></a>Выключение самостоятельного сброса пароля

Отключить самостоятельный сброс пароля так же просто, как и включить. Откройте клиент Azure AD, выберите **Сброс пароля > Свойства**, а затем для параметра **Разрешен самостоятельный сброс пароля** установите значение **Нет**.
