# <a name="understanding-resource-access-in-azure"></a>Основные сведения о доступе к ресурсам в Azure

Управление доступом в Azure начинается с оплаты. Владелец учетной записи Azure, доступ к которой осуществляется через [Центр управления учетной записью Azure](https://account.windowsazure.com/subscriptions), является администратором учетной записи (АУ). Подписки — это контейнеры для оплаты, которые в то же время обеспечивают защиту: у каждой подписки есть администратор службы, который может добавлять, удалять и изменять ресурсы Azure в этой подписке на [портале Azure](https://portal.azure.com/). По умолчанию АУ новой подписки становится и ее АС, однако АУ может изменить АС в центре учетных записей Azure.

<br><br>![Учетные записи Azure][1]

С подпиской также связан каталог. Каталог определяет набор пользователей. Это могут быть пользователи из компании или учебного заведения, в которой был создан каталог, или они могут быть внешние гостевых пользователей. Подписки доступны подмножеству тех пользователей, которые были назначены администраторами служб (SA) или Соадминистраторами (CA); Единственным исключением является то, что, для обратной совместимости, учетные записи Майкрософт (ранее Windows Live ID) можно назначать в качестве SA или CA без их присутствия в каталоге.

<br><br>![Управление доступом в Azure][2]

Функциональные возможности портала Azure позволяют SA, которые вошли под учетной записью Майкрософт для изменения каталога, с которым связана подписка. Эта операция влияет на управление доступом этой подписки.

<br><br>![Простой пользовательский процесс входа в][3]

В самом простом случае организация (например, Contoso) принудительно внедряет выставление счетов и управление доступом в один и тот же набор подписок. То есть каталог привязывается к подпискам, принадлежащим одной и той же учетной записи Azure. После успешного входа на портал Azure пользователи видят две коллекции ресурсов (показаны оранжевым цветом на предыдущем рисунке).

* Каталоги, в которых существует их учетная запись пользователя (созданная или добавленная как внешний субъект). Обратите внимание, что каталог, используемый для входа в систему, в данном случае не имеет значения, поэтому ваши каталоги будут отображаться всегда, где бы вы ни выполняли вход.
* Ресурсы, составляющие часть подписок, связанных с каталогом, который используется для входа в систему и к которому пользователь может получить доступ (будь то АС или СА).

<br><br>![Пользователь с несколькими подписками и каталогами][4]

Пользователи с подписками в нескольких каталогах имеют возможность переключения текущего контекста портала Azure с помощью фильтра подписки. При этом осуществляется отдельный вход в другой каталог, хотя за счет единого входа пользователь этого не замечает.

В связи с единым видом каталога такие операции, как перемещение ресурсов между подписками, могут усложняться. Чтобы выполнить перенос ресурсов, может потребоваться впервые использовать команду **Изменить каталог** на странице "Подписки" диалогового окна **Параметры**, чтобы привязать подписки к одному и тому же каталогу.
