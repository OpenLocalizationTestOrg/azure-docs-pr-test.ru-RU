# <a name="create-an-access-review-of-group-members-or-application-access-with-azure-ad"></a>Создание проверки доступа для участников группы или приложения с помощью Azure AD

Назначения доступа становятся "устаревшими", когда у пользователей имеются права доступа, которые им больше не нужны. Чтобы уменьшить риск, связанный с устаревшими назначениями доступа, администраторы могут использовать Azure Active Directory (Azure AD), чтобы создать проверку доступа участников группы или пользователей, назначенных для приложения. Дополнительные сведения об этих сценариях см. в разделах [Управление пользовательским доступом с помощью проверок доступа Azure AD](active-directory-azure-ad-controls-manage-user-access-with-access-reviews.md) и [Управление гостевым доступом с помощью проверок доступа Azure AD](active-directory-azure-ad-controls-manage-guest-access-with-access-reviews.md). 

## <a name="create-an-access-review"></a>Создание проверки доступа

1. В качестве глобального администратора перейдите на [страницу проверки доступа](https://portal.azure.com/#blade/Microsoft_AAD_ERM/DashboardBlade/), а затем выберите вкладку **Программы**.

2. Выберите программу, содержащую элемент управления проверки доступа, который требуется создать. Можно выбрать **программу по умолчанию**, которая всегда доступна, или создать другую программу. Например, вы можете использовать одну программу для каждой инициативы по обеспечению соответствия или бизнес-цели.

3. В программе выберите **Элементы управления**, а затем щелкните **Добавить**, чтобы добавить элемент управления.

4. Укажите имя для проверки доступа. При необходимости укажите описание каждой проверки. Имя отображается для проверяющих.

5. Задайте даты начала и окончания. По умолчанию проверка доступа начинается в тот же день, когда она создается, и завершается через один месяц. Можно изменить даты начала и окончания таким образом, чтобы проверка доступа началась в будущем и длилась необходимое количество дней.

6. Проверки доступа могут выполняться для участников группы или пользователей, которые были назначены для приложения. Можно также сузить область проверки доступа только до гостевых пользователей, которые являются участниками (или назначены для приложения), вместо того чтобы проверять всех пользователей-участников или пользователей с доступом к приложению.

7. Выберите одного или нескольких пользователей для проверки всех пользователей в области. Или можно указать, чтобы участники сами проверяли собственный доступ. Если ресурс является группой, вы можете попросить владельцев группы выполнить проверку. Можно также потребовать, чтобы рецензенты указывали причину при подтверждении доступа.

8. Наконец, щелкните **Запуск**.


## <a name="manage-the-access-review"></a>Управление проверкой доступа

По умолчанию Azure AD отправляет электронные сообщения рецензентам вскоре после запуска проверки. Если вы выбрали не отправлять сообщения через службу Azure AD, обязательно сообщите рецензентам, что им необходимо выполнить проверку доступа. Вы можете показать им инструкции по [проверке доступа](active-directory-azure-ad-controls-perform-access-review.md). Если проверка предназначена для проверки гостями собственного доступа, покажите им инструкции по [проверке собственного доступа](active-directory-azure-ad-controls-perform-access-review.md).

Если некоторые из рецензентов являются гостями, они будут уведомлены по электронной почте, только если уже приняли приглашение.


Отслеживать ход выполнения проверяющими их проверок можно на панели мониторинга Azure AD в разделе **Проверки доступа**. Права доступа в каталоге не изменяются до [завершения проверки](active-directory-azure-ad-controls-complete-access-review.md).

## <a name="next-steps"></a>Дальнейшие действия

После запуска проверки доступа Azure AD будет автоматически отправлять рецензентам электронное сообщение, предлагающее им проверить доступ. Если пользователь не получил электронное сообщение, ему можно отправить инструкции по [проверке доступа](active-directory-azure-ad-controls-perform-access-review.md). 

По завершении периода проверки доступа или после того, как администратор остановил проверку, следуйте инструкциям в статье [Завершение проверки доступа для участников группы или пользователей с доступом к приложению с помощью Azure AD](active-directory-azure-ad-controls-complete-access-review.md).
