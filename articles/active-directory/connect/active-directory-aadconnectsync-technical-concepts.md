## <a name="connector"></a><span data-ttu-id="e96c2-115">Соединитель</span><span class="sxs-lookup"><span data-stu-id="e96c2-115">Connector</span></span>
<span data-ttu-id="e96c2-116">Соединителями называются модули кода, используемые для взаимодействия с подключенным каталогом (ранее они назывались агентами управления).</span><span class="sxs-lookup"><span data-stu-id="e96c2-116">The code modules that are used to communicate with a connected directory are called connectors (formerly known as  management agents (MAs)).</span></span>

<span data-ttu-id="e96c2-118">Соединители предоставляют возможность безагентного преобразования путем использования протоколов удаленных систем вместо развертывания специализированных агентов.</span><span class="sxs-lookup"><span data-stu-id="e96c2-118">The connectors provide the agentless ability to converse by using remote system protocols instead of relying on the deployment of specialized agents.</span></span> <span data-ttu-id="e96c2-119">Это означает снижение риска и сокращение времени развертывания, особенно при работе с критически важными приложениями и системами.</span><span class="sxs-lookup"><span data-stu-id="e96c2-119">This means decreased risk and deployment times, especially when dealing with critical applications and systems.</span></span>

<span data-ttu-id="e96c2-120">На рисунке выше соединитель является синонимом пространства соединителя, но включает все взаимодействие с внешней системой.</span><span class="sxs-lookup"><span data-stu-id="e96c2-120">In the picture above, the connector is synonymous with the connector space but encompasses all communication with the external system.</span></span>

<span data-ttu-id="e96c2-121">Соединитель отвечает за все функции импорта и экспорта в системе. Кроме того, он освобождает разработчиков от необходимости разбираться в том, как обеспечить встроенное подключение к каждой системе при использовании декларативной подготовки для настройки преобразований данных.</span><span class="sxs-lookup"><span data-stu-id="e96c2-121">The connector is responsible for all import and export functionality to the system and frees developers from needing to understand how to connect to each system natively when using declarative provisioning to customize data transformations.</span></span>

<span data-ttu-id="e96c2-122">Операции импорта и экспорта выполняются только по расписанию. Это обеспечивает дополнительную изоляцию от изменений, происходящих в системе, так как изменения не распространяются автоматически на подключенные источники данных.</span><span class="sxs-lookup"><span data-stu-id="e96c2-122">Imports and exports only occur when scheduled, allowing for further insulation from changes occurring within the system, since changes do not automatically propagate to the connected data source.</span></span> <span data-ttu-id="e96c2-123">Кроме того, разработчики могут также создавать собственные соединители для подключения практически к любому источнику данных.</span><span class="sxs-lookup"><span data-stu-id="e96c2-123">In addition, developers may also create their own connectors for connecting to virtually any data source.</span></span>

## <a name="connected-system"></a><span data-ttu-id="e96c2-132">Подключенная система</span><span class="sxs-lookup"><span data-stu-id="e96c2-132">Connected system</span></span>

## <a name="connector-space"></a><span data-ttu-id="e96c2-134">Пространство соединителя</span><span class="sxs-lookup"><span data-stu-id="e96c2-134">Connector space</span></span>
<span data-ttu-id="e96c2-135">Каждый подключенный источник данных представляется как отфильтрованное подмножество объектов и атрибутов в пространстве соединителя.</span><span class="sxs-lookup"><span data-stu-id="e96c2-135">Each connected data source is represented as a filtered subset of the objects and attributes in the connector space.</span></span>
<span data-ttu-id="e96c2-136">Это позволяет службе синхронизации работать локально без необходимости подключения к удаленной системе во время синхронизации объектов, а также ограничивать взаимодействие только операциями импорта и экспорта.</span><span class="sxs-lookup"><span data-stu-id="e96c2-136">This allows the sync service to operate locally without the need to contact the remote system when synchronizing the objects and restricts interaction to imports and exports only.</span></span>

<span data-ttu-id="e96c2-137">Если источник данных и соединитель могут предоставить список изменений (разностный импорт), эффективность работы существенно возрастает. ## <a name="metaverse"></a><span data-ttu-id="e96c2-140">Метавселенная</span><span class="sxs-lookup"><span data-stu-id="e96c2-140">Metaverse</span></span>
<span data-ttu-id="e96c2-141">Метавселенная — это консолидированное представление всех присоединенных удостоверений из соседних пространств соединителей.</span><span class="sxs-lookup"><span data-stu-id="e96c2-141">The metaverse is the consolidated view of all joined identities from neighboring connector spaces.</span></span>

<span data-ttu-id="e96c2-142">Удостоверения связываются вместе, а центр для различных атрибутов назначается через сопоставление потоков импорта. В итоге центральный объект метавселенной начинает вычислять данные, полученные из нескольких систем.</span><span class="sxs-lookup"><span data-stu-id="e96c2-142">As identities are linked together and authority is assigned for various attributes through import flow mappings, the central metaverse object begins to aggregate information from multiple systems.</span></span> <span data-ttu-id="e96c2-143">В ходе сопоставления сведения из этого потока атрибутов объекта переносятся в исходящие системы.</span><span class="sxs-lookup"><span data-stu-id="e96c2-143">From this object attribute flow, mappings carry information to outbound systems.</span></span>

<span data-ttu-id="e96c2-144">Объекты создаются, когда полномочная система передает их в метавселенную.</span><span class="sxs-lookup"><span data-stu-id="e96c2-144">Objects are created when an authoritative system projects them into the metaverse.</span></span> <span data-ttu-id="e96c2-145">Как только все подключения удаляются, удаляется и объект метавселенной.</span><span class="sxs-lookup"><span data-stu-id="e96c2-145">As soon as all connections are removed, the metaverse object is deleted.</span></span>

<span data-ttu-id="e96c2-146">Объекты в метавселенной нельзя изменить непосредственно.</span><span class="sxs-lookup"><span data-stu-id="e96c2-146">Objects in the metaverse cannot be edited directly.</span></span> <span data-ttu-id="e96c2-147">Все данные в объекте должны изменяться с помощью потока атрибутов.</span><span class="sxs-lookup"><span data-stu-id="e96c2-147">All data in the object must be contributed through attribute flow.</span></span> <span data-ttu-id="e96c2-148">Метавселенная поддерживает постоянные соединители в каждом пространстве соединителя.</span><span class="sxs-lookup"><span data-stu-id="e96c2-148">The metaverse maintains persistent connectors with each connector space.</span></span> <span data-ttu-id="e96c2-149">Эти соединители не требуют повторной оценки для каждой синхронизации.</span><span class="sxs-lookup"><span data-stu-id="e96c2-149">These connectors do not require reevaluation for each synchronization run.</span></span> <span data-ttu-id="e96c2-150">Это означает, что Azure AD Connect Sync не должна каждый раз обнаруживать удаленный объект.</span><span class="sxs-lookup"><span data-stu-id="e96c2-150">This means that Azure AD Connect sync does not have to locate the matching remote object each time.</span></span> <span data-ttu-id="e96c2-151">Это устраняет потребность в использовании затратных агентов для предотвращения изменений в атрибутах, которые обычно должны отвечать за сопоставление объектов.</span><span class="sxs-lookup"><span data-stu-id="e96c2-151">This avoids the need for costly agents to prevent changes to attributes that would normally be responsible for correlating the objects.</span></span>

<span data-ttu-id="e96c2-152">При обнаружении новых источников данных, в которых уже могут существовать нуждающиеся в управлении объекты, в службе синхронизации Azure AD Connect используется специальный процесс. Он называется правилом присоединения и применяется для оценки потенциальных кандидатов, с которыми будет установлена связь.</span><span class="sxs-lookup"><span data-stu-id="e96c2-152">When discovering new data sources that may have preexisting objects that need to be managed, Azure AD Connect sync uses a process called a join rule to evaluate potential candidates with which to establish a link.</span></span>
<span data-ttu-id="e96c2-153">После установки связи эта оценка не повторяется. Между подключенным удаленным источником данных и метавселенной может проходить обычный поток атрибутов.</span><span class="sxs-lookup"><span data-stu-id="e96c2-153">Once the link is established, this evaluation does not reoccur and normal attribute flow can occur between the remote connected data source and the metaverse.</span></span>

## <a name="provisioning"></a><span data-ttu-id="e96c2-154">Подготовка</span><span class="sxs-lookup"><span data-stu-id="e96c2-154">Provisioning</span></span>
<span data-ttu-id="e96c2-155">Когда полномочный источник передает новый объект в метавселенную, в другом соединителе может быть создан новый объект пространства соединителя. <span data-ttu-id="e96c2-157">Процесс, во время которого правило определяет, что необходимо создать новый объект пространства соединителя, называется подготовкой.</span><span class="sxs-lookup"><span data-stu-id="e96c2-157">Whenever a rule determines that a new connector space object needs to be created, it is called provisioning.</span></span> <span data-ttu-id="e96c2-158">Но поскольку эта операция происходит только в пространстве соединителя, она переносится в подключенный источник данных только в ходе экспорта.</span><span class="sxs-lookup"><span data-stu-id="e96c2-158">However, because this operation only takes place within the connector space, it does not carry over into the connected data source until an export is performed.</span></span>
