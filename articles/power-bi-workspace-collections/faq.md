# <a name="power-bi-workspace-collections-faq"></a>Вопросы и ответы о коллекциях рабочих областей Power BI

> [!IMPORTANT]
> Использовать службу "Коллекции рабочих областей Power BI" не рекомендуется. Она будет доступна до июня 2018 года или до даты, указанной в договоре. Мы советуем организовать перенос приложения в Power BI Embedded, чтобы избежать прерываний в его работе. Сведения о том, как перенести данные из коллекций рабочих областей Power BI в Power BI Embedded, см. в [этой статье](https://powerbi.microsoft.com/documentation/powerbi-developer-migrate-from-powerbi-embedded/).

## <a name="what-is-microsoft-power-bi-workspace-collections"></a>Что такое коллекция рабочих областей Microsoft Power BI?
Эта служба Azure позволяет разработчикам встраивать прекрасные, полностью интерактивные отчеты и визуализации в клиентские приложения, не тратя время и деньги на создание собственных элементов управления с нуля. Коллекция рабочих областей Power BI доступна с соглашением об уровне обслуживания в девяти центрах обработки данных по всему миру. Теперь она содержит усовершенствованные функциональные возможности, включая поддержку защиты данных с применением безопасности на уровне строк для улучшенной фильтрации. Кроме того, мы упростили и обновили модель ценообразования для коллекций рабочих областей Power BI.

## <a name="who-would-want-to-use-microsoft-power-bi-workspace-collections-and-why"></a>Кому и чем будут полезны коллекции рабочих областей Microsoft Power BI?
Коллекции рабочих областей Microsoft Power BI предназначены для разработчиков приложений, которые хотят предложить своим пользователям широкие интерактивные возможности для визуализации данных на любых устройствах без необходимости создавать визуализации самостоятельно. Благодаря коллекциям рабочих областей Power BI разработчики могут предоставлять самые актуальные представления в сочетании с прямыми запросами. Разработчики также могут программно разворачивать, автоматизировать службу Power BI и управлять ею с помощью API-интерфейсов Azure Resource Manager и Power BI. Как и во всем, что касается Power BI, встроенная служба автоматически масштабируется в соответствии с использованием и потребностями вашего приложения. Служба коллекций рабочих областей Power BI использует модель ценообразования с оплатой по мере использования.

## <a name="how-does-power-bi-workspace-collections-relate-to-the-power-bi-service"></a>Как служба коллекций рабочих областей Power BI связана со службой Power BI?
Коллекции рабочих областей Power BI и служба Power BI — это разные предложения. Коллекции рабочих областей Power BI используют модель ценообразования с оплатой по мере использования и развертываются на портале Azure. С помощью этой службы независимые поставщики программного обеспечения могут встраивать в свои приложения визуализации данных для использования клиентами. Служба Power BI тарифицируется и развертывается через портал O365. Служба Power BI — это автономное предложение BI общего назначения, предназначенное преимущественно для внутреннего корпоративного использования. Дополнительные сведения о службе Power BI см. на сайте [www.powerbi.com](https://powerbi.microsoft.com).

## <a name="how-does-power-bi-workspace-collections-improve-my-app"></a>Как коллекции рабочих областей Power BI помогут улучшить приложение?
Использование прекрасных интерактивных визуализаций данных прямо в приложении позволяет пользователям оперативно принимать решения и значительно улучшает приложение. Коллекции рабочих областей Power BI позволяют улучшить приложение с помощью интерактивных и актуальных визуализаций данных с широкими возможностями. Вы сможете оптимизировать удобство использования, повышая удовлетворенность и лояльность пользователей, а также легко передавать контекстную аналитику на любые устройства.

## <a name="are-there-any-rules-or-restrictions-about-how-i-can-use-power-bi-workspace-collections-in-my-app"></a>Существуют ли все правила или ограничения по использованию коллекций рабочих областей Power BI в приложении?
Коллекции рабочих областей Power BI предназначены для приложений, предоставляемых внешним пользователям. Если вы хотите использовать коллекции рабочих областей Power BI для создания внутреннего бизнес-приложения, всем внутренним пользователям нужно предоставить лицензии USL Power BI Pro. При этом кроме лицензионных платежей с вашей организации будет взиматься плата за использование службы коллекций рабочих областей Power BI. Чтобы пользователи могли избежать двойной оплаты за лицензию USL Power BI Pro и использование коллекций рабочих областей Power BI при работе с внутренними приложениями, в службе Power BI предусмотрены собственные средства встраивания содержимого, которые не используют коллекции рабочих областей Power BI и предоставляются бесплатно владельцам подписки USL Power BI (dev.powerbi.com).

## <a name="can-power-bi-workspace-collections-be-used-to-create-internal-applications"></a>Можно ли создавать внутренние приложения с помощью коллекций рабочих областей Power BI?
Нет. Коллекции рабочих областей Power BI предназначены для внешнего использования и их не следует применять во внутренних бизнес-приложениях. Чтобы встроить содержимое Power BI для использования во внутренних бизнес-приложениях, вам нужно использовать службу Power BI. Все пользователи, которые используют это содержимое, должны иметь действительную лицензию на пользовательскую подписку Power BI (бесплатная версия) или Power BI Pro. Дополнительные сведения об интеграции внутренних приложений со службой Power BI см. на сайте [https://dev.powerbi.com](https://dev.powerbi.com).

## <a name="is-this-service-available-globally"></a>Эта служба доступна по всему миру?
Служба коллекций рабочих областей Power BI уже доступна в большинстве центров обработки данных. Актуальные сведения о доступности можно просмотреть [здесь](https://azure.microsoft.com/status/).

## <a name="what-is-the-available-sla-for-the-service"></a>Какое соглашение об уровне обслуживания доступно для службы?
Для коллекций рабочих областей Power BI предоставляется стандартное соглашение об уровне обслуживания Azure. Дополнительные сведения см. в на странице [соглашений об уровне обслуживания](https://azure.microsoft.com/support/legal/sla/).

## <a name="what-is-a-report-session-and-how-is-it-billed"></a>Что такое сеанс отчетов и как за него выставляются счета?
Сеанс — это последовательность взаимодействий между пользователем и отчетом коллекций рабочих областей Power BI. Сеанс инициируется при каждом отображении отчета коллекций рабочих областей Power BI для пользователя. Счет за каждый сеанс выставляется владельцу подписки. Сеансы тарифицируются по фиксированной ставке независимо от количества визуальных элементов в отчете или частоты обновления его содержимого. Сеанс завершается, когда пользователь закрывает отчет, или по истечении времени ожидания сеанса — через час.

## <a name="do-you-offer-any-tools-or-guidance-to-help-me-estimate-how-many-renderssession-i-should-expect-how-will-i-know-how-many-renders-have-been-completed"></a>Существуют ли какие-то инструменты или рекомендации, которые помогут оценить предполагаемое количество отрисовок или сеансов? Как узнать о том, сколько отрисовок выполнено?
Сведения о числе сеансов работы с визуализациями и (или) отчетами, выполненных и подлежащих тарификации для вашей подписки, отображаются на портале Azure.

## <a name="do-i-need-a-power-bi-subscription-in-order-to-develop-applications-with-power-bi-workspace-collections-how-do-i-get-started"></a>Необходима ли подписка Power BI для разработки приложений с использованием коллекций рабочих областей Power BI? Как начать работу?
Для создания отчетов и визуализаций, которые будут использоваться в приложении, вам как разработчику приложения подписка Power BI не требуется. Вам нужна подписка Microsoft Azure и бесплатное приложение Power BI Desktop.

Сведения о том, как использовать службу коллекций рабочих областей Power BI, приводятся в документации по нашей службе.

## <a name="i-have-an-azure-subscription-can-i-use-power-bi-workspace-collections-using-my-existing-subscription"></a>У меня есть подписка Azure. Можно ли использовать коллекции рабочих областей Power BI с существующей подпиской?
Да. Для подготовки и использования службы коллекции рабочих областей Microsoft Power BI можно использовать существующую подписку Azure.

## <a name="does-my-application-end-user-need-a-power-bi-license"></a>Нужна ли лицензия Power BI конечным пользователям моего приложения?
Нет. Пользователям вашего приложения не требуется приобретать отдельную подписку Power BI для доступа к визуализациям данных в приложении. В модели коллекций рабочих областей Power BI вам, как поставщику приложения, выставляется счет за использование службы по данным индикатора потребления Azure. Дополнительные сведения см. на [странице сведений о ценах и лицензировании](http://go.microsoft.com/fwlink/?LinkId=760527).

## <a name="how-does-user-authentication-work-with-power-bi-workspace-collections"></a>Как выполняется аутентификация пользователей в коллекциях рабочих областей Power BI?
Вместо явной аутентификации пользователей служба коллекции рабочих областей Power BI использует маркеры приложений. В модели с маркерами приложений ваше приложение самостоятельно отслеживает аутентификацию и авторизацию пользователей. При необходимости ваше приложение создает

и отправляет маркеры приложений, которые сообщают службе о том, что необходимо подготовить запрошенный отчет. Этот подход не требует использования Azure AD для проверки подлинности и авторизации пользователей, хотя вы можете это сделать. Дополнительные сведения о маркерах приложений см. [здесь](app-token-flow.md). Для расширенных сценариев фильтрации безопасности в коллекциях рабочих областей Power BI мы внедрили функцию безопасности на уровне строк.

## <a name="what-data-sources-are-currently-supported-with-power-bi-workspace-collections"></a>Какие источники данных сейчас поддерживаются коллекциями рабочих областей Power BI?
Будет поддерживаться доступ к облачным источникам данных, которые используют базовые учетные данные через прямые запросы. Это означает, что прямо сейчас поддерживаются такие источники данных, как база данных SQL Azure и хранилище данных SQL Azure. В ближайшие месяцы мы добавим поддержку других источников данных и типов доступа. Дополнительные сведения см. в разделе [Подключение к источнику данных](connect-datasource.md).

## <a name="how-does-the-tenancy-model-work-for-power-bi-workspace-collections"></a>Как работает клиентская модель для коллекций рабочих областей Power BI?
В модели коллекций рабочих областей Power BI явно не требуется, чтобы ваши клиенты были клиентами Azure AD. Вы можете потребовать от своих клиентов использовать Azure AD, а можете и не делать этого. Это означает, что клиентская модель для коллекций рабочих областей Power BI полностью определяется архитектурой и инфраструктурой вашего приложения.

Разработчикам и сотрудникам, работающим над созданием вашего приложения, потребуется учетная запись AAD для управления подпиской Azure и коллекциями рабочих областей на портале Azure. Программные интерфейсы API для импорта отчетов, изменения строк подключения и получения URL-адресов встраивания для проверки подлинности используют маркеры приложения и поэтому не требуют AAD.

## <a name="how-do-i-get-started"></a>Как начать работу?
Сейчас вы можете начать работу бесплатно! Теперь, если у вас есть подписка Azure, вы можете подготовить коллекции рабочих областей Power BI прямо на портале Azure. Вы также можете создать [бесплатную учетную запись Azure](https://azure.microsoft.com/free/). Когда вы подготовите службу коллекций рабочих областей Power BI, вы сможете легко использовать интерфейсы REST API Power BI напрямую или через пакет SDK для разработчиков, который доступен на сайте [GitHub](http://go.microsoft.com/fwlink/?LinkID=746472). В нем также представлены примеры по использованию пакета SDK для разработчиков.
