# <a name="microsoft-genomics-common-questions"></a>Геномика корпорации Майкрософт: Часто задаваемые вопросы

В этой статье перечислены самых популярных запросов, которые могут быть связаны с Геномика корпорации Майкрософт. Дополнительные сведения о службе Microsoft Геномика см. в разделе [новые возможности Microsoft Геномика?](overview-what-is-genomics.md) 


## <a name="what-is-the-sla-for-microsoft-genomics"></a>Что такое соглашение об уровне ОБСЛУЖИВАНИЯ для Геномика Майкрософт?
Мы гарантируем, что 99,9% службы Геномика Microsoft времени будут доступны для получения запросов API рабочего процесса. См. дополнительные сведения о [Соглашении об уровне обслуживания](https://azure.microsoft.com/support/legal/sla/genomics/v1_0/).

## <a name="how-does-the-usage-of-microsoft-genomics-show-up-on-my-bill"></a>Как использование Microsoft Геномика не отображаются на мой счет?
Microsoft Геномика счетов на основе количества gigabases обработки для каждого рабочего процесса. Дополнительные сведения см. в разделе [цены](https://azure.microsoft.com/pricing/details/genomics/).


## <a name="where-can-i-find-a-list-of-all-possible-commands-and-arguments-for-the-msgen-client"></a>Где можно найти список всех возможных команды и аргументы для `msgen` клиента?
Полный список доступных команд и аргументы можно получить, выполнив `msgen help`. Если дополнительные аргументы не указан, он отображает список доступную справку в разделах, один для каждого из `submit`, `list`, `cancel`, и `status`. Чтобы получить справку по определенной команде, введите `msgen help command`, например `msgen help submit` перечислены все параметры отправки.

## <a name="what-are-the-most-commonly-used-commands-for-the-msgen-client"></a>Каковы наиболее часто используемые команды для `msgen` клиента?
Наиболее часто используемые команды, аргументы для `msgen` клиента включают: 

 |**Команда**          |  **Описание поля** |
 |:--------------------|:-------------         |
 |`list`               |Возвращает список заданий, которые вы отправили. Для аргументов, в разделе `msgen help list`.  |
 |`submit`             |Отправляет запрос к службе рабочего процесса. Для аргументов, в разделе `msgen help submit`.|
 |`status`             |Возвращает состояние рабочего процесса, определяемого `--workflow-id`. См. также `msgen help status`. |
 |`cancel`             |Отправляет запрос на отмену обработки рабочего процесса, определяемого `--workflow-id`. См. также `msgen help cancel`. |

## <a name="where-do-i-get-the-value-for---api-url-base"></a>Где получить значение для `--api-url-base`?
Перейдите на портал Azure и откройте страницу Геномика учетной записи. В разделе **управления** заголовок, выберите **ключи доступа**. Найти, URL-адрес API и ключи доступа.

## <a name="where-do-i-get-the-value-for---access-key"></a>Где получить значение для `--access-key`?
Перейдите на портал Azure и откройте страницу Геномика учетной записи. В разделе **управления** заголовок, выберите **ключи доступа**. Найти, URL-адрес API и ключи доступа.

## <a name="why-do-i-need-two-access-keys"></a>Почему необходимо два ключа доступа?
Требуется два ключа доступа в случае, если вы хотите обновить (повторное создание) их, не создавая помех для использования службой. Например требуется обновить первый ключ. В этом случае все новые рабочие процессы перейти на использование второго ключа. Подождите завершения уже запущенные процессы с помощью первого ключа. Только после обновления ключа.

## <a name="do-you-save-my-storage-account-keys"></a>Следует ли сохранять Мои ключи учетной записи хранения?
Ключ учетной записи хранилища используется для создания краткосрочной токены доступа для службы Microsoft Геномика входные файлы читать и записывать выходные файлы. Срок действия маркера по умолчанию — 48 часов. Срок действия маркера можно изменить при помощи `-sas/--sas-duration` параметр команды "Отправить"; значение указано в часах.

## <a name="how-do-i-format-my-command-line-arguments-as-a-config-file"></a>Как форматировать my аргументы командной строки в файл конфигурации? 

msgen принимает файлы конфигурации в следующем формате:
* Все параметры предоставляются как пары «ключ значение» со значениями, разделенными точкой с запятой из ключей.
Пробелы игнорируются.
* Строки, начинающиеся с `#` игнорируются.
* Любой из аргументов командной строки в полном формате можно преобразовать с ключом, удаление его начальные тире и заменяя тире между словами с символами подчеркивания. Ниже приведены некоторые примеры преобразований.

 |Аргумент командной строки            | Строка файла конфигурации |
 |:-------------                   |:-------------                 |
 |`-u/--api-url-base https://url`  | *api_url_base:https://URL*    |
 |`-k/--access-key KEY`            | *access_key:Key*              |      
 |`-pa/--process-args R=B37m1`     | *process_args:R-b37m1*        |
