# <a name="use-azure-portal-to-assign-and-manage-resource-policies"></a>Назначение политик ресурсов и управление ими с помощью портала Azure
Портал Azure позволяет назначить группам ресурсов и подпискам политики ресурсов. Пользовательский интерфейс позволяет с легкостью выбрать политику, которую требуется назначить, и указать значения параметров политики, чтобы ее настроить. 

Для назначения политики помощью портала в вашей подписке уже должно существовать определение политики. Подписка имеет несколько встроенных определений политики, которые можно назначить группам ресурсов или подпискам. Вы увидите эти встроенные политики, а также все определенные вами пользовательские политики, когда воспользуетесь порталом для назначения политик. Введение в политики, а также сведения об определении пользовательских политик, см. в статье [Общие сведения о политике ресурсов](resource-manager-policy.md).

Политики наследуются всеми дочерними ресурсами. Это значит, что политики, применяемые к группе ресурсов, применяются также ко всем ресурсам в этой группе. В этой статье термин **область** обозначает группу ресурсов или подписку, которым назначена политика. 

Политики оцениваются при создании и обновлении ресурсов (операции PUT и PATCH).

## <a name="assign-a-policy"></a>Назначение политики

1. Чтобы назначить политику группе ресурсов или подписке, выберите соответствующую группу ресурсов или подписку. В разделе "Параметры" выберите **Политики**.

   ![выбор политик](./media/resource-manager-policy-portal/select-policies.png)

2. Чтобы создать назначение политики для этой области, выберите **Добавить назначение**.

   ![добавление назначения](./media/resource-manager-policy-portal/add-assignment.png)

3. Выберите политику, которую необходимо назначить. Даже если вы не добавили в свою подписку ни одного определения политики, отобразятся встроенные политики, которые можно назначить. Эти встроенные политики охватывают множество распространенных сценариев.

   ![выбор определения](./media/resource-manager-policy-portal/select-definition.png)

4. После того, как политика выбрана, отображается описание политики, а также ее параметры. Например, на следующем изображении показан параметр **Allowed locations** (Допустимые расположения), который необходим для политики, ограничивающей доступные расположения.

   ![отображение параметров](./media/resource-manager-policy-portal/show-parameters.png)

5. Через пользовательский интерфейс выберите значения для параметров политики (например, укажите расположения, которые можно использовать для развертывания).

   ![выбор значений параметров](./media/resource-manager-policy-portal/select-parameters.png)

6. Укажите значения остальных параметров. Область назначается автоматически, исходя из колонки, выбранной в начале процесса назначения политики. По окончании нажмите кнопку **ОК** .

   ![определение параметров](./media/resource-manager-policy-portal/define-parameters.png)

  Вы назначили политику для указанной области.

## <a name="view-policy-assignments"></a>Просмотр назначенных политик

После назначения политики она появиться в списке политик для данной области. На вкладке **Сведения** отображается сводка по назначению политики.

![отображение сведений](./media/resource-manager-policy-portal/show-details.png)

На вкладке **Правило назначения** отображается файл JSON для определения политики.

![отображение правила назначения](./media/resource-manager-policy-portal/show-assignment-rule.png)

## <a name="change-an-existing-policy-assignment"></a>Изменение существующего назначения политики

Чтобы изменить политику, щелкните **Изменить назначение** или **Удалить**.

![изменение или удаление назначения](./media/resource-manager-policy-portal/edit-delete-policy.png)

## <a name="assign-custom-policies"></a>Назначение пользовательских политик

Если в подписке определены пользовательские политики, то их можно назначить с помощью портала. Эти политики содержат в своем названии приставку **[Custom]**.

![пользовательские политики](./media/resource-manager-policy-portal/show-custom-policy.png)
