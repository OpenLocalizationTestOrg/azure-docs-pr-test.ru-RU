1. На портале слева щелкните **+** и выполните поиск по запросу "Шлюз виртуальной сети". Найдите **шлюз виртуальной сети** в списке результатов и щелкните соответствующую запись. На странице **Шлюз виртуальной сети** щелкните **Создать** в нижней части страницы, чтобы открыть страницу **Create virtual network gateway** (Создание шлюза виртуальной сети).
2. На странице **Create virtual network gateway** (Создание шлюза виртуальной сети) укажите необходимые значения параметров.

  ![Поля страницы Create virtual network gateway (Создание шлюза виртуальной сети)](./media/vpn-gateway-add-gw-rm-portal-include/gw.png "Поля страницы Create virtual network gateway (Создание шлюза виртуальной сети)")
3. **Имя**. Назовите свой шлюз. Имя шлюза отличается от имени подсети шлюза. Это имя объекта шлюза, который создается.
4. **Тип шлюза**. Выберите **VPN**. VPN-шлюзы используют тип шлюза виртуальной сети **VPN**.
5. **Тип VPN**. Выберите тип VPN, который указан для конфигурации. Для большинства конфигураций требуется тип VPN на основе маршрута.
6. **SKU**. Выберите номер SKU шлюза из раскрывающегося списка. Номера SKU, перечисленные в раскрывающемся списке, зависят от выбранного типа VPN.
7. **Расположение**. В поле **Расположение** укажите расположение виртуальной сети. Если расположение не указывает на регион, в котором размещена виртуальная сеть, такая виртуальная сеть не будет отображаться в раскрывающемся списке с виртуальными сетями.
8. Выберите виртуальную сеть, в которую нужно добавить шлюз. Щелкните **Виртуальная сеть**, чтобы открыть страницу **Выберите виртуальную сеть**. Выберите виртуальную сеть. Если ваша виртуальная сеть не отображается, проверьте, указан ли в поле **Расположение** регион, в котором находится эта сеть.
9. **Общедоступный IP-адрес.** Создается объект общедоступного IP-адреса, которому динамически назначается общедоступный IP-адрес. Щелкните **Общедоступный IP-адрес**, чтобы открыть страницу **Выбрать общедоступный IP-адрес**. Щелкните **+Создать**, чтобы открыть страницу **Создать общедоступный IP-адрес**. Введите имя общедоступного IP-адреса. Нажмите кнопку **ОК** , чтобы сохранить изменения. IP-адрес динамически назначается ресурсу при создании VPN-шлюза. В настоящее время VPN-шлюз поддерживает только *динамическое* выделение общедоступных IP-адресов. Однако это не означает, что IP-адрес изменяется после назначения VPN-шлюзу. Общедоступный IP-адрес изменяется только после удаления и повторного создания шлюза. При изменении размера, сбросе или других внутренних операциях обслуживания или обновления IP-адрес VPN-шлюза не изменяется.
11. **Подписка**. Проследите, чтобы отображалась правильная подписка.
12. **Группа ресурсов**. Параметр группы ресурсов зависит от выбранной виртуальной сети.
1. Не изменяйте сведения в поле **Расположение** после указания предыдущих параметров.
2. Проверьте параметры. В нижней части страницы можно выбрать параметр **Закрепить на панели мониторинга**, если необходимо, чтобы шлюз отображался на панели мониторинга.
3. Щелкните **Создать** , чтобы создать шлюз. После проверки параметров шлюз будет развернут. Создание шлюза может занять до 45 минут.
4. Создав шлюз, можно просмотреть назначенный ему IP-адрес в разделе сведений о странице виртуальной сети. Шлюз будет отображаться как подключенное устройство. Щелкните подключенное устройство (шлюз виртуальной сети), чтобы получить дополнительные сведения.
